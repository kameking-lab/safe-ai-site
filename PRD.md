# PRD.md

## プロダクト名
安全AIサイト

## 目的
労働安全コンサルタントや現場担当者が、法改正・現場リスク・事故知見・朝礼要点を
スマホ/PC/サイネージで素早く確認できる「現場運用ポータル」を作る。

## 現在フェーズの目的
第1弾完了済み機能を安定運用しつつ、第2弾の主要機能を「たたき台」として広く可視化する。

### 第1弾（完了済み）
1. 法改正リスト表示
2. 各法改正のAI要約表示
3. 選択中の法改正に対する質問チャットUI
4. 今日の現場リスク（地域選択・作業種別）
5. 事故DB最小版
6. /signage サイネージ画面（朝礼/常時表示向け）
7. mock/live の土台、API route、E2E/CI の最小基盤

## 対象ユーザー
- 労働安全コンサルタント
- 建設・製造などの現場安全担当
- 中小企業の現場責任者
- 安全衛生に関心のある管理者
- 朝礼広場の進行担当
- ゼネコン社内デジタルサイネージ運用担当

## 対応端末
- スマホ最優先
- PC対応
- サイネージ・常時表示モニター
- レスポンシブWebサイト

## 第2弾（今回着手するたたき台）
1. ホーム画面再構成（現場運用ポータル化）
2. 通知設定UI（警報/法改正/事故DB/朝礼リマインド）
3. メール配信設定UI（宛先、頻度、対象、プレビュー）
4. Eラーニング入口・テーマ・簡易クイズ
5. KY用紙入力画面（印刷/PDF前提）
6. PDF出力入口（軽量実装または差し替え可能な土台）
7. マイク入力のクライアント側たたき台
8. 事故DB拡張（カテゴリ/フィルタ/他機能導線）
9. service/types/route の最小整理

## 技術方針
- Next.js App Router
- TypeScript
- Tailwind CSS
- ダミーデータ/暫定データで段階開発
- 将来Supabase連携しやすい構造
- 将来AI API連携しやすい構造
- mock/live 境界を service 層で維持
- 外部本接続や秘密情報埋め込みはしない

## UI方針
- スマホでは縦並び
- PCでは2カラムまたは横並び
- 難しい操作を避ける
- 初見で何のサイトかわかることを優先
- 朝礼と常時表示で1〜2秒で意味が伝わる短文を優先
