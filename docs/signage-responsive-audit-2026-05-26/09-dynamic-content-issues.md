# 09. 動的コンテンツ表示（6言語・本日の安全啓発・カウントダウン）

## 観点と所見

### 6言語切替
- ラベルは`clamp`で字面サイズは追従するが、言語によりテキスト長が変化（英/葡/西は日本語より長い）→ **高さが増え縦オーバーフローが悪化**。幅方向は折返しで吸収。
- 言語トグル `<select>` 自体は機能。

### 本日の安全啓発（SignageAccidentEducation）
- 日替わりで内容長が変動 → **セクション高さが動的に増減**し、フィット計算に影響。固定高さ前提のレイアウトでは溢れ量が日によって変わる。

### 唱和カウントダウン
- カウントダウン数字 `clamp(4rem,10vw,12rem)` は大きく、セクションが嵩む。`print:hidden`なので印刷では消える（良い）。
- 数字更新（毎秒）でレイアウト高さは実質一定。

### 天気・日付
- 日付＋天気は1行。問題なし。

## 改修方針（→doc11）
- **1画面フィットは「実コンテンツ高さを測ってから縮尺」する方式**にし、6言語・安全啓発の動的高さに自動追従させる（record/lang/啓発内容の変化で再計算）。
- これにより「言語や日替わり内容が変わっても常に1画面に収まる」を保証。
