# 02. スマホ縦向き（375×667〜430×932）

シナリオ: 朝礼での個人スマホ閲覧、6桁コードで手元表示。

## 実測
- iPhone SE 375×667: 実コンテンツ 2491px → **+1824px 超過**（3.7画面分）
- 他: 390×844/412×915/430×932 でも全項目表示には大きくスクロールが必要

## 所見
- フォントは`clamp`下限（主作業1.8rem=28.8px等）で**可読性は良好**。問題は縦の長さ＝スクロール量。
- 個人スマホ閲覧は本来スクロール可だが、**朝礼の"一目"用途では下部（行動目標/指差呼称/安全啓発/唱和）まで辿り着きにくい**。
- ヘッダ操作ボタンが`text-xs px-3 py-1`でタップ標的<44px。`flex-wrap`で縦に嵩む。
- `min-h-screen`(100vh)でURLバー分が隠れる端末あり。

## 改修方針（→doc11）
- スマホ縦は**可読サイズ維持＋セクション余白圧縮**でスクロール量を低減（自動縮尺はかけない＝文字を小さくしない）。
- 重要順（主作業→リスク→行動目標/指差呼称→安全啓発→唱和）の縦順は維持。
- タップ標的を44px以上に。`100dvh`化。
- 任意: アンカー/簡易ページネーション（任意・P2）。
