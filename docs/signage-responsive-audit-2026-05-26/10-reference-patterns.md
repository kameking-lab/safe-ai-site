# 10. 参考事例・パターン

## 紙ベースの朝礼ボード（現場で実際に使われる）
- A判1枚に「日付・作業・危険ポイント・対策・本日の目標・指差呼称」を**1枚＝1画面で一覧**。スクロールの概念が無い＝"一目で全部見える"が本質。
- 示唆: サイネージも**1画面に全項目**が大前提。情報を縦に伸ばさず、面（2カラム等）で配置。

## TV天気予報・ニュースのテロップ
- 画面を**領域分割（グリッド）**して情報密度を上げ、固定の画面枠に収める。文字は遠方視認できる大きさで、かつ枠内に必ず収まる。
- 示唆: 横長画面は**2カラム/グリッド**で高さを使い切りつつ収める。文字は「収まる範囲で最大化」。

## 他社デジタルサイネージ（建設現場の入退場ゲート掲示等）
- 縦型・横型で**専用レイアウト**を持ち、コンテンツは画面にフィット（スクロール無し・自動更新）。
- 示唆: orientation（縦/横）で分岐。縦＝単一カラム、横＝2カラム。常時フィット＋自動更新（本サイトは8秒ポーリング実装済）。

## レスポンシブ技法（採用候補）
- **fit-to-screen（実寸測定→`transform: scale()`）**: 実コンテンツ高さを測り viewport にフィット。動的内容・多言語に強い。最も確実な"必ず収まる"。
- **`100dvh`/`100svh`**: モバイルURLバー対応。
- **orientation/aspect 分岐 + grid**: 横長2カラム・縦長1カラム。
- **`clamp()`（既存）** はフォント流体に有効、ただし**高さ連動でないため単体では不足**→ fit-to-screen と併用。

## 結論
紙の朝礼ボード／TVテロップに倣い「**1画面＝全項目、横長は2カラム、縦長は1カラム、常にフィット**」を実装方針とする。
