# 08. フルスクリーン時の挙動

## 現状
- `toggleFullscreen` が `document.documentElement.requestFullscreen()` を呼び、`fullscreenchange` で状態同期（実装済・良好）。
- フルスクリーン化しても**レイアウトは通常と同じ**（`min-h-screen`の単一カラム）。

## 所見
- フルスクリーンにしても**縦オーバーフローは解消しない**（doc05のとおり1920×1080で+1347px）。フルスクリーン＝画面全体を使うのに、肝心の内容が画面外という状態。
- フルスクリーンAPIは正しく動作。黒帯はブラウザ依存（コンテンツ側のアスペクト比未対応）。

## 改修方針（→doc11）
- 1画面フィット（doc11 P0）が入れば、フルスクリーン時こそ全項目が画面いっぱいに収まり、サイネージとして最大効果。
- フィットは `100dvh`/`100vh` 実寸に対して計算するため、フルスクリーン・非フルスクリーン双方で機能。
- 既存のフルスクリーンAPI・スリープ抑止は**破壊しない**。
