# 05. ノートPC・デスクトップ（1280×720〜1920×1080）

シナリオ: 事務所モニター・現場PCでの掲示、フルスクリーン表示。最頻の掲示シナリオ。

## 実測
- ノートPC 1366×768: 実2101px → **+1333px 超過**
- デスクトップ 1920×1080: 実2427px → **+1347px 超過**（スクショ上、日付＋主作業＋リスク2件までで、リスク3件目以降〜唱和は全て画面外）

## 所見
- **最も使われる"モニター掲示"で半分以上が画面外**。サイネージは通常スクロールされないため、後半セクション（行動目標/指差呼称/安全啓発/唱和カウントダウン）が事実上表示されない。
- 横長（16:9）なのに単一カラムで縦に伸び、左右は余白・下は溢れる二重の非効率。

## 改修方針（→doc11）
- **1画面フィット（自動縮尺＋2カラム化）をP0で適用**。16:9では2カラムにして縦を圧縮し、1080/768の高さに全項目を収める。
- フルスクリーン時も同様にフィット（doc08）。
