# 06. 大画面・4K（2560×1440〜3840×2160）

シナリオ: 現場掲示板・大型モニター・TV投影。遠方から視認。

## 実測
- QHD 2560×1440: 実2443px → **+1003px 超過**
- 4K 3840×2160: 実2443px → **+283px 超過**（縦はほぼ収まるが少し溢れる）

## 所見
- 4Kは縦がほぼ収まるが、**`max-w-[2200px]`＋vwフォント上限**により、3840px幅に対しコンテンツが2200pxで止まり**左右に各約820pxの大余白**。「レイアウトが空きすぎ」（軸6の懸念どおり）。
- 文字は上限（主作業6rem=96px等）で頭打ち。4K大画面・遠方視認ではもっと大きくできる余地。
- QHDは縦が+1003px溢れる（4Kより低いため）。

## 改修方針（→doc11）
- **1画面フィットで縦を収める**（QHD超過解消）。
- 大画面は**自動縮尺の上限を解放 or 2カラムで横幅を活用**し、左右余白を縮小＋文字を画面いっぱいに拡大（遠方視認性向上）。`max-w`を大画面で拡張 or フィット倍率を1超まで許容。
