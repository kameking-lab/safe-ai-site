## 2. 「ごちゃごちゃブロック」の正体（3つの複合）

1. **サーバー側の二重表示（主因）**: `/api/chatbot`・`/api/chatbot/stream` が
   回答本文の末尾に「📎 出典」「🏛 所管省庁資料」「【関連通達・告示】」
   「【関連リーフレット・教材】」「【合わせて確認すべき法令】」「⚠️ 注記」を
   生テキストで追記していた（route.ts 旧531-622行・stream/route.ts 旧325-383行）。
   同じ情報は構造化フィールド（citations / attachedNotices / attachedLeaflets /
   relatedLaws / scopeWarnings）でも返され、UI が絵文字付き折りたたみカードで
   別途表示している。つまり**1回答につき同じ出典・通達の一覧が本文テキストと
   カードで2回ずつ**出ていた。本文側はURLもプレーンテキストでリンクにならず、
   「詳しい説明」折りたたみの中が生URL・記号の壁になっていた。
2. **markdown の生漏れ**: Gemini が出力する `*   **…**` 箇条書き・`---` 水平線を、
   UI のミニレンダラ（`renderBold`＝`**太字**`のみ解釈・whitespace-pre-wrap）が
   記号のまま表示。スマホ390pxで「`*   **労働安全衛生規則…**`」のような
   行が折り返されて読めない。
3. **免責文の三重掲示**: モデルへの指示（旧 AI_DISCLAIMER_SYSTEM_INSTRUCTION）で
   本文末尾に「---＋⚠️ 本回答はAI…」を書かせていたが、UI は入力欄直下に
   常設免責バナーを持ち、共有ビューにも独自バナーがある。

## 3. 直し方（応答の型の再設計・柱0準拠）

**本文=回答だけ、参照・警告=構造化フィールドだけ**に分離（構造的再発防止）。

- サーバー（route.ts / stream/route.ts）: 上記テール追記を全廃。出典・通達・
  リーフレット・関連法令は構造化フィールドのみで返す。Layer 2 条文検証の警告
  （⚠️ 引用条文の検証結果）・範囲外注記も answer 追記をやめ scopeWarnings へ
  一本化（UIの赤警告枠が表示）。モデルが自前で書いた免責・出典風テールは
  新設 `stripAnswerTailBlocks()` で生成直後に除去（防壁）。
- プロンプト（SYSTEM_PROMPT）: 「結論1〜2文→根拠は条文ごとに要点1〜2文の要約
  （逐語引用禁止・全文は参照条文欄が自動表示）→補足、目安600字」
  「箇条書きは「・」、markdown記法（* - # 表 --- コードブロック）使用禁止、
  強調は**太字**のみ」「免責文・出典一覧を本文に書かない」を明記。
- UI（chatbot-panel.tsx / share/[id]/page.tsx）: 新設
  `formatAnswerForDisplay()`＝`stripAnswerTailBlocks`（旧形式の保存済み
  セッション・キャッシュ応答の後方互換）＋`normalizeAnswerMarkdown`
  （`* `箇条書き→「・」、`#`見出し→太字、`---`除去）。表示・コピー・共有・
  音声読み上げ・エクスポートすべて同じ整形を通す。エクスポートには免責を
  末尾1回だけ付与。
- 正確性不可侵: いずれも書式記号と二重表示ブロックのみ対象。結論・条文名・
  数値など法令内容の語句は一字も変えない（`chatbot-answer-format.test.ts` で固定）。

## 4. 精度（GQ12 偽範囲外警告の構造是正）

- GQ02（酸欠×資格）・GQ23（解雇予告）は PR #798（2026-07-04 マージ・本番実測済）で
  ○化済みのため再実装せず（実測: 2026-07-04 本番eval 完全正答 20/21 = 95.2%）。
- 残る既知欠陥 GQ12 の false-scope-warning（間欠）: `detectOutOfScopeLawReferences` が

  回帰テスト2件を `chatbot-enrichment.test.ts` に追加。
