# 02. データ源調査（最重要）

## 2.1 公式系（優先）

### 厚労省 死亡災害DB / 職場のあんぜんサイト 災害事例DB
- 収録: 業種・規模・作業内容・事故型・発生原因・年齢・性別・発生月・都道府県等。本リポジトリに `deaths-mhlw/records-*.jsonl`（2019〜2024、約739件/年）として取込済。
- **会社名・発注者名は無し（匿名）**。
- 取得性: 既に取込済。年次更新（個票は2024まで公開、2025/2026は未公開）。
- 用途: **類型パターン学習・統計・重大災害事例ブラウザの中核**（会社名不要の安全データ）。

### 都道府県労働局「労働基準関係法令違反に係る公表事案」
- 収録: **企業・事業場名称／所在地／公表日／違反法条項／事案概要**。死亡・重傷・重大法令違反が対象。
- ⚠️ **掲載は公表日から約1年で削除**（是正確認等で早期削除も）。各労働局サイトでHTML/PDF。安定APIは無く、様式も局ごとに差。
- 取得性: スクレイピングは様式不安定＋規約配慮＋**永続再掲が期間制限を超える法的問題**（doc04）。**自動取込は非推奨**。利用するなら「出典リンク誘導」に留める。

### 厚労省 重大災害事例集・労働災害統計
- 集計・代表事例。会社名なし。参考リンク・統計の裏付けに利用可。

## 2.2 報道機関（著作権配慮）

- 各社RSS。**既存 news-feed が稼働中**（`scripts/etl/fetch-news-feed.mjs` RSS取得＋キーワード前処理 → `news-ai-judge.mjs` Geminiジャッジ4スコア[関連/著作権/誤情報/重複] → 承認分のみJSON）。NHK・厚労省等、現在10件。
- 取得性: 実装済。**見出し＋自作要約＋出典リンク**のみ（本文転載なし）。会社名は報道に含まれるが、自サイトが事実として再掲＝名誉毀損配慮（doc04）。
- 用途: **最新の重大事故の気づき**（出典リンクで原文へ誘導）。

## 2.3 訟結・送検・判例系（公表事実）

- 送検事案＝上記労働局公表（期間制限あり）。判例は裁判所公開だが収録・要約は労力大。本Phaseは**出典リンク誘導**に留める（自動取込非対象）。

## 2.4 データ源カバレッジ評価

- 取得可能で安全に使えるデータ源: **(1)匿名公式DB（取込済・中核）／(2)報道RSS（実装済・見出し＋要約＋リンク）** の2系統で、コンサルの類型学習・最新把握をカバー。
- 会社名を含む系統（(3)労働局公表／(4)報道の社名）は**出典リンク誘導**で間接利用。常設DB化は法的に非推奨（doc04）。
- **カバレッジ達成度: 安全データで約80**（会社名常設を除く全用途を充足）。会社名常設を含めると技術的には可能だが法的に「高」。

## 2.5 自動取得ETLの実現性

- 匿名公式DB: 取込済（年次）。差分のみコミット方針。
- 報道RSS: 日次cron稼働中（無料枠内）。
- 労働局公表: 自動取込せず（様式不安定＋法的期間制限）。
- → **新規env不要・既存パターン（月次速報ETL/news-feed）流用で安全データは完結**。
