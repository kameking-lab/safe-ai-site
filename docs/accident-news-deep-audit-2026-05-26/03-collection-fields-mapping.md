# 03. 収録項目マッピング（社長要件 × データ源）

社長要件の各項目を、どのデータ源でどこまで埋められるか。○=安全に取得可 / △=限定 / ✕=不可or高リスク。

- **発生日時**: 公式匿名DB ○（発生年月・時間帯）／報道 △（要約依存）
- **場所（都道府県）**: 公式匿名DB ○（prefecture）／報道 △
- **業種**: 公式匿名DB ○（industry）／報道 △
- **作業内容**: 公式匿名DB ○（description/cause内）／報道 △
- **被害（死亡数等）**: 公式匿名DB ○（死亡災害は1件1名基準）／報道 ○（見出し）
- **発生原因（報道判明範囲）**: 公式匿名DB ○（cause/accidentType）／報道 △
- **背景（同種事故頻度）**: **既存事故DB集計から自動算出 ○**（業種×事故型の件数）
- **関連会社名**: 公式匿名DB ✕（匿名）／労働局公表 △（約1年で削除）／報道 △（著作権・名誉毀損）→ **常設掲載は高リスク（doc04）。出典リンク誘導のみ**
- **発注者**: 公共工事は公開だが事故との紐付けは限定／民間は公表対象外 → **✕（名誉毀損リスク特大、本Phase非掲載）**

## マッピング結論

- **会社名・発注者を除く全項目**は、**匿名公式DB＋既存集計**で安全に構造化できる（発生日時・場所・業種・作業・被害・原因・背景）。
- → 「重大災害事例ブラウザ」の収録フィールド: 発生年月・都道府県・業種・作業/原因（description）・事故型・規模・年齢層（公的範囲）・**同種頻度（自動集計）**・**出典（あんぜんサイト/死亡災害DB）**。
- 会社名/発注者は**載せない**（社長承認時のみ安全モードで送検公表リンク誘導、doc05/12）。
- これにより「業種・規模・作業内容で類型化（パターン学習）」「コンサル説明資料」は**会社名なしで達成**。
