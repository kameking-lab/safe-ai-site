# 08. 既存8機能との統合

- **KY**: 該当業種×作業の重大事例を `/ky/paper` で参考表示／「この作業の重大災害事例を見る」`/accident-news?industry=&type=` 導線（既存 hub-filter prefill パターン）。
- **打合せ書（safety-diary）**: 議題の業種・作業に紐づく代表事例を提示（情報提示のみ、現場強制なし）。
- **サイネージ `/ky/morning`**: 既存「本日の安全啓発」（過去労災事例・日替わり・6言語）が稼働中＝重大事例の朝礼啓発は実装済。重大災害事例ブラウザと同じ匿名データ源で整合。
- **化学物質RA**: 化学物質関連事故型（中毒・酸欠・有機溶剤等）→ `/chemical-ra` 誘導。
- **事故DB `/accidents` / `/accidents-analytics`**: 「この類型の全件・統計を見る」相互リンク（統合検索）。
- **チャットボット**: 法的根拠＋事例の同時提示（事例カードから `/chatbot?q=` で関連条文質問、前セッションの context prefill 流用）。
- **法改正追跡**: 改正のきっかけになった事故類型→該当法改正へ（任意・P2）。
- **新着ハブ `/whats-new` / RSS**: 重大事例の新着を media/accident カテゴリで集約（既存 news-hub 流用）。

## 結論
連携はいずれも**情報提示・導線追加**で低リスク。サイネージの安全啓発は既に同種データで稼働しており整合性が高い。会社名なしで全連携が成立。
