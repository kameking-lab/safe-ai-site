# フェーズC: データ投入パイプライン（段階的・全件実在確認）

## アクセス面の重要な実測findings
- 厚労省「あかるい職場応援団」判例インデックスへの**自動取得は HTTP 403 Forbidden** で拒否された。
  → 政府ポータルでも**機械的な一括ハーベストは技術的にブロック**される。フェーズAの「一括取得できる
  商用利用可ソースは無い」という結論が、アクセス面でも裏づけられた。
- したがってデータ拡充は、**WebSearch等で事件名・裁判所・判決年月日を複数の信頼できる出典で確認 →
  当サイト要約＋出典リンクで1件ずつ追加**という、検証可能だが手間のかかるパイプラインになる。
  これは一朝一夕では終わらない。だからこそ毎サイクル着実に積み上げる（社長の方針と一致）。

## パイプライン（毎サイクルの手順）
1. 論点・分野の不足を確認（争点/分野/裁判所/年代のファセット件数＝フェーズBで可視化済）。
2. WebSearchで候補判例を探し、**事件名・裁判所・判決年月日を2つ以上の信頼できる出典**（厚労省ポータル・
   裁判所/CiNii・労働新聞社・大学リポジトリ・労働基準判例検索等）で照合。
3. 照合できたものだけ、`CourtCase` 構造ガード（争点/分野の統制語彙・事実記述の holding・有効ISO日付）に
   合わせて追加。**1件でも確証が取れなければ載せない**（捏造防止）。
4. lint/tsc/test/build を通し、1挑戦1ブランチ→CI緑→マージ。

  正直に示す。判例DBとしての中立性・実務的価値のため。

## 規模に向けた現実的見通し（正直に）
- 上記パイプラインは「正確だが低速」。数百件は**多数サイクルの継続**でしか到達しない。
- 加速の可能性: 法務省「民事判決情報データベース」が一般公開・再利用可能な形で整備されれば、一括取り込みで
  桁が変わる（フェーズAで「未整備」と確認済。公開され次第、再評価して取り込み方式を切り替える）。
- それまでは政府公開・著作権法13条の範囲で、出典付き・実在確認済みのキュレーションを積み上げる。
