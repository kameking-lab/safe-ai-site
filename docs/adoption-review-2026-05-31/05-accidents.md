# 05 事故DB (/accidents) — 安全担当・コンサルが過去事例を調べる

**操作観察**: 「収録 292件（厚労省 253／cur…）」。業種・原因・作業区分で検索、厚労省DBへのリンク、業種別分析レポート・統計ダッシュボード・重大災害事例へのハブ。

## 現状の手段
厚労省「職場のあんぜんサイト」労働災害事例（無料・公式・数千件規模・写真や図解付き）、災害事例集の書籍、社内の過去災害記録。

## 現状の痛み（本当に痛いか）
**中程度。** あんぜんサイトは検索UIが古く絞り込みにくいという不満はある。ただし“公式・網羅”という安心感が強く、コンサルは出典として公式を引用したい。

## このサービスは革命的に解消するか
**しない。** 決定的な問題は2つ。(1)**データ源が厚労省と同じ**で、本サイト収録は292件＝公式の網羅性に大きく劣る。「調べる」用途で件数が足りなければ最初の数回で“載っていない”に当たり離脱する。(2)コンサルは引用の正統性から公式（あんぜんサイト/e-Stat）を一次情報として使いたい。検索UIが多少良くても「これでなければならない」理由にならない。

## スイッチングコスト
低い（無料・登録不要）。だが前述の通りスイッチングコスト以前に“乗り換える便益が無い”。

## 判定: 使わない
便益 < 現状（公式の網羅性）。検索体験の改善だけでは公式を置き換えられない。

## Yesに変える決定的な一押し
1. **公式に無い「分析」を提供**: 業種×事故型×原因のクロス集計、頻度ランキング、季節性、AIによる原因クラスタリング——“事例を引く”のではなく“傾向から対策を導く”価値（統計ダッシュボード/業種別レポートはこの方向。事故DB単体ではなくそちらを主役に）。
2. **KY/RA/裁判例への直結**: 「この事故型→KYのこの危険ポイント→関連条文→該当判例」の動線。事例単体ではなく実務アクションへ繋ぐ。
3. 件数で公式と張り合うのは非現実的。**“調べる”を捨て“分析して使う”に用途を切り替える**のが正解。
