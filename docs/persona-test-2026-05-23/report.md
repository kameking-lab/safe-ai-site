## 2. 判定方法

評価軸4つを下記基準で◯/△/✗:

- 辿り着けたか
  - ◯ = 既存例示質問・URL クエリ・既存ナビ動線でメイン3機能のいずれかに 1 クリック以内で到達
  - △ = 到達するが導線が複数ステップ、または専用ツール存在に気付きにくい
  - ✗ = 該当機能・該当ページが存在しない、または存在しても該当文脈に紐付かない
- 答えが見つかったか
  - ◯ = サイト上で根拠条文 / 事例 / 推奨対策のいずれかが本文に確認できる
  - △ = 部分的にカバー、または 33法令外で範囲外警告が出る、もしくは通達領域に分岐するが現在のサイト網羅が浅い
  - ✗ = 33法令カバー外かつ専用ツールも未整備
- 5分以内で完結したか
  - ◯ = 3クリック以内、情報密度が読みやすい
  - △ = 何度か追加質問が必要、または専用ツールへの遷移が必要
  - ✗ = 用途とサイト機能が根本的にミスマッチ
- また使いたいと思うか
  - ◯ = 条文番号 / 施行日 / 出典 / 事例 / 業種特化等で ChatGPT 汎用回答より明確に上
  - △ = 一部差別化はあるが、汎用 Web 検索とそこまで差がつかない
  - ✗ = 機能不在 / 用途不一致

## 7. 失敗パターン上位5件と根本原因

- パターン1: 「33法令の枠外」の質問で範囲外警告
  - 該当困りごと: D6, D9, C2, C5, E5, A6, E3 (7件)
  - 根本原因: コンテンツ不足。RAG コーパスに労契法、感染症法、貨物自動車運送事業法、道交法、厚労省カスハラ対策指針、土砂崩壊点検通達等が未収録
- パターン2: 「日常運用ネタ供給」型機能の不在
  - 該当困りごと: A4(朝礼ネタ)、A9(KYネタ)、D4(安全集会ネタ)、B2(委員会報告書テンプレ) (4件)
  - 根本原因: 機能不在。年1回の年次計画と単発の chatbot 質問はあるが、「毎日・毎週・毎月の運用ネタを定期供給する」プロアクティブ機能がない
- パターン3: 専用ツール存在の発見性低下
  - 該当困りごと: A10(WBGT)、B4(化学物質RA)、B9(騒音管理区分)、E10(熱中症R7) (4件)
  - 根本原因: 動線断絶。chatbot 応答の digDeeperLinks は実装済みだが、業種・キーワードに応じた専用ツール推奨が固定的
- パターン4: 「実務対応フロー」領域(法令でなく通達 / マニュアル)のカバー不足
  - 該当困りごと: C2(夜勤暴力対応)、C3(針刺しフロー)、C5(感染就業制限)、D10(労基署対応) (4件)
  - 根本原因: コンテンツ不足。安衛法本則は強いが、現場の「次に何をするか」のフロー化情報が薄い
- パターン5: 「年次計画」と「月次・週次運用」のギャップ
  - 該当困りごと: A8(元請月次提出)、B2(衛生委員会月次報告書) (2件)
  - 根本原因: 機能不在 + 用途ミスマッチ。plan-generator は年次計画書を 1 回作って印刷するモデルで、月次の繰返し作業を支えない
