# 10 既存機能との統合体験（軸10）

## 現状
- 化学物質RA → 事故DB: P1-3で実装済（物質詳細に「この物質関連の過去事故」accident-cross-section）。逆方向（化学物質RAの作業→事故）はchemical work-hints。
- 打合せ書 → 事故: 一部連携（lib/safety-diary に monthly-summary / schema で事故参照の痕跡）。
- KY → 事故: **直接動線が弱い**（KY作業内容→該当業種/作業の過去事故サジェストは未実装）。
- サイネージ → 事故: 朝礼での過去事故啓発表示は未実装（signage-news-accidents データはあるが朝礼サイネージ未連携）。
- チャットボット → 事故: 参照あり。

## あるべき統合
- I-1 **KY連携**: KY作業内容（業種・作業）→ 類似の過去事故をサジェスト（accident-cross-search/related流用＋AI軸7）。KY用紙のCTAから /accidents へ。chemical の work-hints CTA パターン流用。
- I-2 打合せ書連携: 議論する作業→関連事故事例を自動提案。
- I-3 サイネージ連携: 朝礼サイネージに「本日の業種で起きやすい事故」1件表示（signage-news-accidents活用）。
- I-4 横断: 化学物質RA(P1-3)と同じ「関連事故」UIを各機能に展開。

## 優先度
- **P1: I-1 KY→事故サジェスト動線**（chemical の work-hints CTAパターン流用、低リスク・高寄与、社長「一体運用」に直結）。
- P2: I-3 サイネージ朝礼の事故啓発、I-2 打合せ書連携。

## 結論
事故DB単体は成熟。伸びしろは**現場フロー（KY/サイネージ）への事故事例の差し込み**。化学物質RAで確立した連携パターンを横展開するのが最短。
