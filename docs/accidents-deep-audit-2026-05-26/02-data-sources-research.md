## 「直近の事故」を合法・無料で取り込む現実解 トップ3
1. **月次速報Excel（あんぜんサイト直リンク・R8.4以降の分析用Excel）** — 毎月HTTP GETで定期取得、死亡/死傷区分。最も新しい公式データ。
2. **e-Stat API**（無料登録＋アプリID、商用可、出典明示義務） — 業種別・事故型別の多観点分析に最適。
3. 年次確定プレス＋蓄積系DB（過去分析の土台）。

## 重要な制約（実装判断）
- **e-Stat API はアプリケーションID（APIキー）が必須**。本プロジェクトは「環境変数の新規追加禁止」。→ e-Stat API自動統合は**env追加のオーナー判断が前提**（現状は実装不可）。
- 月次速報Excelの直リンク取得はキー不要だが、Excelパーサ＋定期実行（cron/ビルド）の構築が必要（中規模）。スクレイピングでなく公開ファイルの定期DLは規約上許容範囲だが出典明示必須。
- 中災防/建災防は商用利用可否未確認＝**取り込まずリンク誘導に留める**（安全）。
- すべて出典明示＋「編集・加工した旨」の併記が必須（政府標準利用規約）。

## 既存5,026件の鮮度
- 統計フルコーパスは2021-2025のmock生成（Excel差し替え前提）。**実データの月次更新運用は未構築**。「直近の事故」を名乗るには月次速報の取込パイプラインが必要。

## 推奨（軸6と連動）
- P1: 月次速報・e-Statへの**明示リンク＋出典バッジ**（合法・即可）。
- P2: 月次速報Excelの定期取込パイプライン（中規模・env不要）。
- 要オーナー判断: e-Stat API統合（APIキー=新規env追加の可否）。
