# 13 改修案 優先順位付け（P0/P1/P2/P3）

各案: 対象/問題/改修/流用パターン/工数/寄与。流用記号: A=ローカルファースト+同期, B=storage, C=AI(Gemini)+免責, D=純粋関数+テスト, E=deep-linkクエリ, F=A4印刷, G=多言語局所辞書, H=規制/関連タグ, I=デバイス実測。工数 S=半日 M=1日 L=2日+。

## P0（最優先・高寄与・創作不要・ブロッカーなし）

### P0-1 AI事故注意喚起
- 対象: /accidents（新規セクション）。問題: AI解釈が無く「専門家レベルの即答」が未達（軸7/12）。
- 改修: 業種・作業内容入力 → 既存事故DBから類似ケース抽出（accident-cross-search/related流用）→ Geminiで危険ポイント・再発防止策を要約（参考明記）。新規 /api/accidents/analyze。
- 流用: C/D / 工数 M / 寄与: 最強競合(あんぜんサイト)のAI欠如を突く差別化。**最重要**。

## P1（高優先）

### P1-1 KY→事故サジェスト統合動線
- 対象: ky-paper-view。問題: KYから事故DBへの動線が弱い（軸10）。
- 改修: KY作業内容→該当業種/作業の過去事故・AI注意喚起へ誘導（chemical work-hints CTAパターン流用、/accidents?industry= 等）。
- 流用: E/C / 工数 M / 寄与: 社長「一体運用」直結。

### P1-2 直近事故 公式リンク＋鮮度表示
- 対象: /accidents。改修: 月次速報・年次確定・死亡災害DBへ出典バッジ付きリンク＋「最新公表時期」表示（軸6 D-1）。合法・即可。
- 流用: H / 工数 S。

### P1-3 多軸AND絞り込み＋関連事故サジェスト強化
- 対象: /accidents 検索、詳細ページ。改修: 業種＋事故型＋被害程度＋年代のAND絞り込み、詳細に「類似事故」（accident-related流用）。
- 流用: D / 工数 M。

### P1-4 「今月の労災トレンド」AI要約
- 対象: /accidents。改修: 集約データをGeminiで平易要約（参考明記）。※直近取込前は既存集約で。
- 流用: C / 工数 M。

## P2（中期）
- P2-1 月次速報Excel定期取込パイプライン（env不要・公開ファイルDL→パース→集約更新、出典明記）。
- P2-2 サイネージ朝礼に「本日の業種で起きやすい事故」表示（signage-news-accidents活用）。
- P2-3 多言語固定ラベル6言語（chemical-ra-labelsパターン）。
- P2-4 保存検索・履歴（localStorage）。
- P2-5 打合せ書→関連事故自動提案。

## P3（要オーナー判断・長期）
- P3-1 e-Stat API統合（**APIキー=新規env追加の可否はオーナー判断**）。多観点年次データの機械取得。
- P3-2 時間帯/曜日/経験年数の観点追加（公式データ取込が前提、創作禁止）。
- P3-3 自社事例の追加・蓄積（認証/個人情報の慎重判断）。

## Phase B 推奨着手順
P0-1 AI注意喚起 → P1-2 直近リンク → P1-1 KY動線 →（時間許せば）P1-3 / P1-4。
e-Stat API（P3-1）はenv追加可否のオーナー判断が前提。月次速報取込（P2-1）は中規模で次セッション可。
