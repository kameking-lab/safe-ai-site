# 00 事故DB 徹底レビュー エグゼクティブサマリー（社長向け3分）

調査日 2026-05-26 / main HEAD 29ed250a / 本番 https://www.anzen-ai-portal.jp/accidents

## 一言結論
事故DBは「多観点の統計ダッシュボード」が既に充実（業種別・月次・年次・年齢別・事故型別・業種比較・予防チェックリスト・印刷・3デバイス対応）。最大の伸びしろは **AIによる解釈（専門家レベルの即答）** と **直近事故の鮮度** と **現場フロー（KY等）への差し込み**。社長要求「みんなが専門家レベルで気軽に分析」はAI注意喚起で最短到達できる。

## 最重要発見トップ10
1. **AI分析が無い**: 分析パネルは統計のみ。最強競合「職場のあんぜんサイト」もAIなし。**無料・登録不要のAI注意喚起**は明確な差別化（軸7）。←最重要P0。
2. **統計ダッシュボードは成熟**: /accidents-reports に業種別/月次/比較/予防チェックリスト/印刷。多観点（業種/事故型/年齢/月/年）の集約データ完備（軸1/4）。
3. **直近性が弱い**: 統計コーパス（5,026件）は2021-2025のmock（Excel差し替え前提）。月次速報/e-Stat未取込。社長要求「直近の事故」は未達（軸6）。
4. **直近を合法・無料で取り込む道はある**: 月次速報Excel直リンク（毎月最新・キー不要）、e-Stat API（無料・商用可だが**APIキー＝新規env追加が必要**）。まずは公式リンク＋鮮度表示で即対応可。
5. **競合での立ち位置は強い**: あんぜんサイトはDL型・PC寄り・AI無・鮮度ラグ。当方は「スマホ完結×AI×直近×現場ツール一体×無料登録不要」で差別化（軸3）。
6. **現場フロー差し込みが弱い**: KY→事故サジェスト動線が無い（化学物質RAはP1-3済）。chemical の work-hints CTAパターンを横展開すべき（軸10）。
7. 検索は mhlw-search タブ＋業種フィルタ。多軸AND絞り込み・関連事故サジェストに伸びしろ（軸5）。
8. 多言語なし（軸8）。サイネージ/化学物質RAの局所辞書パターンで6言語化可能。
9. 3デバイス横スクロールなし・グラフ収まり良好（軸9）。
10. 個人特定情報は非表示・公開範囲。公式データは政府標準利用規約2.0で商用可だが出典明示必須。中災防/建災防は商用可否未確認＝リンク誘導に留める（軸11）。

## 数値サマリー（推定）
- 現状完成度: 約78/100（統計分析は強いがAI/直近/連携が弱い）
- 5,026件の鮮度: 2021-2025のmock（実データ月次更新は未構築）
- データ源カバレッジ: 公式リンクは可、自動取込は0（月次速報取込・e-Stat未実装）
- 「専門家レベルで気軽に」基準達成度: 約70/100
- 競合ポジション: あんぜんサイトに「AI・スマホ・直近・現場一体・無料登録不要」で優位、「網羅性・全数死亡DB」で劣位

## これだけは絶対やる（Phase B推奨）
1. **P0-1 AI事故注意喚起**（業種・作業→類似事故＋危険ポイント＋対策、Gemini）← 専門家レベルの即答を無料で
2. P1-2 直近事故の公式リンク＋鮮度表示（合法・即可）
3. P1-1 KY→事故サジェスト動線（chemical work-hintsパターン流用）

## Phase B 着手判断
P0-1/P1-2/P1-1 から着手（創作不要・低リスク・高寄与）。月次速報取込（P2-1）は中規模で次セッション可。e-Stat API（P3-1）はAPIキー＝新規env追加の可否がオーナー判断。
