調査方法: ソースコード全件精読（読み取り専用）。本番URLへの実機アクセス・ブラウザ実行は本調査環境では不可のため、画面挙動はコードからの再構成。

## 0. 結論サマリ（先に読む）

- KY機能は「**1枚作る**」ことに関しては競合に引けを取らない作り込みがある（業種プリセット11種、実例DB150件、音声入力、手書き署名、リスクアセスメント、朝礼サイネージ、日誌連携、オフライン対応）。
- 一方で「**毎日続ける**」「**元請に出す**」の2点で、紙のKY用紙にすら負けている欠落がある（後述）。
- さらに、KY入力画面が**3つに分裂**しており（`/ky`・ホーム内・`/pdf`）、項目がバラバラ。これが混乱と「足りない項目」の温床になっている。

## 2. `/ky`（メイン画面）でいま「できること」全件

### 入力モード（3種）
- **シンプル（推奨・初期値）**: 基本情報＋現地KY（リスク3件）＋参加者署名 のみ表示。
- **詳細（4ラウンド法）**: 全6セクション＋リスクの可能性×重大性スコア・再評価まで表示。
- **カスタム**: 6セクションを個別ON/OFF。

### セクション（最大6）
1. **① 基本情報**: 年・月・日、天候、備考、（詳細/カスタムのみ）協力会社1〜3次の会社名・安全衛生責任者。
2. **② 作業内容**（詳細/カスタムのみ・4行）: 作業箇所、作業内容、使用機械、火気/高所マーク、使用保護具、安全衛生指示事項、責任者、元請サイン。
3. **③ 現地KY（リスクアセスメント）**: 危険（どんな危険が潜んでいるか）、可能性1-3・重大性1-3チップ、評価値=可能性×重大性（色分け）、低減対策、再評価、リスク行の追加可。初期5行（自由記述＋①〜④）。
4. **④ 参加者署名**（初期6名）: 氏名、必要資格No.、体調（作業前/退場時）、**手書き署名**。参加者の追加・削除可。
5. **⑤ 墜落防止点検**（任意）: 足場・吊足場・構台の良/否/済、是正場所・内容。
6. **⑥ 終了確認**（任意）: 休憩時間、安全チョッキ、退出時間（大型/中型/普通車）、終了報告。

### 入力支援
- **初回ウィザード**: 業種→工種を2タップで選ぶと、作業内容＋想定リスク3件を自動入力。
- **業種プリセット**: 建設・製造・医療福祉・介護施設・訪問介護・運輸・林業・食品・小売・脚立はしご・清掃の**11種**。各3〜4リスク。
- **過去事例から提案**: 業種/作業種別で実例DB（150件）を絞り込み「この事例を反映」。利用履歴で再提案。
- **「AI危険下書き」「AI対策」ボタン**（詳細/カスタムのみ）: `/api/ky-assist`を呼ぶが、中身は**8種の危険文・7種の対策文をテンプレ選択する擬似AI**（実モデル未接続。免責文のみGemini由来）。
- **MHLW類似事例**: 作業内容のキーワードからプリセットを推定し参考表示。
- **音声入力**: 各テキスト欄の🎙ボタン（Web Speech API。Chrome/Edge/新しめのSafariで動作、非対応ブラウザはメッセージ表示）。
- **外部からの自動流し込み**: 事故事例（`?fromAccident`）、業種ページ（`?industry&topic`）、AIリスク予測（`?import=risk-prediction`）、日誌（`?fromDiary`）、脚立/プリセット（`?preset`/`?template`）。

### 保存・出力・連携
- **自動保存**: 入力1秒後に `localStorage["ky-record"]` へ自動保存（朝礼サイネージはこれを読む）。
- **手動「保存」**: `localStorage["safe-ai:ky-instruction-record:v1"]` に保存し、一覧（最大30件）へ要約を追加。
- **保存済みKY記録一覧**: 日付・天候・会社名・作業内容を表示、削除可。
- **「この内容で日誌を書く →」**: 手動保存済みのKYから安全衛生日誌へワンクリック転記。
- **「PDF出力 / 印刷」**: ブラウザの `window.print()` を呼ぶ（専用PDF生成ではない）。
- **「朝礼サイネージ表示 →」**: `/ky/morning` を開く。
- **「昨日コピー」**: `/ky?fromYesterday=1`。ただし**直近のKYではなく直近の「日誌」の作業内容のみ**を流用（後述の重大な落とし穴）。

## 3. `/ky/morning`（朝礼サイネージ）でできること

- 端末内 `localStorage["ky-record"]`（=`/ky`の自動保存）を読み、大画面用に表示。
- 表示: 日付・天候、本日の主な作業、**本日のリスクTop3**（危険＋対策）、唱和カウントダウン（10秒、ゼロで「ご安全に！」とビープ音）。
- 「KY編集に戻る」リンク。データ無い時は案内表示。

## 4. データの保存場所（重要・分かりにくい）

| キー | 誰が書く | 誰が読む |
|------|----------|----------|
| `ky-record` | `/ky`の**自動保存**（1秒ごと） | 朝礼サイネージ `/ky/morning` |
| `safe-ai:ky-instruction-record:v1` | `/ky`の**手動「保存」時のみ** | 「この内容で日誌を書く」/ ホーム |
| `safe-ai:ky-record-list:v1` | 手動「保存」時に要約追記（最大30件） | 保存済みKY一覧 |
| `ky-signatures` | 署名確定時 | `/ky`再表示時 |
| `ky-suggestion-history-v1` | 事例反映時（最大20件） | 事例の再提案 |

- すべて**端末内（localStorage）のみ**。クラウド保管・端末間同期・アカウント紐付けは**なし**。
- 朝礼サイネージと日誌連携で**読むキーが違う**（自動保存 vs 手動保存）点が後述の不具合の原因。

## 6. 「現状できないこと」総覧（詳細は各ドキュメントへ）

- 現場名・工事名の入力欄が**ない**（`/ky`）→ 02・08
- 職長名（リーダー）欄が**ない** → 02
- チーム行動目標・重点実施項目・指差呼称・ワンポイントの専用欄が**ない**（4Rの締めが自由記述に埋没） → 02・07
- 保存したKYを**開き直して再編集・複製できない**（一覧は閲覧と削除のみ） → 07
- 「昨日コピー」が危険・対策・参加者を引き継がない（作業内容だけ・しかも日誌から） → 07
- 元請（元方）確認の専用欄・承認フローが**ない** → 08
- PDFが「画面そのまま印刷」で、紙のKY用紙の見た目にならない／詳細モードで可能性・重大性が印刷に出ない → 08
- 複数日・複数現場の一括出力／ZIP提出が**ない** → 08
- 写真添付が**ない**（競合は標準装備が多い） → 03
- クラウド共有・端末間同期が**ない**（朝礼サイネージは作成端末と別端末では表示不可） → 05・07
