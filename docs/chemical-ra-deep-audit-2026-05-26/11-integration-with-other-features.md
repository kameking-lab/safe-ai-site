# 11 既存機能との統合体験（軸11）

## 現状（最大のギャップ）
- KY（/ky/paper）→ 化学物質RA/DB への**直接動線なし**。
- 打合せ書（/safety-diary）: presets に予想災害「化学物質ばく露」の文字列はあるが、化学物質DB/RAへの deep-link なし。
- チャットボット: chemical-ra/chemical-database への参照あり（部分連携）。
- 事故DB・サイネージ → 化学物質: 連携確認できず。

→ KY/打合せ書/サイネージは相互に育ってきたが、**化学物質RAだけが島になっている**。社長の中核コンセプト「現場のめんどくさいを解決」「一体運用」に対し、化学物質が分断。

## あるべき統合体験
- I-1 KY連携: KYの作業内容に化学物質（塗装/溶接/洗浄/接着等）が含まれる → 「この作業で使う物質の規制・ばく露注意」を該当物質一覧で自動表示し、KYのリスク欄に取り込み。
- I-2 打合せ書連携: 作業に該当物質 → ばく露注意事項・必要防護具を打合せ書に自動追加（マスター選択・AI提案の範囲で「現場に何かさせない」を遵守）。
- I-3 サイネージ連携: 物質取扱現場の朝礼サイネージに「本日の取扱物質と注意」を表示（既存サイネージのモード追加）。
- I-4 チャットボット連携: 物質名問い合わせ → /chemical-database/[cas] へ誘導（既存参照の強化）。
- I-5 事故DB連携: 物質詳細に「同物質の過去事故事例」を表示（5,026事故DBと突合）。

## 流用パターン
- KYのcloud/storage-adapter、deep-linkクエリ（/chemical-ra?cas=... は既存）、AI提案（Gemini）。

## 優先度
- **P0/P1 最優先: I-1/I-2（KY・打合せ書→化学物質の動線）**。社長コンセプトの一体運用に直結し、化学物質RAの利用を現場フローに乗せる。低〜中リスク（既存deep-link `?cas=` を活用）。
- P1: I-5 事故事例連携（既存事故DB活用）。
- P2: I-3 サイネージ連携、I-4 チャットボット強化。

## 結論
機能単体の完成度より、**「化学物質RAを現場フロー（KY/打合せ書）に接続する」ことが、日本一簡単で便利への最短距離**。ここが本監査の最重要発見の一つ。
