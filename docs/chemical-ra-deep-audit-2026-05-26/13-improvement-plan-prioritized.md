# 13 改修案 優先順位付け（P0/P1/P2/P3）

各案: 対象/問題/改修/流用パターン/工数/寄与。工数 S=半日 M=1日 L=2日+。
流用パターン記号: A=ローカルファースト+背景同期, B=storageアダプタ, C=AI提案(Gemini)+免責, D=純粋関数+テスト, E=deep-linkクエリ, F=A4印刷(@media print), G=多言語(language-context), H=規制タグ(regulation-tag-labels), I=デバイス実測(Playwright)。

## P0（最優先・効果大・低リスク・創作不要）

### P0-1 KY/打合せ書 → 化学物質RA/DB 統合動線
- 対象: ky-paper-view / safety-diary。問題: 化学物質RAが現場フローから孤立（軸11）。
- 改修: KY/打合せ書の作業内容に化学物質キーワード（塗装/溶接/洗浄/接着/有機溶剤等）→「該当物質の規制・ばく露注意を見る」リンク（既存 `/chemical-ra?cas=` / `/chemical-database` へ）。マスター選択・AI提案の範囲、現場に強制しない。
- 流用: E/C / 工数 M / 寄与: 社長コンセプト一体運用に直結。**最重要**。

### P0-2 物質詳細「全法律ワンストップ」サマリー強化
- 対象: /chemical-database/[cas]。問題: 横断表示はあるが、未カバー法律（消防/大気/水質/土壌）の扱いが無く「全法律見える」感が不完全。
- 改修: 冒頭に「この物質の規制サマリー」を集約（カバー法律はタグ、未カバー/物性判定型は「該当する可能性—要確認」と判定条件を明示）。創作せず「未確認」を明記。
- 流用: H / 工数 M / 寄与: 核心要求の完成度UP。

## P1（高優先）

### P1-1 RA結果のA4印刷「実施レポート」
- 対象: /chemical-ra。改修: RA結果をA4印刷/PDF（記録保存・監督署提示）。流用 F / 工数 M。

### P1-2 RA結果のクラウド保管
- 対象: chemical_ra_records 新規（Supabase）。改修: KYのstorage-adapterパターンで保存・履歴・再編集。**新規テーブルはfix.sqlと同様 service_role GRANT を忘れない**。流用 A/B/D / 工数 L。

### P1-3 特殊健診の対象自動判定
- 対象: 物質詳細/RA。改修: 特別則タグ（特化則/有機則/鉛/石綿等）→ 必要健診項目・頻度を提示。流用 H / 工数 S〜M。

### P1-4 防護具選定の高度化（化学防護手袋素材）
- 対象: chemical-equipment-mapping。改修: 物質＋接触形態→手袋素材の耐透過性。ケミサポ対抗。流用 C / 工数 M。

### P1-5 多言語（固定ラベル）
- 対象: 化学物質画面。改修: language-context 辞書に化学物質固定ラベルを6言語追加。流用 G / 工数 M。

### P1-6 事故DB連携
- 対象: 物質詳細。改修: 同物質の過去事故事例（5,026事故DB）を表示。流用 E / 工数 M。

### P1-7 土壌汚染対策法 26物質タグ追加
- 対象: regulation-tag-labels + データ。改修: 環境省公式の特定有害物質26を**正確に取得**してCASタグ追加（創作禁止）。流用 H / 工数 M。
