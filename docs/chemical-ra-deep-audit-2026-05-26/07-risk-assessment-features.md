# 07 リスクアセスメント機能（軸7）

## 現状
- `/chemical-ra`（chemical-ra-panel 915行）: 物質名/CAS/別名・取扱量・換気状況等 → ばく露リスク区分（I〜IV）を簡易評価。CREATE-SIMPLE準拠を標榜（安衛法57条の3・2024自律的管理）。
- `app/api/chemical-ra/route.ts`（464行）: Gemini による対策提案・ばく露区分支援。
- 防護具選定（chemical-equipment-mapping）と連携。

## 改善余地
- R-1 **クラウド保管なし**: RA結果が chemical_ra_records 等のSupabaseテーブルに保存されない（KY/打合せ書と非対称）。再編集・別端末・履歴・監督署提示に不利。→ Phase B でテーブル新規＋storage-adapterパターン流用が有力。
- R-2 **行政提出帳票**: A4印刷/PDFの「実施レポート」出力（ケミサポが持つ強み）。記録保存義務に直結。
- R-3 定量根拠の明示: CREATE-SIMPLE準拠を謳うなら、ばく露推定の根拠（取扱量・換気・GHS区分→区分）の透明性を高める。ACGIH/産衛学会数値は法的根拠/著作権で除去済の方針堅持、濃度基準値（告示）は利用可。
- R-4 混合物RA: 複数成分の合成リスク（product-search連携）。
- R-5 AI措置提案の出典/免責: 既存の免責表示を踏襲。

## 流用パターン（KY/打合せ書で確立）
- ローカルファースト＋背景同期（storage-adapter）、純粋関数＋テスト、AI提案の免責、A4印刷（@media print）。

## 優先度
- P1: A4印刷の実施レポート（R-2、低リスク・高需要）、クラウド保管（R-1、KYパターン流用）。
- P2: 混合物RA（R-4）、定量根拠の透明化（R-3）。
