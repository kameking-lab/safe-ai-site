# 05 物質検索UX（軸5）

## 現状（良好）
- `/chemical-database` 検索は **物質名 / CAS番号 / 別名(synonym/aliases) / 用途キーワード** に対応（placeholder「例: ベンゼン / 71-43-2 / 金属脱脂」）。
- 実測: 「ベンゼン」入力 → 745ms で該当表示（サブ秒、クライアント側フィルタ）。
- `/chemical-ra/product-search`: 製品名から物質を探す導線あり。

## 改善余地
- S-1: 化学式入力（例: C6H6 → ベンゼン）の対応は未確認/未対応。分子式インデックス追加で対応可。
- S-2: サジェスト/インクリメンタル候補表示の強化（タイプミス・表記ゆれ吸収。例「トルエン/toluene/メチルベンゼン」）。
- S-3: 混合物検索（製品の成分単位での横断）。product-searchの拡張余地。
- S-4: 検索ゼロ件時のフォールバック（「NITE-CHRIPで探す」外部リンク、AIに物質名を尋ねる導線）。

## 結論
検索UXは中小現場には既に実用的（名前・CAS・別名・用途で引ける）。優先度は中。S-2サジェスト強化と S-4 ゼロ件フォールバックが体験を一段上げる（Phase B P1〜P2）。
