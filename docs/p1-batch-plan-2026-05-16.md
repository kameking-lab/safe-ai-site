## バッチ詳細

### Batch 1 — 表示誤認リスク修正 (推奨着手: 2026-05-17)

**含まれるfinding:** A-001, A-002  
**推定工数:** 9h  
**依存先:** なし  
**PR名案:** `fix(exam-quiz): rename "過去問" to "学習用問題" + add source disclaimer`

#### A-001: 「過去問クイズ」表示と実装の不一致

- 対象: `/exam-quiz`, `/quiz`
- 修正方針:
  1. `/exam-quiz` ページ見出し「過去問クイズ」→「学習用問題」
  2. 各カテゴリトップに「公式試験問題の逐語コピーではなく、出題範囲を踏まえた学習用問題」と明示
  3. 実際の公表過去問は厚労省・JISHA・試験運営機関の公式ページへの外部リンクに限定
- 影響範囲: `/exam-quiz` と同系統のクイズコンポーネント全体
- リスク低減: 景表法(優良誤認)リスクの即時解消

#### A-002: exam-quiz ページに出題ソース注記なし

- 対象: `/exam-quiz`
- 修正方針:
  - ページ上部に固定インフォバー追加:  
    「※当サイトの問題は学習用に作成したものであり、実際の試験で出題されたものではありません」
- 影響範囲: exam-quiz ページのみ
- A-001 と同 PR での実装を推奨

**マージ後の効果:**
- 「過去問」誤認リスクをゼロに近づける
- G-001(P0、景表法リスク)の観点も同時に解消

### Batch 2 — 未稼働ページ 非公開化一括 (推奨着手: 2026-05-18)

**含まれるfinding:** F-001, F-002, F-003, F-004  
**推定工数:** 7h  
**依存先:** なし  
**PR名案:** `fix(nav): remove/noindex 4 unfinished pages (lms/api-docs/handover/pricing)`

#### F-001: /lms — 未稼働のウェイトリストページ

- 修正方針: ナビメニューから除外 + `<meta name="robots" content="noindex">` または metadata.robots 設定
- ページ本体は「LMS機能は法人化後にβ提供予定」1行に縮小

#### F-002: /api-docs — 実APIなし

- 修正方針: ページ削除(または `/admin/api-docs` に移動)+ 301リダイレクト設定
- 代替: ナビから除外 + noindex + 「API提供は法人化後に予定」1行に縮小

#### F-003: /handover — 内部用ページの外部露出

- 修正方針: ページ削除 または `/admin/handover` に移動
- A-006(P2) とも重複。ナビから除外 + noindex で即時対応可能

#### F-004: /pricing — 準備中プランのみ

- 修正方針: ナビから除外 + noindex
- ページは「料金は法人化後に提示予定」1行に縮小

**マージ後の効果:**
- 「未完成な企業プロダクト感」を即時解消
- 第三者の信頼性スコア向上

### Batch 3 — 体裁整合・DPA整理 (推奨着手: 2026-05-19)

**含まれるfinding:** A-005, G-002  
**推定工数:** 9h  
**依存先:** Batch 2 完了推奨（非公開化済みページへの整合）  
**PR名案:** `fix(branding): clarify personal-project scope + noindex /dpa`

#### A-005: 「個人運営」表記と企業向け機能(LMS/DPA/API/pricing)の体裁不整合

- 修正方針:
  1. フッタの「個人運営の研究プロジェクト」表記に加え、法人化前サービスの範囲を明示

- 影響範囲: フッタ、`/about`、ナビゲーション全体

#### G-002: /dpa — 未整備テンプレートの外部公開

- 修正方針:
  - `/dpa` をナビから除外 + noindex
  - ページ本体は「DPA等の企業契約は法人化後に提供」1行に縮小
  - Batch 2 の /api-docs 対応と同様のパターン

**マージ後の効果:**
- 「個人研究プロジェクト」と「企業向けプロダクト」のブランド乖離を解消
- 法的責任が曖昧なページをすべて撤退

### Batch 4 — コンテンツ正確性・保険情報更新 (推奨着手: 2026-05-20)

**含まれるfinding:** B-001, G-003  
**推定工数:** 12h  
**依存先:** なし（独立。Batch 1/2 完了後推奨）  
**PR名案:** `fix(content): synthetic-accident badge + insurance roadmap update`

#### B-001: 2025年事故事例が実質創作 — バッジ不明確

- 対象: `/accidents-reports`, `/accidents`
- 修正方針:
  1. 代表パターン事例に専用バッジ「想定例(統計に基づく合成)」を新設
     - 色: amber/yellow 系(現行の "速報" バッジとは異なる色)
  2. 実報告事例(個票)と視覚的に明確に分離
  3. 確定値(R07個票)公開後に置換するロードマップを該当ページ上部に常時表示
- 影響範囲: `real-accident-cases-2025-preliminary.ts` を参照するコンポーネント全体

#### G-003: 賠償責任保険「未加入」の明記

- 対象: `/insurance`
- 修正方針:
  1. 「未加入」状態の透明性は維持しつつ、取得ロードマップを明示
     - 短期: 個人事業者向けPL保険検討
     - 中期: 法人化後にIT賠償責任保険取得
  2. `/insurance` ページの最終更新日フィールドを追加(定期的な状況更新を可視化)
- 影響範囲: `/insurance` ページのみ

**マージ後の効果:**
- コンテンツ誠実性(合成データの明示)の向上
- 法的信頼性(保険ロードマップ)の向上
