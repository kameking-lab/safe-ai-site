## 3. 指摘（条文ID・原文・plain・問題点）

### 3-A. 誤読リスク（実害級）

**A-1. 安衛則 第117条（gap-fill 由来）— ただし書（適用除外）の完全欠落**
- 実法文（e-Gov スナップショット `347M50002000032.json` で確認）: 「…覆いを設けなければならない。**ただし、直径が五十ミリメートル未満の研削といしについては、この限りでない。**」
- plain: 「事業者は、回転中の研削といしが労働者に危険を及ぼすおそれがあるときは、覆いを設けなければなりません。研削盤やグラインダーなど…基本義務で、覆いは破裂時の飛散を防ぐ目的で設けます。」
- 問題: 50mm 未満の適用除外が corpus（gap-fill 要旨）段階で落ちており、plain は**無条件義務**として提示。おまけに要旨の編集コメント（「基本義務で…目的で設けます」）を条文の言い換えとして表示。漢数字＋gap-fill の二重盲点でゲートは緑。

**A-2. 安衛則 第552条（架設通路）— 現行法の要求の欠落（コーパス陳腐化の透過）**
- 実法文: 第1項第4号は「手すり等（85cm以上）**＋中桟等（35cm以上50cm以下）**」の2点セット、さらに第2〜4項（臨時取り外し時の墜落制止用器具措置・立入禁止・原状復帰・労働者の使用義務）がある。
- plain: 「…墜落のおそれがある箇所には高さ八十五センチメートル以上の丈夫な手すり等を設けること…使用してはいけません。」（1文208字）
- 問題: **中桟等の要求と第2〜4項が丸ごと無い**（corpus が旧・簡略文のため）。ハッシュ照合は「コーパスと一致」しか見ず、e-Gov スナップショットが同リポジトリにあるのに突合されていない。

**A-3. 有機則 第2条（適用除外）— 許容消費量の計算式と気積150m³上限の黙殺**
- 原文: 「W＝（１／１５）×Ａ／W＝（２／５）×Ａ／W＝（３／２）×Ａ…Ａ 作業場の気積（…）。ただし、気積が**百五十立方メートル**を超える場合は、百五十立方メートルとする。」
- plain: 「…有機溶剤の種類(第1種・第2種・第3種で計算式が異なります)ごとの許容消費量…以下であれば対象外です。」omissions 宣言なし。
- 問題: 適用除外の判定に必要な**係数そのものと150m³キャップが消えている**のに省略宣言がない。全角スラッシュ分数・漢数字・「立方メートル」「グラム」が全てゲート語彙外のため不可視。

**A-4. 鉛則 第58条（呼吸用保護具等）— 漢数字参照の黙った欠落**
- 原文: 「鉛装置の内部における業務**（令別表第四第九号）**に労働者を従事させるときは…」
- plain: 「事業者は、鉛装置の内部における業務に労働者を従事させるときは…」（参照消滅・omissions なし）
- 問題: `REF_RE` が算用数字前提のため漢数字参照は抽出されず、ref-missing が発火しない実例。

**A-5. 安衛則 第34条（文書の交付）— 義務の希釈（〜するものです）**
- 原文: 「法第57条第2項の規定による文書は、…譲渡し、又は提供する際に**交付しなければならない**。」
- plain: 「…譲渡・提供する際に**交付するものです**。」
- 問題: 義務が説明文に弱まっている。主語「文書は」が SUBJECTS 16語に無いため、この条の義務はゲートの照合対象ゼロ。同型のリスクは主体語彙外の全条文（「〜装置は…でなければならない」等）に潜在。

**A-6. 酸欠則 第6条（見本法令）— 義務の本文放逐＋omissions迂回**
- 原文4項: 「事業者は、…請負人に対し、要求性能墜落制止用器具等を使用する必要がある旨を**周知させなければならない**。」
- plain: 「…請負人に作業の一部を任せる場合の周知義務や、器具の定義の参照（安衛則第130条の5第1項）は**原文をご覧ください**。」
- 問題: 義務を「原文をご覧ください」で本文から追い出し、omissions 宣言もしていない（「事業者」と義務マーカーが別の文にあるためゲートは緑）。README の「免責はUI側が付ける。plainText に書かない」にも違反。**見本自身がこれをやっているため部隊にも伝播**（有機則第30条の3、酸欠則第12条）。

**A-7. 有機則 第30条の3（健康診断結果報告）— ゲート合格のための参照残しが誤読を誘発**
- 原文の「第6条第1項」は**情報通信技術活用法**（電子申請の根拠）の条。
- plain: 「項目の詳細な内訳(第29条第2項第4号や別表の各項目、**第6条第1項の関連規定を含みます**)は原文をご覧ください。」
- 問題: 別法令の条番号を健診項目の括弧に押し込んだため、有機則第6条（第3種有機溶剤の設備）と誤読させる。ref-missing を黙らせるための機械的な参照残しの副作用。

**A-8. じん肺法施行規則（15条全部）＋ gap-fill 由来 plain 全般 — 「要旨の言い換え」を条文の言い換えとして提示**
- 例: じん肺則第4条 原文（コーパス）「じん肺健康診断の検査項目のうち胸部に関する臨床検査の方法（…）**を規定する**。」→ plain「…の方法で行います。」
- 例: じん肺法第23条（gap-fill）「…事業者は粉じん作業に従事させてはならない。…配置転換等の措置を講じる必要がある。」→ plain がそのまま断定（実第23条は「療養を要するものとする」であり、従事禁止・配置転換は編集による他条の要約）。
- 問題: 原文自体がメタ記述・編集要旨の条で「現場ことば版（わかりやすい言い換え）」の看板を出すと、**編集者の要約が条文として現場に流通する**。UI 上の区別なし。該当: じん肺則15条、安衛則11条（第96,117,131,164,332,333,352,574,588,194の22,151の21）、安衛法第15条の3、じん肺法第23条、クレーン則1条、特化則1条。

### 3-B. omissions 濫用・運用ばらつき

**B-1. 使用率の法令間格差が異常**: 安衛則11 / 見本3 / 有機則2 / 高圧則2 / 特化則1 / 鉛則1 — **残り13法令は0件**。部隊4（粉じん・じん肺則・電離・石綿）と部隊5は逐語コピー戦略のため省略自体が少なく0でも嘘ではないが、A-3/A-4 のように「ゲートが見えない省略」は宣言されないまま素通りしている。つまり omissions は「ゲートに叱られたものだけ書く」運用になっており、自主的な省略宣言としては機能していない。
- 参考: 高圧則第10条は逆に、本文で実質カバーしている内容（測定対象ガスの内訳）まで omissions 宣言しており、宣言の粒度が部隊間でバラバラ。

**B-2. 「原文をご覧ください」3件（酸欠則第6・12条、有機則第30条の3）**は omissions の代替として本文に書かれた省略。宣言欄を通らないため UI の「※この言い換えで省いている内容」にも載らない。

### 3-C. 文体不統一（法令間・部隊間）

**C-1. 見本と部隊のトーン乖離**: 見本（酸欠則）だけ「関係者以外を入れない掲示が現場の基本です」「隠さず・すぐ・監督署へ、が鉄則です」「使う前に、その日必ず点検するのが基本です」等の現場向け一言が付く。Sonnet 部隊19法令にはこの型がほぼゼロ。見本＝品質基準のはずが、文体仕様として伝播していない。

**C-2. 部隊4は逐語ます化**: 電離則・石綿則・粉じん則は原文の語順のまま語尾だけ変えた翻訳調で、「並びに」「及び」「当該」「〜に係る」「同症状」が残存（例: 石綿則第1条「…労働者の人数並びにばく露される期間及び程度を最小限度にするよう努めましょう」、電離則第56条は検査項目を列挙するだけの1文168字）。部隊3・5の「書き直し調」との落差が大きく、同一UIで品質が二層化している。

**C-3. 数字表記の混在**: コーパス表記をそのまま踏襲した結果、高圧則第14・18条「毎分〇・〇八メガパスカル」・第38条「六月以内ごとに一回」、安衛則第552条「三十度」「八十五センチメートル」vs 鉛則第53条「6月…1回」等、同じ概念が法令によって漢数字/算用数字で揺れる。「現場ことば」なら 0.08MPa・85cm が自然。

**C-4. 括弧の全角/半角混在**: 部隊3ファイル（有機・鉛・四アルキル・特化）は半角 `()`、他部隊は全角 `（）`。UI 上で並ぶと目につく。

### 3-D. 現場語として不自然

**D-1. 超長文の一文列挙（ラチェットで恒久免除）**: 施行令第6条（1文778字・全体824字）・第20条（393字）、クレーン則第33条（223字。三つの号を読点だけで接続:「…表示すること、…行わせないこと**という措置を**講じなければなりません」）、同第29・35条、特化則第7・28・38条の3、安衛則第552・563・36条、ボイラー則第62条など、括弧除き200字超の文が15条超。すべて 2026-07-11 生成で長さ上限の適用外。
**D-2. 機械的転写の珍妙表現**: 事務所則第1条 plain「労働者を常時就業させる室**（室）**の気積を…」（「以下「室」という」の無意味な残骸）。
**D-3. 罰則条が参照の羅列**: 安衛法第119・120条はほぼ条番号リスト。参照保存要件と平易化が正面衝突しており、現場ことば版としての価値がない（何が「捕まる違反」かの要約がない）。

## 4. ゲート強化タスク案（機械化できる穴）

優先度順。いずれも `web/src/lib/plain/fidelity.ts` と周辺の変更で実装可能。

1. **[G1] 漢数字の正規化を数値・参照抽出の前段に導入**（A-1〜A-4の根本）
   - 既存の `normalizeKanjiNumbers`（`web/src/lib/article-number-normalize.ts`、fulltext-gates で実証済み）を `extractNumericFacts` / `extractRefsAndBlank` の前処理に接続。「〇・〇八」「百五十」「五十ミリメートル」「別表第四第九号」を可視化。
   - 検証: 鉛則第58条で ref-missing、有機則第2条で number-missing が新規に赤くなることをテストで確認。

2. **[G2] 単位語彙の拡充**: `UNIT_CANON`/`UNIT_ALT` に ミリシーベルト/シーベルト、キロパスカル/メガパスカル/パスカル、ミリグラム/グラム、ミリメートル/キロメートル、立方メートル/平方メートル、ボルト/アンペア、**円/万円**（罰金額）を追加。全角スラッシュ分数 `（１／１５）` 用に `/(\d+)／(\d+)/`（正規化後）を FRACTION 系に追加。等価キー（0.05mg と 五十マイクログラム等）は canonical 側で吸収。

3. **[G3] 長さ上限ラチェットの解消**: 現状 512条中383条が恒久免除。(a) 07-11 在庫の超過分（括弧除き1文120字超は50条規模）を機械抽出して BACKLOG-plain-stale 相当の「分割キュー」ファイルに自動出力する常設テスト（fail ではなく警告→期限後 fail）、(b) `LENGTH_CAP_SINCE` を撤廃して全件強制に移行する期日を決める。

4. **[G4] 「読者放り出し」の禁止 lint**: `/原文をご覧ください|原文を参照|詳しくは原文|e-Gov(を|で)/` が plainText にあれば style 違反。省略は omissions のみ許可（README 規約のゲート化）。既知3件が赤くなる。

5. **[G5] ただし書・適用除外の保存チェック**: 原文（漢数字正規化後）に「ただし」「この限りでない」「適用しない」「〜を除く」があるとき、plain＋omissions 側に例外マーカー（ただし/例外/除きます/除く/対象外/不要/かまいません/限ります 等）を最低1つ要求。粗いが「例外の黙殺」（A-1型）を構造的に止める。

6. **[G6] 義務チェックの2段強化**（A-5/A-6型）
   - (a) 主体語彙外の義務: 原文の義務マーカー総数（文単位）と、plain の義務マーカー総数＋omissions 宣言数の下限比較（`plain義務数 + 宣言数 >= 原文義務数` の粗い保存則）。「〜するものです」希釈が number として現れる。
   - (b) (主体,種別) の dedup をやめ、項・号単位の出現数で照合（同一主体の複数義務の片落ち検出）。
   - (c) `omissionsCover` が主体語1語で当該主体の全義務を免除している現状を、「主体×種別」単位の宣言要求に狭める。

7. **[G7] コーパス↔e-Gov 全文の突合ゲート拡大**(A-2型の検出。plain ゲートというよりコーパス層だが、plain の信頼はここに律速される)
   - `laws-fulltext/347M50002000032.json` は既にあるので、安衛則については「コーパス収載条 vs fulltext の数値トークン差分・ただし書有無差分・項数差分」を警告する常設テストを追加（第552条・第117条系の陳腐化・欠落が即座に見える）。他法令のスナップショット取得は data 班タスクとして起票。

8. **[G8] 要旨コーパス由来の plain を機械識別**(A-8)
   - `corpusGapFillArticles` のキー集合と plain キーの突合テストを追加し、該当 plain に (a) 生成禁止 or (b) `PlainArticle` に `sourceKind: "digest"` を義務付けて UI 文言を変える（「概要の言い換え」等）。
   - 原文が `/を規定する。?|旨を規定/` を含む条（じん肺則型メタ記述）からの plain 生成を CI で検出・警告。

9. **[G9] 表記統一 lint**: plainText 内の「漢数字＋単位」（〇一二三…十百千＋メートル/度/月/回など）を検出して算用数字を要求（固有名詞・条番号引用は除外）。括弧は全角に統一（または半角に統一。どちらかに寄せて regex で強制）。

10. **[G10] 参照チェックの対象拡大**: `REF_RE` に「様式第N号」「令別表第N」漢数字型（G1で吸収）を追加。相対参照の解決（「前条」→実条番号）を plain 側で書いた場合の照合（現状「前条（第519条）」のような良い書き方が報われない）。

11. **[G11] 罰則チェックの強化**: 「罰」1字ではなく、罰則種別語（拘禁刑/懲役/罰金/科料/過料）のいずれかと、G2 導入後の金額・期間数値の保存を要求。

## 6. 総括

ゲートは「算用数字＋既知単位＋16主体語彙＋算用参照」の範囲では良くできているが、**(1) 照合先コーパス自体が要約層である**こと、**(2) 漢数字・単位語彙外・主体語彙外が全て盲点**であること、**(3) 長さ上限が75%の在庫に効いていない**ことの3点で、「fidelity 全緑」の申告は実際の法令忠実度をかなり過大表示している。実害級の欠落（研削といし50mm・架設通路中桟・有機則150m³）はいずれも盲点の交点で起きていた。
