# 09. 新たに発覚した問題（軸6・Phase 1–7 実装後）

Phase 4–7 の実装・一本化で **新たに生じた／顕在化した** 問題。コード静的確認ベース。重要度は 10 で優先順位化。

## NEW-1【P0級・後退】過去KYの一覧・再編集UIが消失
- **事象**: Phase 7 で旧 `/ky`（`ky-page-content.tsx`）と `home variant="ky"`（`KyRecordList`）を削除した結果、**保存済みKYを一覧する／開いて再編集する画面がどこにも無くなった**。
- **データは在る**: クラウド `ky_records` に履歴、`GET /api/ky/records` は最新＋一覧（最大30）を返す。localStorage 一覧キー（`safe-ai:ky-record-list:v1`）も残る。**だが表示するUIが無い**。
- **影響**: 前回 P0-3／P1-7（保存KYを開く・検索）が"UIごと"未達に逆戻り。PC事務利用・過去票再利用・現場別管理が不能。紙のバインダーに負ける。
- **根本**: 一本化を `/ky/paper`（単票ビュー）に寄せた際、一覧系を引き継がなかった。

## NEW-2【P1・後退】`/ky` のクエリ起点フローが無効化
- 旧 `/ky` は `?preset=ladder` / `?import=risk-prediction` / `?fromDiary=...` / `?fromYesterday=1` / `?q=...` を解釈して**他機能からKYへ取り込む導線**を持っていた（risk-prediction-panel、safety-diary、ladder-stats-card、chatbot 等から）。
- 現在これらは `/ky/paper` へ 308 転送されるが、**`/ky/paper` はクエリを解釈しない**＝取り込みが効かない（リンク自体は生きているので「飛ぶが何も起きない」）。
- **影響**: 事故DB/リスク予測/日誌/チャットボット→KY の **クロスツール連携が静かに切れている**。前回「統合資産の連鎖」を強みと書いた部分が一部断線。

## NEW-3【軽微・事実】リダイレクトは 301 でなく 308
- `permanentRedirect()` の実HTTPは **308**（メソッド保持の恒久リダイレクト）。社長メモの「301」と異なる。SEO上は 301/308 とも恒久で実害は小さいが、**「301」を厳密に要する場合は別途設定が必要**。

## NEW-4【P1・データ整合の体感】クラウド同期は"手動保存時のみ・ローカル優先"
- 設計上：**自動保存（`ky-record`）はローカルのみ**でクラウドへ送らない。**手動「保存」時にクラウド push**。取得は**ローカルが空のときだけ**クラウド pull。
- **影響**: 端末Aで保存→クラウド反映後、端末Bに古いローカルが残っていると **Bは自動で最新に置き換わらない**（ローカル優先）。また「書きかけドラフト」は端末間で共有されない（手動保存しないと乗らない）。
- これは「オフライン耐性優先」の意図的設計だが、**ユーザーは"クラウドだから常に最新同期"と誤解しやすい**。挙動の明示 or 「クラウドから最新を取得」ボタンが要る。

## NEW-5【軽微】`/pdf` 統合で「朝礼要点PDF」が廃止
- 旧 `/pdf` は KY用紙PDFに加え「朝礼要点（morning-briefing）」PDFプレビューを持っていた。`/ky/paper` へ転送統合で**朝礼要点PDFの出口が消えた**。要否は社長判断（00・残課題）。

## NEW-6【軽微・コスト】Gemini 予算上限が IP レート制限のみ
- `api/ky/suggest` は IP 10回/分＋circuit breaker。**全体の月次予算上限（コスト天井）は未実装**。env設定後に想定外利用が増えるとコスト増の可能性。最低限の上限/監視を推奨。

## NEW-7【軽微・セキュリティ】6桁コードは低エントロピー・認証なし
- 100万通り・24h有効・認証なし。コードを知る/総当たりする第三者が当該KYを閲覧可能（書込は不可）。KYは低機微だが、**現場名・人員が見える**点は留意。レート制限/試行制限や有効期限短縮で緩和可。

## NEW-8【監視】SEO：インデックス済み `/ky` の 308 化
- `/ky`（HowTo構造化データ・被リンク有り）が `/ky/paper` へ恒久転送。schema/canonicalは移設済だが、**検索評価の移行には時間**がかかる。サーチコンソールで `/ky/paper` のインデックス・カバレッジを監視推奨。

## NEW-9【確認】既存localStorageユーザーの移行
- `storage-migration.ts` が旧手動保存キー→`ky-record` を引き継ぐ（空のときだけ）。**ただし `safe-ai:ky-paper:v1`（簡易フォーム）等のレガシーは引き継ぎ対象外**。旧データ種別によっては拾われないものがある（影響は限定的だが要把握）。

## まとめ
- 実装品質（テスト/型/build）は高い一方、**一本化の副作用（NEW-1/NEW-2）が実害として大きい**。
- 残りは設計意図の明示（NEW-4）と運用上の小リスク（NEW-6/7/8）。
- いずれも **実装は可能**。優先順位は 10 を参照。
