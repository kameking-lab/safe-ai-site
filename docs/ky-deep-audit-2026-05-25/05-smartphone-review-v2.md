## コードから読める構造的所見（375px）
1. **用紙本体は固定幅 820px＋`scale(zoom)`**（`style={{ transform: scale, width: 820, maxWidth:"100%" }}`、`overflow-x-auto`）。375pxでは用紙が画面より広く、**初期表示で横スクロールまたは縮小（zoom 0.6〜）が要る**。「用紙を一望」したい朝礼用途では、初期 zoom が 1.0 固定な点が小画面で不利な可能性（★実機要確認）。
2. **ズーム操作**：60〜160%、±10%ボタン（`px-3 py-1.5`）。スマホの親指操作で押せるサイズだが、ピンチズーム併用可否は未検証。
3. **入力欄**：音声/キーボード兼用（`InputWithVoice`/`TextareaWithVoice`）。用紙内のセルは `text-xs`〜`text-sm` で小さめ。375pxでの可読・誤タップは要実機。
4. **参加者チェック**：作業員チップ（`px-3 py-1.5 rounded-full`）をタップ選択。手書き署名の回し不要＝スマホ最大の改善。タップ領域は最小44px推奨に対しやや小さめの可能性（要実機計測）。
5. **下部固定アクションバー**：保存/前回複製/別端末で共有/サイネージへ/印刷。`print:hidden`。固定バーとソフトキーボードの干渉（前回P2-4指摘）は **未解消の可能性**（コード上、キーボード回避処理は見当たらず）→ 要実機。
6. **AI提案パネル**：ボタン＋結果カード（`text-[11px]`〜）。小画面で結果が縦に伸びる。可読性は要実機。
