# 09. 「もし機能Xがあれば」追加機能候補

> レビューしてて「これがあれば実務がもっと楽になる」と思う追加機能候補(あくまで候補、提案レベル)。
> 既存機能の組合せで実現できるもの / 新規開発が必要なもの。各候補の必要性レベル(必須/便利/あったらいい)。

---

## 必須(現場で使い続けるためにないと困る)

### F1: 計画ジェネレータの「月次運用接続」(現状のPDCAサイクル切れを解消)
- **問題**: 計画ジェネレータが「年に1回作って棄てる」状態。月次運用との接続がない。
- **追加内容**:
  - プレビュー下に「今月の実施項目」「未達/達成のチェック欄」「月例レビュー」リンク
  - 月次ダッシュボード(/strategy/plan-monthly/[year])で「今月分のタスク・達成率」表示
  - 既存の /safety-diary の月次まとめと連携
- **既存素材**: SAFETY_PLAN_TEMPLATES の monthlySchedule・safety-diary の月次まとめ
- **工数**: 24h
- **必要性**: 必須(年次計画ツールの価値を担保)

### F2: 条文番号直接URL(/law/[law-slug]/[article-num])
- **問題**: 「518条」と打って原文に到達するまで4クリック+入力。
- **追加内容**:
  - URL ルート `/law/anzeneho-rule/518` を新設
  - ホームヒーローに「条文番号を直接入力 → ジャンプ」ボックス
  - 短縮URL(条文番号QR出力で現場掲示)
- **既存素材**: 安衛則だけで7,076行収録済、漢数字対応の `kanjiToNum` 関数
- **工数**: 12h
- **必要性**: 必須(法令系の最頻ユースケース)

### F3: KY → 日誌 ワンクリック転記
- **問題**: 朝礼でKYを書いて、別途日誌に手コピペ。3-5分目標が崩壊。
- **追加内容**:
  - KY保存後「この内容で日誌を書く」ボタン
  - workDetail/riskRows を workContent/kyResult に自動マッピング
- **既存素材**: 業種プリセット辞書、 schema 共通化済
- **工数**: 8h
- **必要性**: 必須(用紙ファーストUIの完成度を担保)

### F4: 「直近の重大労災ピックアップ」(ホーム)
- **問題**: 朝礼前に「今週どこで死亡事故あった?」を見たい。/accidents-reports は直近フィルタなし、/accidents は severity ソートのみ。
- **追加内容**:
  - ホームに「直近1週間の重大労災 3件」をピックアップ表示
  - 業種フィルタ付き
  - 「業種コンテキスト検出時に建設業の事故を優先」
- **既存素材**: accidents データセット、occurredOn フィールド
- **工数**: 6h
- **必要性**: 必須(ホームの実務価値担保)

### F5: 月例「安全衛生委員会報告書テンプレート」
- **問題**: ペルソナB2(製造業ライン長)で「月1の安全衛生委員会報告書テンプレ」が完全✗。専用機能が存在しない。
- **追加内容**:
  - /safety-committee/template 新設
  - 議題テンプレ(産業医報告/職場巡視結果/災害発生状況/対策方針/法改正情報)
  - 月次日誌・KYの集計を自動転記
  - PDF/DOCX出力
- **既存素材**: safety-diary 月次まとめ、産業医制度ガイド、法改正カレンダー
- **工数**: 20h
- **必要性**: 必須(50人以上事業場の月例義務)

### F6: 「労働者本人セルフチェック」(57項目ストレスチェック)
- **問題**: /mental-health-management に「自社整備状況」しかなく、本人診断ができない。来訪者半数の期待を裏切る。
- **追加内容**:
  - /mental-health-management/self-check 新設
  - 厚労省57項目調査票(現実は端末内処理・匿名)
  - 高ストレス判定→「面接指導 申出フォーム」リンク
- **既存素材**: 既に厚労省57項目はパブリックデータ
- **工数**: 8h
- **必要性**: 必須(メンタル対策ハブの主要ユースケース)

## 便利(あると現場の効率が大幅に上がる)

### F7: 化学物質「複数混合RA」(加算則の暫定実装)
- **問題**: 塗料・洗浄剤の混合作業は実務頻出。現状の chemical-ra は単一物質のみ。
- **追加内容**:
  - 複数物質ピッカー
  - 加算則(各物質のばく露量÷OEL の合計が1以下か)で簡易判定
- **工数**: 16h
- **必要性**: 便利(製造業の中位ニーズ)

### F8: 「日報→月報→年報」の自動集計
- **問題**: 日誌が日単位で完結し、月次・年次の集計が現場担当者の手作業。
- **追加内容**:
  - safety-diary 既存の月次まとめを拡張
  - 年次レポートPDF(月別労働延人数・KY実施率・ヒヤリ件数・労災発生率の年次推移グラフ)
- **既存素材**: localStorage v3 の日誌記録、Recharts
- **工数**: 12h
- **必要性**: 便利(月次まとめ既存)

### F9: 「現場ダッシュボード」(WBGT+業種別事故+R7チェック一体型)
- **問題**: 朝礼前に「今日の気象+今週の事故+R7チェック未完項目」を1画面で見たい。
- **追加内容**:
  - /signage の subscenario「朝礼前」モードを拡張
  - WBGT実測値+業種別事故Top3+R7チェック未完を1画面
  - 既存サイネージとの連動(全画面・自動更新)
- **既存素材**: WBGT計算機、accidents-reports、R7チェックリスト、signage shell
- **工数**: 8h
- **必要性**: 便利(Wow候補)

### F10: 「アルコール検知器・点呼チェック」(運輸業対応)
- **問題**: ペルソナD6(運送業)で「アルコール検知器=貨物自動車運送事業法」が33法令外✗。
- **追加内容**:
  - /for/transportation の専用ランディング(現状なし)
  - 点呼チェックリストテンプレ・PDF出力
  - 道交法・労契法・改善基準告示の条文収録拡張
- **既存素材**: industries/transportation のコンテンツ
- **工数**: 24h
- **必要性**: 便利(運輸業向け展開)

### F11: 「日常運用ネタ供給」自動生成
- **問題**: ペルソナA4「朝礼ネタ」、A9「KYネタ」が完全△。日常運用ネタが供給されない。
- **追加内容**:
  - /briefing-ideas 新設(朝礼ネタ・KYネタを月次配信)
  - 業種・季節・天気で自動選定
  - 直近事故・法改正・通達と紐付け
- **既存素材**: accidents・circulars・law-revisions
- **工数**: 16h
- **必要性**: 便利(継続利用率向上)

## あったらいい(理想的)

- **追加内容**: 条文/通達/KYカード/サイネージにQR出力。スマホで現場掲示物を撮影してその場で閲覧。
- **工数**: 4h
- **必要性**: あったらいい

### F13: 「条文diff表示」(法改正前後比較)
- **追加内容**: 法改正カレンダーから「改正前/改正後」の条文を並列表示。
- **工数**: 12h
- **必要性**: あったらいい(法務担当向け)

### F14: 「Word/Excel出力」(計画ジェネレータ)
- **問題**: 多くの安全管理者は印刷後Wordで再作成。
- **追加内容**: `.docx`/`.xlsx` 出力
- **工数**: 8h
- **必要性**: あったらいい

### F15: 「修了証発行」(eラーニング・特別教育)
- **状況**: 現状「2026年秋」予定。
- **追加内容**: 受講者進捗→修了スコア→PDFで証明書発行(社名・氏名・スコア・QR)
- **工数**: 16h
- **必要性**: あったらいい(法人向け)

### F16: 「業種別 /for/[業種]」を10業種展開
- **問題**: /for/construction のみ存在。役職別UIパターンが建設業に閉じている。
- **追加内容**: 製造業・運輸・医療福祉・小売・飲食・サービス・倉庫・卸売・農業・林業の9業種に展開。
- **工数**: 80h(1業種8h × 10業種)
- **必要性**: あったらいい(SEO+ペルソナ非建設対応)

## 「もし追加するなら」トップ3(オーナー判断材料)

### 1位: F1 月次運用接続(計画ジェネレータ)
- 工数: 24h
- なぜトップ: 既存の plan-generator を「使い捨て」から「PDCAサイクル」に格上げできる
- 効果: 計画ジェネレータの実用度が劇的に上がる(継続利用率がゼロから70%へ)

### 2位: F6 労働者本人セルフチェック(メンタル)
- 工数: 8h
- なぜ2位: 既存ハブの主要欠落、来訪者半数の期待を埋める
- 効果: メンタル対策ハブが「事業者向け実務」から「事業者+労働者の両輪」に

### 3位: F4 直近の重大労災ピックアップ(ホーム)
- 工数: 6h
- なぜ3位: ホームの実務価値を担保、ペルソナA(建設業職長)の朝礼前ニーズに直接応える
- 効果: 「ホーム着地で当日価値を得られる」体験 = リテンション向上

合計工数: 38h(約5営業日)で「サイト全体の価値提案」が一段上がる。

## 必須6件の優先順位(全部実装の場合)

1. F2 条文番号直接URL(12h) → 法令系の最頻ユースケース
2. F3 KY→日誌転記(8h) → 用紙ファーストUI完成
3. F4 直近重大労災(6h) → ホーム価値
4. F6 本人セルフチェック(8h) → メンタル価値
5. F1 月次運用接続(24h) → 計画PDCA
6. F5 月例委員会報告書テンプレ(20h) → 月次義務対応

合計工数: 78h(約10営業日)

## オーナー判断ポイント

- **追加路線か削減路線か**: 削減41h で 9機能統合・削減 → 体験が上がる。さらにF4(6h)+F6(8h)+F3(8h)=22h で「必須3件」追加。合計63h で全体最適化が見える。
- **F5(月例委員会報告書テンプレ)**: 20h と重いが、有料化の有力候補機能(法定義務対応)
- **F1(月次運用接続)+F5(委員会報告書)**: セット展開で「年次→月次→委員会」のフローを完成

オーナーが判断すべき:
1. **追加機能を作る vs 削減・統合に集中する** の方針
2. **オーナー確認案件(F1, F5, /community-cases永続化)** の優先順位
3. **/for/[業種] 展開**(建設業以外の9業種・80h)を着手するか
