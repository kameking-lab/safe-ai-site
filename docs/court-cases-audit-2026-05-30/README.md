# 労災裁判例コーナー 事前レビュー（データ源・法的リスク・見せ方）

作成: 2026-05-30〜31 / 自律ループ R4 タスクB / Opus 4.8

労働安全コンサル業務で価値の高い「労災裁判例集」新設にあたり、実装前にデータ源の取得可能性・法的リスク・見せ方を調査した記録。**捏造防止を最優先**とし、実在確認できる確定判例のみを扱う。

---

## 1. データ源の取得可能性

| データ源 | 取得方法 | 機械可読性 | 本コーナーでの扱い |
|----------|----------|------------|--------------------|
| 裁判所「裁判例検索」 (courts.go.jp/app/hanrei_jp) | Webの判例検索。判例ごとに `detail2?id=XXXXX` の固定URL | 公開API無し。スクレイピング規約は未明示だが**判決文自体は著作権の対象外**（著作権法13条3号：裁判所の判決・決定・命令は権利の目的とならない） | **手動キュレーション**。各判例の固定URLを出典リンクに使用 |
| 厚労省 各種判例解説 (mhlw.go.jp, kokoro.mhlw.go.jp, check-roudou.mhlw.go.jp) | 公的PDF・解説ページ | PDFは抽出困難。HTML解説は読める | 要旨の裏取り（事実確認）に使用。出典リンク候補 |
| 判例集の市販DB（労判・民集 等） | 有償 | — | 引用情報（巻号頁）の表記にのみ使用。本文は転載しない |

**結論**: スクレイピングや自動取込みはせず、**人手で実在確認した代表判例を手動キュレーション**する（レベルA）。各判例に裁判所サイトの固定URL等の一次情報リンクを付す。

## 2. 法的リスク評価

- **著作権**: 判決文（裁判所の判決・決定）は著作権法13条により著作権の目的とならない＝引用・要約は適法。ただし**判決文の長文転載は行わず、自サイトの要約＋原文/出典へのリンク**とする（労災ニュース /accident-news と同じ作法）。解説記事（法律事務所等）の文章はコピーしない。
- **当事者の特定**: 裁判所「裁判例検索」は**当事者を匿名化済み**（個人名は記号、会社名も多くは伏字/通称）。本コーナーは判例の通称（陸上自衛隊事件・川義事件 等、学術・実務で定着した呼称）を用いる。新たな個人特定情報は載せない。
- **断定的法的助言の禁止**: 「この判決は〜と判断した」（事実）はOK。「だからあなたは〜すべき」（個別の法的助言の断定）は**禁止**。実務ポイントは一般原則レベルに留め、最後に専門家（弁護士・社労士・労働安全/衛生コンサルタント）相談へ誘導する。
- **正確性**: 判決年月日・裁判所・法廷・争点・判断要旨は、複数の信頼できる出典で裏取りした事実のみ記載。**「ありそうな判例」を作らない**。

## 3. 見せ方（UX）

- 既存 /accident-news（重大災害事例）と同じ「要旨＋出典リンク、長文転載なし」の作法を踏襲。
- 各判例: 事件名（通称）/ 裁判所・法廷 / 判決年月日 / 出典（巻号頁・固定URL）/ 争点 / 事案の概要 / 裁判所の判断（要旨）/ 実務上のポイント（一般原則）/ 関連機能リンク。
- フィルタ: 争点（安全配慮義務 / 過失相殺 / 元請・下請責任 / 国・行政責任 / 労働者性）・分野（過労/メンタル・じん肺/石綿・建設 等）・年代。
- 連携: KY・打合せ書（安全配慮義務の現場実装）/ 事故DB（賠償と過失相殺）/ チャットボット（条文・判例案内）。
- **冒頭に免責**: 「判例の要旨は当サイトによる要約。正確な内容は出典の判決原文を確認。個別案件は専門家へ相談」。

## 5. 実装方針（レベルA：キュレーション型）

- 新規ルート `/court-cases`（一覧＋争点フィルタ）と `/court-cases/[id]`（詳細）。
- データは `web/src/data/court-cases.ts` に型付き定数で保持（モックではなく**実在判例の事実データ**）。各レコードに `verifiedSources`（裏取りURL）を持たせ、出典リンクを必須化。
- 既存の AccidentHubNav 等と同様のハブ内に導線を追加。/accident-news と並ぶ「調べる」系コンテンツとして配置。
- noindex はしない（公表事実の解説。SEO価値あり）。ただし免責・出典を明示。
- 架空生成は一切しない。AI要約も使わず、上記の裏取り済み事実のみを人手で記述。
