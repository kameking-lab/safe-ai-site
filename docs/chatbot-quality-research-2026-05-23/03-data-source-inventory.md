## 3. 著作権・利用規約

### 3.1 法令本文(e-Gov 由来)

- 著作権法第13条: 「憲法その他の法令」は権利の目的とならない
- 再配布可、改変可
- 出典明記推奨だが法的義務ではない
- 本プロジェクト方針: e-Gov に直接リンク + 本文の引用は要点抜粋に限定(全文転載は技術的にも避ける)

### 3.2 通達・告示・指針

- 政府標準利用規約 第2.0版(CC BY 4.0 互換)
  - 帰属表示: 「出典: 厚生労働省」必須
  - 改変: 注記すれば可
  - 再配布: 可
- 本プロジェクト方針: title + noticeNumber + issuedDate + 出典URL の構造化収録は問題なし。 本文転載は引用要件を満たす範囲(出典明記、改変なし、本文の従属性)

### 3.3 リーフレット原文 PDF

- 厚労省刊行物: 政府標準利用規約
- JNIOSH(労働安全衛生総合研究所): 政府標準利用規約 + 帰属義務
- 建災防: 個別資料ごとに利用条件が異なる。 公開ページからのリンクは可、本文転載は要許諾
- 本プロジェクト方針: メタデータ + 直リンク URL の保持に限定。 本文転載なし

### 3.4 JAISH 編集解説

- 著作権あり(中央労働災害防止協会の編集著作物)
- 二次利用不可
- 本プロジェクト方針: リンクのみ。 本文転載・要約禁止

### 3.5 「個人運営研究プロジェクト」体裁との整合(PR #182 P0)

PR #182 で確定した方針(docs/brand-consistency-audit-2026-05-17.md):
- 「専門家チーム」表現 → 「労働安全衛生コンサルタント(登録番号260022)」に変更済
- 「無料相談」 CTA → 「ご意見・改善提案を送る」に変更済
- 「完全対応」「完全網羅」表現 → 「対応」「主要」に軽減
- サービス体裁 → 研究プロジェクト体裁

通達収録拡大との整合性:
- 「研究プロジェクト」体裁を維持しつつ通達 DB を拡張することは可能
- 「商用サービス」とは表記せず、「個人の研究プロジェクトが公開している学習用資料」として位置づける
- 通達 1,069 件 → 1,500 件以上に拡張する場合も、著作権表示と出典 URL を全件付与する前提で問題なし
- 重要: 通達本文の AI 生成要約は「JAISH 編集解説の置換」とみなされるリスクあり。 要約は title + 拘束力 + 出典 URL の構造化情報レベルに留める

## 4. 拡張優先度の総括(トップ5)

優先度 = (建設業頻出度 × 実装容易性 × 著作権リスクの低さ) で簡易評価:

- 優先1: 安衛則 第518〜575条群(足場・墜落・掘削) PIN 整備 + 不足条文補強
  - 建設業頻出度: 最高
  - 実装容易性: 既収録(ashibaSagyoKisoku)、PIN 拡張のみ
  - 著作権リスク: 無し(法令本文)
  - 推定工数: 1-2日
- 優先2: 建災防規程の追加条文(現状8条 → 全条文)
  - 建設業頻出度: 高
  - 実装容易性: 公開資料、構造化収録可
  - 著作権リスク: 低(公的団体の規程)
  - 推定工数: 1-2日
- 優先3: 通達 DB に「建設業労働災害防止計画(第14次)」関連通達を追加
  - 建設業頻出度: 高
  - 実装容易性: 既存スキーマ + jsonl 追記
  - 著作権リスク: 低(政府標準利用規約)
  - 推定工数: 2-3日
- 優先4: e-Gov 法令API v2 の月次バッチ取込み実装
  - 建設業頻出度: 中(間接効果)
  - 実装容易性: API 仕様シンプル
  - 著作権リスク: 無し
  - 推定工数: 2-3日(初回実装)、以降は自動
- 優先5: リーフレット mhlw-leaflets.ts の建設業向け資料を 50 件以上追加
  - 建設業頻出度: 高(現場用ポスター・教材)
  - 実装容易性: メタデータ + 直リンク URL 追加のみ
  - 著作権リスク: 低(リンクのみ)
  - 推定工数: 1-2日
