## 1. 改修の目的(社長要求の品質基準)

- 自由な日本語入力 → 安衛法本文+関連条文を参考に何でも回答
- 該当条文がない場合は最も近い条文を提示。条文番号は絶対に間違えない
- 安衛則・クレーン則・有機則・特化則等を必要に応じて動的取得
- 通達・告示・リーフレットの原文URLを添付

## 2. 現状の到達度(2026-05-23時点)

- 法令コーパス: 50法令(33基幹 + 拡張 + 施行令類)。LawArticle 配列で構造化済み
- RAG パイプライン: 同義語展開 → トークン化 → BM25+デンス → リランク → PIN(55トピック)→ 信頼度判定 → Gemini Flash 呼出。ファイル: web/src/lib/rag-search.ts(919行)
- 通達 DB: 約 1,069 件(mhlw-notices.ts)。binding/indirect/reference の3段階拘束力分類
- リーフレット DB: 600件超(mhlw-leaflets.ts)。pdfUrl/sourceUrl/detailUrl の3 URL を保持
- 国交省資料 DB: 100件(mlit-resources.ts)
- 評価: 内部ベンチ Recall@5 = 100%(main n=115)、Recall@5 = 100%(fresh n=100)
- ハルシネーション対策: detectOutOfScopeLawReferences(37法令ホワイトリスト)、detectUngroundedAssertions(weasel word 検出)、PIN による条文番号強制ピン留め

## 3. 主要発見(3件)

1. **「条文番号は絶対に間違えない」の保証は現状 確率的軽減にとどまる**。 RAG 検索結果に含まれる条文番号のホワイトリストを Gemini プロンプトに同梱して逸脱を拒否する Pre-generation ガードがない。Post-generation の detectOutOfScopeLawReferences は法令略称レベルのフィルタで、条番号の枝番ゆれ(例: 第563条 → 第563条の2)は検出できない。
2. **ペルソナテスト(PR #261)で失敗した 7+4=11 件のうち、コンテンツ不足由来は最大 7 件**(貨物自動車運送事業法、道交法、労契法、感染症法、カスハラ指針、土砂崩壊点検通達等)。残り 4 件は「実務対応フロー」型(法令でなく通達 + マニュアル)で、Phase 4 の通達 URL 添付で改善余地。
3. **建設業向け振り切り方針との整合で優先度トップ3 は (a) 安衛則の足場・建設関係条文の収録範囲確認 + 建災防規程の補強、(b) 通達/告示の構造化(主要1,069件のうち建設業関連の binding level 再キャリブレーション)、(c) 厚労省リーフレット原文URLの応答内インライン表示**。 e-Gov 法令API、貨物自動車運送事業法、労契法は次点。

## 4. オーナー判断必須事項(全 7 件)

詳細は 08-owner-decisions.md。要約:

- (D1) 建設業振り切りの程度: 33法令→50法令→「建設業重視で 60法令+α」のどこまで広げるか
- (D2) 通達/告示原文URL の応答内表示形式: インライン展開 vs 折りたたみ vs 別ページ
- (D3) e-Gov 法令API v2 の定期取込みを実装するか(現状静的データ)
- (D4) Gemini Flash → Pro 切替の是非(品質 vs コスト)
- (D5) 「個人運営研究プロジェクト」体裁との整合(PR #182 P0)を保ちつつ通達収録を増やせる範囲
- (D6) ハルシネーション Pre-gen ガード実装時のフォールバック動作(rejection 後の応答内容)
- (D7) サブスク課金との結合(高度法令検索を有料化するか)

## 5. 実装ロードマップ(4 Phase)

詳細は 07-implementation-roadmap.md。

- Phase 1: RAG コーパス整備(3-4日相当)。 構造化条文DB完成、不足省令補強
- Phase 2: ハルシネーション絶滅3層実装(5-7日相当)。 Pre-gen 同梱・Post-gen 照合・Fallback の3層
- Phase 3: 「最も近い条文」フォールバック(3-5日相当)。 完全一致/部分一致/不一致の3分岐
- Phase 4: 通達・告示・リーフレットURL添付(3-5日相当)。 条文-通達のマッピングDB完成

合計 14-21日相当(1人実装ベース)。順次実装が前提で、Phase 2 着手は Phase 1 完了後。

## 6. 次に着手すべき Phase

- Phase 1 を先行着手すべき。 ハルシネーション絶滅(Phase 2)は構造化条文DBの存在を前提とするため。 Phase 1 完了で Recall@5 → Citation Accuracy@1 という評価軸への切替が可能になる
- ただし Phase 4(通達URL添付)は既存データを使うため Phase 1 と並列実装可。 建設業頻出通達のメタデータ精査だけ先行する
