# 05. 機能間の回遊性（軸5）

評価: **83 / 100**

## 現状
- **相互リンクは多数実装済**: `RelatedPageCards` / `cross-tool-links` / `main-feature-next-actions` などで「合わせて使う」「このデータを活かす」セクションを25+ページに配置。
- **転記動線**: KY→安全工程打合せ書のワンクリック転記（P0-010）、各「本日トピック」→AI注意喚起文生成→朝礼利用、など“次の一歩”が設計されている。
- **業種ハブ `/industries/[industry]`** が事故レポート・KY例・通達・化学物質・特別教育・年次計画を業種軸で束ねる横串として機能。

## 問題

### 【P1】事故系4ルートの回遊が分かりにくい
`/accidents`（10年DB検索）/ `/accident-news`（重大災害事例・匿名）/ `/accidents-reports`（業種別分析）/ `/accidents-analytics`（統計）。
- 4つの違い・使い分けを説明する**「事故情報のハブ」が無い**。各ページは相互リンクするが、初訪問者は「どれを見ればいいか」の地図を持てない。
- 名称も4ページで割れる（→ `10`）。

### 【P2】教育・法令クラスタの回遊
- 教育3系統・法令4系統間の遷移はリンクで繋がるが、「今いるのがどれで、隣が何か」のオリエンテーションが弱い。各ハブ冒頭の1行ガイドで解消可能。

### 【P2】「次に何をすべきか」の一貫性
- `main-feature-next-actions` の様な“次の一歩”誘導が主要機能には付くが、ロングテール機能（熱中症・メンタル等）では実装にばらつき。

## 改修方針
- 事故系・教育系・法令系それぞれに**軽量な「クラスタ案内」**（冒頭1行＋兄弟機能チップ）を置き、回遊の地図を与える。既存リンク資産を結線するだけで実装可能。
