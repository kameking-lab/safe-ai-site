# 06. ターゲット別の入口（軸6）

評価: **76 / 100**

## ターゲットと現状の入口

| ターゲット | 専用入口 | 現状 |
| ---- | ---- | ---- |
| 中小ゼネコン・現場監督 | ✅ `/for/construction` | 職長・元請担当・現場代理人の3ロール、法令・化学物質・統計を集約。**手本となる出来**。ヒーロー直下に昇格済。 |
| 個人事業主・一人親方 | ❌ 無し | `/diversity`・問い合わせフォームに言及はあるが専用ランディング無し |
| 労働安全コンサル | ❌ 無し | 問い合わせフォームに `safety-consulting` 選択肢あり。だが「コンサル向けに何ができるか」の入口無し |
| 安全担当者 | △ 部分 | `/organization`（全社集計の見せ方）等が近いが「安全担当者向け」と銘打った入口は無し |

## 主要な問題

### 【P1】ペルソナ入口が建設業に偏在
- 4ターゲットのうち**専用入口は建設業のみ**。`src/app/(main)/for/` 配下は `construction/` だけ。
- 問い合わせフォーム（ContactForm）には `solo`（一人親方）・`safety-consulting`（コンサル）の区分が既にあり、**運営側はペルソナを認識しているのに入口が無い**ギャップ。
- 一人親方は「無料・登録不要・中小・現場」という中核コンセプトのど真ん中の層。入口欠落は機会損失。

### 影響
- 検索流入したコンサル/一人親方が「自分向け」と感じる着地点が無く、汎用トップに流れて離脱しやすい。

## 改修方針（段階・新規ページなので既存非破壊）
- `/for/construction` を**テンプレート**に、最小構成のペルソナ入口を追加検討:
  - `/for/solo`（一人親方・個人事業主）: 特別加入・KY・熱中症・事故事例・必要資格判定 を1ページ集約
  - `/for/consultant`（労働安全コンサル）: 法令検索・通達判例・体系マップ・事故分析レポート・年次計画テンプレ・チャット を集約
  - 安全担当者は当面 `/organization` を整備し導線を明示
- **注意**: 薄いページの量産は軸4のロングテール問題を悪化させる。**中身のある集約ページ**としてのみ追加（社長判断推奨）。本セッションでは設計提示に留める。
