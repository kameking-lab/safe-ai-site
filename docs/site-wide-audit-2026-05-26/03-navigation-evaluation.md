# 03. ナビゲーション評価（軸3）

評価: **82 / 100**

## ナビ面の棚卸し
| 面 | 表示条件 | 内容 | 件数 |
| -- | ---- | ---- | ---- |
| PC左サイドバー（AppShell） | lg+ | 6動詞カテゴリ | ~30 |
| 上部 FlagshipNav（横並び＋hover popover） | lg+ | FLAGSHIP_FEATURES | 10 |
| PC上部バー | lg+ | ⌘K検索・屋外モード | 2 |
| モバイルヘッダ | <lg | 検索・テーマ・ユーザー・メニュー＋ふりがな/文字大 | — |
| FlagshipNav モバイル（右ドロワー） | <lg | 主要機能10＋サブ | 10 |
| MobileBottomNav（固定） | ~<480px | ホーム/KY/AIチャット/打合せ書/もっと | 4+5 |

## 良い点
- **6カテゴリ再編＋各カテゴリ1行説明**（`description`）で選択肢過多を解消済。
- **現在地表示**: `navActive()` で active 強調（emerald 背景）。`aria-current` も底部ナビで付与。
- **NEWバッジに有効期限**（`badgeUntil`）。30日で自動消灯し陳腐化を防ぐ良実装。
- **モバイル底部ナビ**: 職長頻用4＋「もっと」シート。タップ48px確保・safe-area対応。
- **a11y**: skip-to-main、`aria-label`/`aria-expanded` 適切。

## 問題点

### 【P1】PCで2系統のグローバルナビが同時表示
左サイドバー（6カテゴリ30項目）と上部 FlagshipNav（10項目）が**同じ画面に常時併存**。
- 同じ機能（KY・チャット・サイネージ等）が2箇所に出る → 情報の二重化。
- 「主要機能」の意味が、サイドバーのカテゴリ分類と FlagshipNav の10選で**二重定義**（軸2 P0と連動）。
- 現在地ハイライトが2系統に分散し、視線が割れる。
**改修案**: 上部 FlagshipNav を**ページ内の文脈ナビ（パンくず/関連）に格下げ**、もしくはサイドバー非表示時（モバイル）専用にする等、PCでは1系統に寄せる。※既存リンクは維持しSEO/導線を壊さない段階展開。

### 【P2】モバイルのナビ面が3層
モバイルヘッダのメニュードロワー（=サイドバー相当）＋ FlagshipNav 右ドロワー＋底部ナビ。3つの“メニュー”が併存し、初訪問者がどれを開くべきか迷う。底部ナビ＋ヘッダメニューの2層に集約余地。

### 【P3】項目数
サイドバー約30項目はカテゴリ化で許容範囲だが、「学ぶ」7・「分析する」7はやや多い。重複系統（軸2 P1）の統合で各5以内に収まる。

## 改修方針
- PC: 2系統ナビを1系統へ寄せる設計（段階）。
- モバイル: ナビ面を2層に整理（段階）。
- いずれも**既存href全維持**を絶対条件に。
