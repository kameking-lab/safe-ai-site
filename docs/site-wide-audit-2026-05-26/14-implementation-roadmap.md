## 原則
1. **既存9機能を壊さない**（href維持・回帰テスト・段階展開）。
2. **URL変更を伴う再構成は社長判断**（SEO評価毀損リスク）。
3. **新規ページは“中身”が伴う場合のみ**（ロングテール問題を悪化させない）。
4. **ビルドコスト中立**（新規依存/重処理を増やさない）。
