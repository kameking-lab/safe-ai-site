## 現状の分類体系
PCサイドバー `NAV_CATEGORIES`（app-shell.tsx）は**動詞ベース6カテゴリ**に再編済（P0-021）:
- （トップ）ホーム / 新着情報 / お気に入り
- **現場で使う**: KY用紙・打合せ書・サイネージ・熱中症・化学物質RA
- **質問する**: 安衛法AIチャット
- **学ぶ**: 法令検索・通達判例・法改正カレンダー・法令体系マップ・用語辞書・FAQ・Eラーニング
- **分析する**: 業種別事故分析・事故DB・重大災害事例・化学物質検索DB・メンタル・多様性・AIリスク予測
- **計画する**: 年次計画・気象リスク・治療と仕事の両立・助成金
- **業種から**: 建設業実務・10業種ハブ・メンタルカスハラ
- プロジェクト: about・contact・goods・通知・機能紹介一覧

この動詞ベース再編は良設計。職長語彙に寄せられている。

## 主要な問題

### 【P0】「主要機能」の定義が4通り
| 定義元 | 内容 | 件数 |
| ---- | ---- | ---- |
| ヒーロー3大CTA／フッター「主要機能」 | AIチャット・事故分析レポート・年次計画 | 3 |
| `FLAGSHIP_FEATURES`（上部ナビ＋グリッド） | 打合せ書・KY・化学物質RA・サイネージ・法改正・AIチャット・事故・教育・業種・作業環境 | 10 |
| `MobileBottomNav` PRIMARY | ホーム・KY・AIチャット・打合せ書 | 4 |
| サイドバー6動詞カテゴリ | 上記30+項目 | — |

**4つの“顔”がある。** AIチャットだけが全てに共通。事故分析レポートと年次計画は「メイン3」なのにグリッドではサブ項目。→ 初訪問者の「主役は何か」認識が定まらない。
**改修**: メイン3＝「最初に試す3機能」、Flagship10＝「機能の全体像」と**役割を明文化**し、コピーを揃える。最低限、メイン3（事故分析・年次計画）をグリッドにも代表カードとして出すか、ヒーローCTAのラベルをグリッドと一致させる。

### 【P1】機能の重複系統（前回監査から継続・未棚卸し）
- **教育3系統**: `/education`（特別教育コース受講）/ `/education-certification`（資格判定DB・60+40種）/ `/e-learning`（クイズ式コース）。違いが説明されず入口が3つ。
- **法令4系統**: `/laws`（改正カレンダー）/ `/circulars`（通達判例）/ `/law-search`（条文検索）/ `/law-hierarchy`（体系マップ）。「学ぶ」配下に4つ並ぶが初見では役割差が不明瞭。
- **事故4系統**: `/accidents` / `/accident-news` / `/accidents-reports` / `/accidents-analytics`（→ `05`）。
- **化学物質2系統**: `/chemical-ra`（RA実施）/ `/chemical-database`（物質検索）。これは役割が明確で許容範囲。

### URL設計・パンくず
- **URL設計は概ね一貫**（機能名/サブ機能のケバブケース、`[param]` 動的）。`/for/construction`・`/industries/[industry]` などハブ構造も論理的。
- **パンくず**: `breadcrumb.tsx` 存在、178ページ中89ページ（約半数）で採用。トップ階層の機能直下ページは未採用が多い。深いページ（`/industries/[industry]`・`/safety-diary/[id]` 等）にこそ必要。

## 改修方針
- メイン3とFlagship10の役割分離を文言で統一（P0）。
- 教育・法令クラスタに**「どれを選べばいい？」1行ガイド**を各ハブ冒頭に（低リスク・高効果）。
- パンくず未採用の主要機能直下ページへ展開（段階）。
