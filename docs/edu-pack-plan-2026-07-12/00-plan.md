# 法定教育スライド無償公開パック — 企画書(2026-07-12)

- **結論**: 既存の教育12プログラム(10枚PPTXサンプル+営業ページ)を、「e-Gov正本スナップショット×CI機械照合で法定科目・時間の充足を保証し、月次ETLで最新災害データに自動追従する、申請不要・編集可の無償フル教材」へ転換する。競合実査で「無償・申請不要・PPTX編集可・法定対応表つき」の4条件を同時に満たす教材は市場に存在しないことを確認済み(07章)。初弾5本=熱中症・フルハーネス・粉じん・低圧電気・酸欠。実装はしない(本docs+BACKLOG注入のみ)。
- 章構成: 01=法令線引き / 02=網羅ゲート / 03=ラインナップ / 04=データ駆動 / 05=利用規約・配布 / 06=CV導線 / 07=競合調査。
- 調査の一次性: 法令・告示・通達は全て一次資料(e-Gov・厚労省法令等DB・JAISH)を2026-07-12に確認。競合はWeb実査(取得不能項目は07§4に正直に残置)。リポジトリ実態はコード実査。

## 1. 差別化の二本柱(社長構想の実装原理)

1. **法定充足の機械保証**(02章): 告示正本の科目・範囲・時間を構造化した「カリキュラム正本レジストリ」と、スライド→法定項目の対応表(covers)をCIで照合。1項目でも未カバーならCIが赤 — fidelityゲート(plain-fidelity.test.ts)と同思想・同実装様式(最近傍の雛形は special-education.test.ts の OFFICIAL_ART36_ITEMS)。「保証」は宣伝文句でなく、教材に印字される対応表+検証日として出す。
2. **災害データの常時反映**(04章): hazard-slides(#881)の確立済み追従モデル(月次ETLコミット→再デプロイ→静的再生成)を流用し、統計・速報・死亡個票事例をデッキに織り込み、データ基準日を教材に印字。競合は全て発行時点で静止している(07章)。

## 2. 法令線引きの要点(01章の要約)

- 特別教育: 実施義務者は事業者(法59条3項)。講師の法定資格要件なし・記録3年(則38条)・実技は対面必須(eラーニング通達)。**教材提供は教育の実施ではない**を固定文言・実技非代替カード・実施チェックリストの3点セットで教材自体に焼き込み、存在を網羅ゲートで機械検証。
- 技能講習: 登録教習機関のみ実施可(法61条1項・76条・77条)。初弾から技能講習系を除外し、将来出す場合も「受講準備・復習用」バナーを機械検証。修了証は一切発行しない(日本一戦略の「参入しない線引き」維持)。
- 通達ベース(熱中症): 則612条の2(2025-06-01施行)+基発0520第6号+基発0318第1号ガイドライン。法定時間が無いため科目網羅のみ照合。
- 副産物: 本番公開中の法令記載誤り3件を発見(フルハーネスの根拠を器具規格の「告示第11号」と取り違え/necchu.yaml の廃止通達残留/研削といしの実技時間欠落)。EDU-D1で是正。

## 4. 要オーナー確認リスト(統合)

Path A(env・外部サービス独断変更禁止)と文言専権に関わるもの。実装タスクはこれらを待たずに着手可能な設計(規約文言は定数化して後差し替え)。

1. **利用規約の最終文言**(05章§4): 販売禁止の線引き(登録教習機関の講習内投影は可、の整理でよいか)・クレジット表記の正式名称・再配布可否の最終判断。
2. **「オンデマンド配信 ¥50,000」との整理**(06章§2): 無償フルデッキ公開後の有償オンデマンドの位置づけ(進捗管理・質問対応付きなら共存可能という起案の可否)。
3. **既存ページの「修了証」記述**(06章§2): fullharness ページの「修了証をPDF発行(2026年秋予定)」は戦略の「修了証に参入しない」と矛盾。講師派遣(人が実施する講習)に限る旨へ修正するか削除するか。
4. **INQUIRY_INBOX / Resend の本番設定状況**(06章): 未設定でも受領ログには残るが、メール通知が届いているかの確認。
5. **講師要件の解釈例規の通達番号**(01章§1-A): 「昭48.3.19基発第145号」は二次情報のみ。教材に番号を載せるなら原本確認が必要(載せない設計で先行)。
6. 競合調査の未確認項目(07章§4): 対外的な比較表現(「業界初」等)を使う場合のみ要確認。使わない設計で先行。
