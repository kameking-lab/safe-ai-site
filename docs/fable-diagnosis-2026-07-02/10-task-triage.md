# タスク仕分け: Fable 5 / Opus 4.8 / Sonnet 5（2026-07-02）

診断書01〜08の全提案タスクを3モデルに振り分ける。原典タスクの完了条件・依存は各診断書を正とし、ここでは仕分けと理由・順序だけを一元化する。

## 仕分けの原則（6月の実証比較＋Sonnet 5の想定を反映）

- **【Fable 5】**（枠は7/7まで・手動セッション・本数を絞る）: 真因不明の診断、前任結論の再精査、監査ツール/パイプラインの発明、新アーキテクチャの「最初の1画面」。完成度より方式確立を成果物とし、量産は下位モデルに渡す。
- **【Opus 4.8】**: 確立方式の重量実装、**法令正確性が絡む是正の全て**、共有シェル・共有API等の衝突しやすい部分、原因調査を伴う修正、検証出荷。
- **【Sonnet 5】**: 確立パターンの量産横展開、UI定型、テスト追加、メタ/文言、掃除系。安価な主力量産機として本数を最も多く持つ。**法令の中身を自分で判断させない**（出典・完了条件が機械検証可能なタスクのみ）。
- 境界で迷ったものは「誤ったときの実害が法令誤案内に至るか」で上位モデルへ寄せた（理由は各所に付記）。
