## 2. 真因（コードで特定）

1. **本番デプロイが2026-06-14で停止している**。`git log` 上、6/14 10:29 JST の #577 以降の実コミットは全て `chore(jma): scheduled data update [skip ci]`（今日の docs コミットまで）。JMAデータは `web/src/app/api/signage/jma/route.ts` が `@/data/jma/*.json` を**ビルド時 import（force-static）**で焼き込む設計のため、GH Actions（`jma-data-update.yml`）が3〜4時間毎にリポジトリを更新しても `[skip ci]` でVercelに反映されず、**最後の実デプロイ（6/14）時点のデータで凍結**。ローカルの `web/src/data/jma/index.json` は `2026-07-02T07:50Z` と新鮮＝リポジトリは動いているのに本番だけ古い、を確認済み。（メモリの loop-runner UntilIso=2026-06-12 失効とも整合。デプロイが再開しても「デプロイ間隔ぶん古い」構造問題は残る）
2. **赤バナー「警報 発表中」の判定が粗い**。`resolveWeatherWarningPanelState` は県単位の `headlineText` が空でなければ `kind:"headline"` → `buildSignageConclusion` が赤「警報 発表中」を表示。実測では東京の headline は**伊豆諸島の注意報**（「…注意してください」）で、新宿区の市区町村別 `selectedWarnings` は**空配列**。つまり (i) 注意報を「警報」と誤表示、(ii) 現場と無関係な離島の情報で全画面赤。この状態が headline の続く限り**何週間も赤のまま**＝色の文法（赤=停止級）を自ら破壊し、信頼を失う最大要因。
3. **タイポグラフィがモバイル基準のまま**。全コンポーネントが `text-[8px]〜text-xs` 起点で `lg:`/`xl:` でも12〜18px止まり。遠距離設計されているのは後付けの結論ストリップ（xl:text-5xl）だけ。
4. **1画面フィット＋内部スクロール構造**（`SignageShell` の `xl:h-[100dvh] xl:overflow-hidden`、各セクション `overflow-y-auto`）に**ローテーション機構が無い**ため、隠れた情報（ニュース2件目以降・法改正2件目以降・時間別天気）は誰にも見えない。
5. **ニュースの選別ロジック**（`web/src/lib/signage/parse-labor-rss.ts`）が重大度スコアのみでソートし日付を無視、dedupeがlink基準（Googleニュースはクエリ毎に別URLを返すため同一記事が素通り）。
6. **地点マスタが30地点ハードコード**、設定はlocalStorage直書きで配信・共有の仕組みなし（真因というより未実装領域）。

## 3. あるべき姿（現場の休憩所ディスプレイに常掲して恥ずかしくない水準）

- **鮮度**: 全ウィジェットのデータ時刻が常に60分以内（防災系は30分以内）。画面上に「◯分前更新」を大きく表示し、2時間以上古いデータは自動で黄帯警告。
- **正確さ**: 赤=その現場の市区町村に警報級。注意報は黄、なしは緑。離島など無関係区域の情報で赤を出さない。
- **可読性**: 3〜5mから読む前提で、本文最小24px・主要情報32px以上・キーナンバー（気温/リスク/無災害日数）64〜96px。操作ボタン類は常掲時は隠す（キオスクモード）。
- **動き**: 15〜20秒周期で「天気→リスク→ニュース→法改正→（図面）」を全件ローテーション。人が触らなくても全情報が3分で一巡する。
- **価値**: ダミー図面をデフォルトから外し、地点の当日気象・WBGT・唱和/スローガン・無災害日数など「毎日変わる・現場の自分ごと」を主役に。
- **設定**: PC/スマホから設定→6桁コード/QRでTVに適用（KY朝礼と同じ方式）。全47都道府県の地点選択。

## 4. 改善方策の具体設計

### A. データ鮮度の回復（真因1）
- `web/src/app/api/signage/jma/route.ts` を**ビルド焼き込み→ランタイム取得**に変更: `force-static`/JSON import をやめ、気象庁 `bosai` JSONを `next: { revalidate: 1800 }` で直接fetch（`/api/signage-data` と同じ構え。既存の `jma-warning-codes.ts`/`parse-jma-warning.ts` を流用）。地震一覧は `bosai/quake/data/list.json`。GH Actions cron はフォールバック（Vercel Blob へ書く形に変えるなら `BLOB_READ_WRITE_TOKEN` 流用）に格下げ。
- **デプロイ健全性の監視**: 既存cron系（`vercel.json`）に「本番 `/api/signage/jma` の fetchedAt が24h超なら通知 or Vercel Deploy Hook を叩く」ジョブを追加。※Deploy Hook 追加は環境変数を伴うためオーナー確認事項。
- ダッシュボード側にも地図と同じ**失敗時3分リトライ**を移植し、更新間隔を60分→15分へ（上流CDN s-maxage=300 なのでEdgeコスト増は限定的。コスト懸念が残るなら30分）。
- **24時間毎のフルリロード**（`location.reload()`、深夜3時等）で新デプロイのJSを取り込む。

### B. 警報バナーの正確化（真因2）
- 判定を `selectedWarnings`（市区町村別・コードベース）主軸に変更: JMAコードの警報/注意報/特別警報区分テーブルを `web/src/lib/jma/` に追加（既存 `JMA_CODE_HINT` の6件ざっくり辞書を廃止）し、`buildSignageConclusion` を「特別警報=紫/赤・警報=赤・注意報=黄・なし=緑」に。県ヘッドラインは黄カード内の補足文に格下げ。
- 対象テスト: `signage-conclusion.test.ts`・`weather-warning-panel-state.test.ts` に「注意報のみ→黄」「離島注意報＋新宿選択→赤にしない」ケース追加。

### C. サイネージ用タイポグラフィ（真因3）
- `signage/` 配下専用のサイズ規約を定義（例: `SIGNAGE_TEXT = { body: "xl:text-2xl", label: "xl:text-xl", big: "xl:text-6xl" }` か Tailwind テーマの signage 用ユーティリティ）。対象: `signage-hourly-strip.tsx`（時刻7-9px→20px+、気温12px→28px+）、`signage-risk-prediction.tsx`（本文12px→24px）、トレンド・法改正カード（見出し18px→28-32px、1画面2件まで）、`signage-map` の右パネル（ヘッドライン11px→24px）。
- 常掲用「キオスクモード」（`?kiosk=1` or 全画面時）でヘッダーのリンクチップ・シナリオボタン等の操作UIを非表示にし、その分を本文サイズへ回す。

### D. 自動ローテーション（真因4）
- 新規 `web/src/components/signage/signage-rotator.tsx`: 子パネル（時間別天気/リスク/ニュース1件ずつ/法改正1件ずつ）を15〜20秒で切替、下部に進捗ドット。ニュース・法改正は**1件を大きく**表示して全件を周回。`prefers-reduced-motion` 配慮とホバー/タップで一時停止。
- 左メインは「図面アップロード済みなら図面、なければ地図＋時間別天気の拡大」をデフォルトに（ダミー図面はデフォルト表示から外し「図面を登録する」CTAに置換）。

### E. ニュース品質（真因5）
- `parse-labor-rss.ts`: (i) `pubDate` 14日以内フィルタ（不足時のみ30日まで緩和）、(ii) タイトル正規化（媒体名除去・空白圧縮）でdedupe、(iii) スコア=重大度×鮮度減衰（例: `seriousness * exp(-ageDays/7)`）。ユニットテスト追加。

### F. 地点と設定配信（真因6）
- 地点マスタを47都道府県庁所在地＋政令市に拡張（JMA `class20s` コードと緯度経度のテーブル生成。`signage-locations.ts` を data 生成スクリプト化）。将来は郵便番号/フリーワード検索。
- 新規 `/signage/settings`（PC向け）: 地点・表示モード・ローテーション間隔・図面/資料を設定→既存のKY 6桁共有（`cloudGetSignageSession` 系、Supabase稼働中）と同じ仕組みで `signage-config` を保存→TV側 `/signage?code=XXXXXX` で適用。図面画像はSupabase Storage or Vercel Blob（**オーナー確認: DBスキーマ追加に該当**）。

### G. 常掲価値の追加（差別化）
- 無災害日数カウンタ（起点日を設定、記録キットの事故記録と連動可能なら自動リセット）、今日の唱和/スローガン（曜日・季節・リスクから自動選択、朝礼スクリプト生成ロジックを流用）、WBGT表示（軸2の熱中症ログ `/heat-illness-prevention/log` と連動）。
