## 0. 結論（3行）

- **(a) 偽陽性は実在し、本番表示中**。機械走査で偽陽性・区分誤り **15件** を確定（例: 塩素を「特化則第三類」と表示＝正しくは第二類、クロロホルムを「有機則第一種」と表示＝2014年改正で有機則から削除され特化則特別有機溶剤。いずれも本番3,515ページ側で実測）。
- **(b) 欠落も実在**。毒劇法タグは法別表レベルの毒物（フッ化水素・水銀・ヒ素・シアン化水素・四アルキル鉛）すら欠落（7件確定・本番実測）。さらに**安衛法系の中でも**特別則表示はハードコード30 CASのみで、硫酸（特化則3類）・メタノール（有機則2種）・ガソリン（有機則3種）等の詳細ページには特別則が一切出ない。
- 真因は「**正本（施行令別表）と突合しない手書きマッピング**」＋「**2014年特別有機溶剤改正の未反映**」＋「**タグ語彙の欠落**（有機則3種・鉛則・四アルキル鉛則のタグが型に存在しない）」。6月のコーパス条番号監査（docs/fable-reexamination-2026-06-10/corpus-egov-full-audit）と同型の欠陥が物質DBに残っている。

**③ タグ語彙そのものの欠落（データモデル欠陥）**

**④ 他法令ドメインの構造的欠落（既知・一部は設計判断）**

**⑤ AI経路の無検証出力（偽陽性・欠落の両ベクトル）**

- `/api/chemical-ra/route.ts`（369〜376行）: 物質名の**部分一致 top-1**（`searchMergedChemicals(chemicalName, 1)`、スコア下限なし・`includes` 一致でもヒット）で拾った物質の規制情報を `relatedHazards` として回答に添付する。入力が曖昧・未収録物質名でも**別物質の法令該当が表示されうる**。
- 同 SYSTEM_PROMPT は Gemini に `regulatoryNotes`（「有機則第2種」「特化則対象外」等）を自由生成させ、**検証なしで表示**。構造化データと矛盾しても検知しない。SDS取込み（sds-extract）の「適用法令(参考)」も同様。

## 3. 真因

1. **正本不在の手書きマッピング**: 経路1・4は施行令別表と突合せずに書かれた。誤りパターンが体系的で、(i)「発がん性が強い＝第一類」という誤解（六価クロム/カドミウム/ヒ素/塩化ビニル/コールタール/石綿を1類に）、(ii)「酸・刺激性ガス＝第三類」という誤解（塩素/フッ化水素を3類に）、の2つの思い込みに集約される。
2. **2014年（平成26年）特別有機溶剤改正の未反映**: クロロホルム等10物質の有機則→特化則移行が反映されず、yuki-1タグと解説文が改正前のまま（12年遅れ）。
3. **監査の盲点**: 既存の `scripts/audit-chemicals.mjs` はCASチェックサム・OEL・皮膚等障害リスト整合のみ検査し、**特化則/有機則の区分は検査対象外**。5/26の deep-audit（docs/chemical-ra-deep-audit-2026-05-26/02）も特別則を「対応」と評価したまま中身を突合していない。ベンチが正本から独立でない問題は6月のコーパス監査（corpus-egov-full-audit-2026-06-10）で既に指摘された構図と同じ。
4. **ETLの出典がサードパーティミラー**: poison-control 等のタグは `github.com/Ameyanagi/ra-law-db` ミラー由来（JSONメタデータに記録）。e-Gov原本と再突合しておらず、毒劇法別表本体の物質が歯抜け。ETLスクリプト（fetch-concentration-limits.mjs）自体もリポジトリに現存しない＝再現不能。
5. **タグ語彙が法体系を表現しきれない**: yuki-3・鉛則・四アルキル鉛則・特定毒物が型に無い。データ整備以前にスキーマで落ちる。

## 4. あるべきデータ設計（法体系横断・物質マスタ）

### 4-1. スキーマ（CAS主キー＋法令ドメイン別レコード）

```ts
// 提案: web/src/data/legal/substance-legal-profiles.json (ETL生成物・手書き禁止)
type LegalDesignation = {
  domain: "anei-tokka" | "anei-yuki" | "anei-namari" | "anei-4alkyl" | "anei-funjin"
        | "anei-sekimen" | "anei-label-sds" | "anei-noudo" | "anei-skin" | "anei-mfg-ban"
        | "dokugeki" | "kakanho-prtr" | "kashinho" | "shobo" | "kouatsu-gas"
        | "taiki" | "suishitsu" | "dojo" | "cwc" | "haiki";
  status: "designated" | "conditional" | "not-designated"; // 明示的な非該当を持てること（偽陰性と未調査を区別）
  classification: string;      // 例 "第二類物質(特別有機溶剤)" "劇物" "毒性ガス"
  basis: { lawId: string; provision: string };  // 例 { lawId:"347CO0000000318", provision:"別表第3第2号11の2" }
  scopeNote?: string;          // 群指定の範囲（例 "粉状の化合物に限る。金属Niは対象外"）
  source: { url: string; retrievedAt: string; sha256?: string }; // 正本スナップショット
  verifiedAt: string;          // 突合日
};
type SubstanceLegalProfile = {
  cas: string;                  // 主キー（群指定は casGroup: string[] ＋代表CAS）
  names: { ja: string; en?: string; aliases: string[] };
  designations: LegalDesignation[];
};
```

設計原則:
- **手書きマッピング禁止**。`OSHA_REGULATION_TAGS_BY_CAS`・mock `categories` は本マスタからの生成物 or 参照に置き換える。
- **「非該当」を明示できる**こと（現在は「タグなし＝未調査 or 非該当」が区別不能で、欠落がそのまま偽陰性になる）。
- 群指定（「〜及びその化合物」）は**CAS展開せず群として持ち、UIで scopeNote を必ず併記**（既に extra-regulations.ts が採っている方針を全ドメインに拡張）。
- 物性型法令（消防法・高圧ガス・廃掃法）は現行の2層UI（要確認）を維持しつつ、**列挙可能な部分**（高圧ガスの毒性ガス/可燃性ガス品名＝一般高圧ガス保安規則第2条、消防法別表第一の品名例示）だけ designated で持つ。

パイプライン（`scripts/etl/build-legal-profiles.mjs`＋`scripts/audit-legal-mapping.mjs` 常設化）:
1. e-Gov XMLスナップショット取得（sha256記録）→ 別表パーサで（区分, 物質名, 号番号）を抽出。
2. 名称→CAS解決は NITE-CHRIP／職場のあんぜんサイトで機械照合し、解決不能は**群指定として保留**（創作禁止）。
3. 生成した designations と、サイトが表示に使う全経路（regulation-tag-labels / mock categories / concentration-limits regulationTags）を**全件突合し、差分をCIで落とす**（`article-caption-integrity.test.ts` の物質版。vitestに「令別表第3スナップショット vs 表示タグ」テストを常設）。
4. AI経路（chemical-ra regulatoryNotes）は、構造化 designations と矛盾する法令名をレスポンスから**除去 or 「AI生成・未検証」ラベル強制**するバリデータを挟む（チャットボットのcitation-validatorと同思想）。
