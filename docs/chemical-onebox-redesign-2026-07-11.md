# 化学物質サービス 一窓化再設計（2026-07-11）

社長要求「入力窓1つに製品名か物質名を入れたらパッと始まる」への根本再設計の設計判断記録。

## 体験の構造（/chemical-ra ＝ 入口）

1. **一窓検索**（UnifiedChemicalSearch）: 物質名・CAS番号・法令名称（溶接ヒューム等の
   CASレス告示名・群指定名）・製品名らしき入力を1つの窓で受ける。
   - 統合DB 3,695物質: クライアント側スリム索引で即時候補
   - DBに無い名称: `/api/chemical/legal-profile` で法令名称解決（cas-law-index の nameKeys）
   - どちらにも無い: **「収載外」を正直に明示**＋次の一歩（SDSのCAS確認→再検索・製品検索・AI調査）
     ＋リスクアセスメント義務/努力義務の一般案内（空白で欺かない）
2. **該当法令の結論カード**（LegalConclusionCard）: designated/not-designated/unverified を
   正本突合APIから取得して断定表示。根拠（号・条項＋e-Gov リンク＋突合日）つき。
   続けて**対策案**＝法定義務（作業主任者・測定・健診・換気等、条文リンク付き）と
   リスク低減の優先順位（代替→工学→管理→保護具）まで一続き。
3. **STEP2 作業状況**（物質確定後に段階表示）→ **STEP3 判定** → A4実施記録（不可侵・従来どおり）。
4. 補助ツール（SDS読取・混合物RA・保存済みRA・職種別チップ）は折りたたみ（details）で下部へ。

## ページの役割分担

- **/chemical-ra** … 入口（1窓）。物質特定→法令結論→RA→対策→A4記録の一続き体験。
- **/chemical-database** … 全物質DBの一覧・フィルタ検索（法令タグ絞り込み・研究用途）。
- **/chemical-database/[cas]** … 物質の恒久詳細ページ（全法令・GHS・濃度基準値。結論カードから遷移）。
- **/chemical-ra/product-search** … 製品名→成分特定の専用導線（一窓の「収載外」カードから誘導）。

URL変更・リダイレクトは行わない（既存リンク・SEOを保全。役割の重複は入口の明示で解消）。

## データ層の拡張（F2/#871 の監査網は維持）

- `cas-law-index.ts` に `nameKeys`（名称解決キー）を追加。CASレス告示名（溶接ヒューム）・
  群指定名（マンガン及びその化合物）を第一級で解決。重複キーは監査テストで禁止。
- `legal-entity-resolver.ts`: CAS/名称 → 法令エンティティ解決（サーバー専用）。
- `GET /api/chemical/legal-profile?q=`: 正本突合プロファイル＋主要義務＋健診＋優先順位を返す。
  受入テスト（route.test.ts）で「溶接ヒューム→特化則2類（別表第3第2号34の2）」
  「カプサイシン→未解決（偽陽性ゼロ）」を固定。
- `legal-duties.ts`: 区分→主要義務（条文リンク付き）。網羅を主張せず一次情報へ誘導。
