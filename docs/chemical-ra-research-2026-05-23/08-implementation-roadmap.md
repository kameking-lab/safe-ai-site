## 推奨実行順序

```
Phase 1（データ整備）
   ↓ ※ ACGIH/JSOH 法的整理を含む
Phase 2（1枚紙RA UI）
   ↓ ※ 用紙ファースト型の意匠固定
Phase 3（詳細判定モード）
   ↓ ※ Phase 2の1枚紙と統合
Phase 4（法令連携）
   ↓ ※ Phase 2/3の出力に法令情報を流し込み
Phase 5（SDS自動取込み）← 別途オーナー判断
```

並列実行は避ける（依存関係が強い + ブランチが分散すると merge conflict 多発）。
