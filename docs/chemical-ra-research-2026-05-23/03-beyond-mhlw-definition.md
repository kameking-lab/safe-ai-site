# 03: 「厚労省超え」の現実的定義

作成日: 2026-05-23
社長要求: 「物質名→基準濃度→わかりやすい解説→1枚紙RA→条件追加で詳細判定、厚労省CREATE-SIMPLE超え」

注意: 「万能」「完全」を捏造的に名乗らない。本資料は実現可能性を踏まえた現実的射程を定義する。

| 軸 | 厚労省CREATE-SIMPLE | 当サイト 目標 | 達成可否 |

| 物質数 | 約3,300 | 15,000-27,000 | ◯ 達成可能（NITE-CHRIP等の統合で） |

## 2. 物質数の目標設定

### 2.1 段階目標

```
現状 8,400 (MHLW統合)
   ↓ Phase 1.1
+ NITE-CHRIP GHS約3,300 = 約 10,500（CAS重複除外後）
   ↓ Phase 1.2

```

### 2.2 「物質数の主張」の出し方

捏造を避けるため、表示は以下のような階層化で:

- 「**規制対象物質: 約12,000（厚労省・NITE・PRTR・化審法・毒劇法を統合）**」
- 「**詳細解説付き: 50物質（社内専門家アノテーション）**」

数字の主張は**必ず内訳と出典明示とセットで**出す。

## 3. 判定軸の拡張

### 3.1 5経路統合判定

CREATE-SIMPLE は吸入中心。当サイトは以下5経路を独立軸で評価:

| 経路 | 入力条件 | 判定根拠 |
|------|---------|---------|
| **吸入** | 換気・取扱量・作業時間・気化性 | 8時間濃度基準値、TLV-TWA、TLV-STEL |
| **経皮** | 接触面積・接触時間・PPE・皮膚障害物質指定 | MHLW皮膚等障害化学物質リスト、JSOH皮膚吸収マーク |
| **経口** | 飲食・喫煙の可能性、手洗い設備 | LD50、ADI、急性毒性区分 |
| **環境影響** | 排出経路、PRTR対象、水生生物毒性 | PRTR制、化審法、GHS 環境分類 |
| **火災爆発** | 引火点、爆発限界、酸化性物質 | GHS 物理化学的危険性、消防法危険物分類 |

各経路を独立に I-IV 判定し、**最大値を総合リスクレベル**とする。1枚紙には総合と内訳の両方を表示。

### 3.2 複合曝露の補正

混合物・複数物質併用時の補正アルゴリズム:

- **加算法（基本）**: Σ (Ci / TLVi) ≤ 1 を遵守判定（ACGIH 推奨方式）
- **相乗作用の警告**: 既知の相乗作用ペア（例: ベンゼン+トルエン）は警告表示
- **拮抗作用は考慮しない**: 保守的設計

オーナー判断必須事項として `09-owner-decisions.md` に明記。

### 3.3 業種テンプレート

建設業特化:

| テンプレート | 想定物質 | 標準入力 |
|------------|---------|---------|
| 塗装 | トルエン、キシレン、酢酸エチル、MEK、エポキシ系硬化剤 | 開放系、刷毛・吹付、屋内/屋外 |
| 防水 | アスファルト、ウレタン、エポキシ、有機溶剤系プライマー | 屋上・地下、施工面積、施工日数 |
| 解体 | アスベスト（既存）、PCB、鉛、結晶質シリカ、ダイオキシン | 解体方式、湿潤化、隔離養生 |
| 地盤改良 | 六価クロム生成のおそれ（セメント系）、薬液注入（水ガラス系） | 改良工法、土壌pH |

各テンプレは「想定物質を選択 → 標準的な換気・取扱量を自動入力 → ユーザーが調整」のフロー。

## 4. 差別化要因の定義

### 4.1 1枚紙RA（用紙ファースト型UI）

- A4縦1枚に必要情報を収める印刷最適化
- KY/日誌(#263)と同じ意匠で統一感
- 印刷時に1ピクセルも崩れない CSS 設計（@media print）
- スマホ画面では「縦長プレビュー → タップで詳細展開」

### 4.2 法令直リンク

「特化則第2類物質」というラベルから:
- → 安衛法 / 特化則 該当条文の本文表示（既存 `/laws` と連携）
- → 厚労省 通達番号への直リンク
- → 関連リーフレットPDFへの直リンク

これは CREATE-SIMPLE には無い差別化要因。

### 4.3 AI による「わかりやすい解説」

物質の難解な化学的特性を中学卒業レベルの語彙で解説:
- Gemini で「この物質はどう危険か」「どう守るか」を3-5行に要約
- 既存 `/api/chemical-ra` の `rawReply` 拡張で実装

社長要求の「わかりやすい解説」を直接満たす。

### 4.4 KY/日誌・教育・保護具との横断連携

- KY用紙 → 当該作業の化学物質RA リンク
- 実務教育(`/education/hoteikyoiku/chemical-ra`) → 教材内の物質をRAへ
- 保護具ファインダー → RA結果のPPE推奨を保護具DBへ
- 化学物質DB → RA への直接遷移（既存）

## 6. 現実的射程の総まとめ

社長判断用、3点に絞った最終結論:

### 6.1 物質数
- Phase 1完了時: **12,000-15,000物質**（厚労省＋NITE＋PRTR＋化審法＋毒劇法 統合）
- 中期目標: 17,000+（建設業特化品目を加える）
- 長期: 27,000+（SDS自動取込み実装後）

### 6.2 判定精度
- 5経路統合（吸入・経皮・経口・環境・火災爆発）の I-IV 判定
- 複合曝露の加算法補正
- 業種テンプレ4種（塗装・防水・解体・地盤改良）
- CREATE-SIMPLE互換モード（行政提出用）

### 6.3 差別化要因
- 1枚紙RA（A4縦・用紙ファースト型・印刷最適）
- 建設業特化（業種テンプレ4種）
- 法令条文への直リンク（条数・通達URL・リーフレット）
- AI解説（中学卒業レベルの語彙で「どう危険か」「どう守るか」）
- 既存機能との横断連携（KY/日誌・教育・保護具ファインダー・化学物質DB）

これらが達成できれば、CREATE-SIMPLE を「現場運用適性」で凌駕すると主張可能。
