## 主要発見3件

### 発見1: 現状実装が想定より高完成度

- `/chemical-ra` は既に CREATE-SIMPLE 準拠の換気×取扱量×時間→I〜IV 判定を実装（`/api/chemical-ra/route.ts:187-258`）
- Gemini API による GHS/PPE/緊急措置の動的生成 + MHLW 公式データのフォールバックの2層構成
- 「1,046物質」は古い情報で、実際は **MHLW統合 8,400+ / 濃度基準値表 1,546 CAS** に到達済
- product-search 経由で SDS→構成成分→自動RA のパイプラインも稼働中（10製品PoC）

### 発見2: NITE-CHRIP 統合で約3.5倍に拡張可能、ライセンスはCC-BY互換

- NITE「化学物質総合情報提供システム（CHRIP）」は GHS 政府分類結果 **約3,300物質** を公開
- 政府著作物のため利用規約上は出典明示で再利用可（NITE FAQ確認済の前提）
- 既存 1,546 CAS と CAS番号でマージすれば **重複除外後で 4,000-5,000 物質規模** に到達
- さらに PRTR制対象 462物質 + 化審法第一種・第二種 + 毒劇法対象 を加えれば **6,000-8,000 物質規模** が現実的射程

### 発見3: CREATE-SIMPLE の弱点 = 建設業現場との乖離

厚労省 CREATE-SIMPLE v3.0 の構造的弱点（実装機会）:

- **物質ベース固定** = 建設現場で頻発する「混合物（シンナー・塗料・接着剤・コーキング剤）」の取扱いが弱い
- **作業ベース不在** = 「塗装」「解体」「防水」「地盤改良」の業種テンプレートが無い
- **PDF出力固定書式** = 現場で書き換えが効かない・1枚化されていない
- **検索動線が薄い** = 物質マスタを別途調べないと使えない（CAS入力前提）
- **法令引用が乏しい** = 安衛則第577条の2・特化則・有機則条文への直リンクなし

→ 当サイトの「1枚紙RA + 業種特化 + 物質DB直結 + 法令逆引き」で全項目クリア可能。

## 「厚労省超え」現実的射程まとめ（3点）

- **物質数**: 現状 8,400 → 目標 **15,000-27,000**（NITE-CHRIP 約3,300 + PRTR 462 + 化審法 + 毒劇法 + 業種特化品目のマージで段階達成）
- **判定精度**: 単一物質の吸入経路のみ → **5経路統合（吸入・経皮・経口・環境・火災爆発）+ 複合曝露補正 + 業種テンプレート**
- **差別化要因**: 1枚紙RAの「用紙ファースト型UI」+ 建設業特化（塗装・防水・解体・地盤改良） + 関連法令条文への直接リンク（条数・通達番号・リーフレットURL） + 既存 KY/日誌・教育・保護具ファインダー との横断連携
