## 2. 方式設計

### 2-1. 位置づけとURL（判断理由）

**新ハブ `/law-navi`（法令ナビ）を新設**し、条文パーマリンクは **`/law-navi/[egovLawId]/[artSlug]`** とする。

- 既存ページとの関係（乱立させない・進化として位置づけ）:
  - `/law-search` = 全文検索**ツール**（インタラクティブ・query string・従来どおり）。法令ナビの検索窓は同じ横断検索エンジンに合流し、条文カードからパーマリンクへ導線を張る。
  - `/laws` = **法改正情報ハブ**（既占有。T6案の `/laws/[id]/[art]` は改正ハブ配下に静的子ページ bcp/glossary… が同居し名前空間が混線するため不採用）。
  - `/law-hierarchy` = 体系図（法→政令→省令）。法令ナビハブから相互リンク。
  - `/search`・⌘K = 横断検索。**分野ページ・別表インデックスを検索アイテムとして追加**し、現場語からの着地面にする。
  - `LawHubNav`（laws/law-search/law-hierarchy/circulars/glossary/chatbot共通サブナビ）に「法令ナビ」タブを追加＝ハブ族の一員として位置づけ。
- URLスキームはT6案Aを踏襲: egovLawId は ASCII安定・e-Gov原文と1:1・法令改称に不変（例 `/law-navi/347M50002000032/151-2` = 安衛則第151条の2）。artSlug 規則: `第61条`→`61`、`第151条の67`→`151-67`（条-枝。項・号はページ内表示であってURLにしない）。**逆写像の一意性はビルド時テストで固定**（幽霊URL 0）。
- 対象: curated条文のうち `LAW_METADATA`（data/law-metadata.ts）に egovLawId を持つ法令のみ（捏造URLゼロ原則）。egovLawId無し法令は従来 /law-search のみ。

### 2-2. 分野インデックスのデータモデル（二層・正本突合）

`web/src/data/law-navi/topics.ts` — `LawNaviTopic`:

```
id / name（例: フォークリフト） / fieldGroup（例: 荷役運搬機械等） / description
aliases: 現場語・俗称・別名（例: リフト, 爪, ツメ, フォーク, パレット…）→ 検索インデックスkeywordsと query-expansion ルールに供給
articles: { lawShort, articleNum, role }[] — 法→令→則の体系順。role=「定義」「資格」「点検」等の現場向けラベル
circularIds: mhlwNotices の id 参照
beppyoIds: 別表インデックス参照
```

- **二層生成**: 候補は `web/scripts/law-navi-topic-scan.mjs`（コーパスkeywords/本文・synonyms・通達タイトルの機械走査）で列挙し、チェックイン済みデータは人手レビュー層（`reviewNote` に採用理由を記録）。手書き丸写しでも自動吐き出しでもなく、**機械候補→人手採録**。
- **正本突合**: vitest 統合テストで (a) articles の全参照が curated コーパスに実在（幽霊参照0）、(b) circularIds が mhlwNotices に実在、(c) beppyoIds が別表インデックスに実在、を機械固定。O18/article-registry と同じ思想。

### 2-3. 俗称ゆらぎ解決層（O5型固定フレーズ過学習の禁止）

3層の既存機構に**辞書エントリを足すだけ**で解く（新エンジンは作らない）:

1. **query-expansion.ts の正規表現ルール**（横断検索・RAG共用）: `/爪|ツメ|つめ/ → フォークリフト・フォーク・車両系荷役運搬機械` 等。正規表現の語中一致なので「爪のやつ」「ツメの機械」「つめが付いたリフト」いずれでも発火＝**言い回し非依存**。固定フレーズ（「爪のやつ」そのもの）は辞書に入れない。
2. **ANDエンジンの部分一致**: 分野ページの SearchItem keywords に短い語幹（爪・ツメ・フォーク…）を持たせる。エンジンが `variant.includes(k)` を許すため「爪のやつ」（1トークン）が keyword「爪」に当たる。
3. **rag/synonyms.ts TERM_EXPANSIONS**: チャットボット側の口語対応も同時に追記（ユンボ・バックホウの先例に倣う）。

### 2-4. 条番号・別表番号の正規化

- 条番号は消化済み（O8-b）。**別表番号を追加**: `article-query.ts` に「別表第三→別表第3」「別表第六の二→別表第6の2」の書換を追加（「別表第」マーカー必須＝日付等の誤変換なし。既存の条書換と同じ流儀・同テストファイルで回帰固定）。

### 2-5. 別表の意味インデックス

`web/src/data/law-navi/beppyo.ts` — `BeppyoEntry`: `{ id, lawShort, label(別表第3), name(特定化学物質), summary, egovUrl, relatedArticles, relatedTopicIds, keywords }`。

- 収載: 安衛令の主要別表（第1 危険物、第2 放射線業務、第3 特定化学物質、第4 鉛業務、第5 四アルキル鉛業務、第6 酸素欠乏危険場所、第6の2 有機溶剤、第7 建設機械、第9 名称等を表示・通知すべき有害物）＋じん肺法・粉じん則等は量産フェーズ。
- 表示面: `/law-navi/beppyo`（1ページ・アンカー深リンク `#<id>`）。横断検索に category='law' で収載（「別表第3」「特化物 別表」「有機溶剤 別表」いずれでも着地）。
- e-GovリンクはLAW_METADATAのegovLawIdから法令トップへ（別表アンカーはe-Gov新UIで安定形が確認できるまで張らない＝幽霊アンカー0の既存方針）。

### 2-6. 条文ページ（原文が主役）

`/law-navi/[lawId]/[artSlug]/page.tsx`（server・generateStaticParams・dynamicParams=false）:

1. **原文全文**（LawArticle.text）を主役に最上部。O18リンカ `linkifyArticleReferences` で条間参照をライブ化し、内部hrefは**生成集合に在る場合のみ**パーマリンクへ書換（`lib/law-navi/permalink-rewrite`、幽霊リンク0）。号マップ（itemNumberMap）があれば号→主題の表を併記。
2. **e-Gov原文リンク**を目立つ位置に（「収録は抄録・原文はe-Gov」の透明性。基条は `#Mp-At_N` アンカー、枝番は法令トップ＝egov-fallback/O18と同方針）。
3. **AI解説ボタン**（client island）: `/api/law-summary` を `mode:"explain"` で呼ぶ（既存Gemini経路再利用・新env不要・CDNキャッシュ・circuit breaker共有・フォールバック内蔵）。出力は原文の**下**に並記し、「原文が正・解説は補助」の免責を生成部に1回表示。根拠条文（当該条の法令名・条番号・e-Gov URL）は応答に自動添付。
4. **用語インライン解説**: glossary 152語のうち条文本文に出現する語を「この条文の用語」として抽出表示（原文テキスト自体は改変しない＝原文不可侵）。
5. **チャット引き継ぎ**: 「この条文についてAIに聞く」→ `/chatbot?q=${lawShort}${articleNum}（${articleTitle}）について教えてください`。既存プリフィル機構のみ使用（チャットボット本体は不変更＝eval非劣化）。
6. **前後条ナビ**: 同一法令のcurated収録順で前後（「収録条文の前後」と明記＝抜けがあることを隠さない）。分野メンバーなら同分野の条文群（同節相当）ナビも表示。
7. パンくず＋JSON-LD（legislation/breadcrumb）・self-canonical・sitemap収載（sitemap-laws.xml 新設、sitemap-index.xml に登録）。

### 2-7. スコープ（本PR＝基盤+最初の1分野）

- 条文ページ: egovLawId保有法令のcurated全条文（route基盤は分野非依存。前後条ナビの破綻防止のため全条文を生成。ビルド規模は /circulars/[id]≈1,158 の実績内）。
- 分野: **フォークリフト（荷役運搬機械等）1分野のみ**。全分野展開は量産タスク（BACKLOG振替）。
- 別表: 安衛令の主要別表のみ。全別表は量産タスク。
- AI解説: オンデマンド生成＋CDNキャッシュ。事前生成対象選定は量産タスク。

### 2-8. 完了条件（実測基準）

「フォークリフト」「爪のやつ」「35条」「三十五条」のいずれからも: /search（⌘K同一）で該当条文付近（分野ページ or 条文）に着地 → 条文ページで原文 → AI解説ボタン → チャット引き継ぎ、が本番で動く。別表第3・別表第6の2が意味（特定化学物質・有機溶剤）で引ける。ゲート: tsc/lint/vitest/build 全緑＋eval:chatbot-gen非劣化＋モバイル390px無読確認。
