# 07. 既存機能との統合体験

対象5機能: KY / 打合せ書(meeting) / サイネージ(signage) / 化学物質RA(chemical-ra) / 事故DB(accidents)

## 7.1 インバウンド動線（各機能 → チャットボット）

| 起点機能 | 状態 | 参照 |
|---------|------|------|
| 化学物質RA | ◎ 強い。`?q=<物質名>の取扱い基準と関連する特化則・有機則の条文を教えて` でディープリンク | `chemical-ra-panel.tsx:894` |
| 事故DB(reports) | ○ あり。「💬 安衛法AIチャットボット」ピル、業種別「💬 AIに条文を質問」（ただし`?q`プレフィルなし） | `accidents-reports/page.tsx:200`, `hub-filter.tsx:298` |
| **KY** | **× なし** | `app/(main)/ky/**`に`/chatbot`参照なし |
| **打合せ書(meeting)** | **× なし** | `components/meeting/**`に参照なし |
| **サイネージ(signage)** | **× なし** | `app/signage/**`に参照なし |
| 共通ナビ | ○ サイドバー/フッター/フラッグシップナビ/コマンドパレット/FAQ約150件 から導線あり | `app-shell.tsx:116` 他 |

**KY・打合せ書・サイネージはチャットボットへの入口が皆無**。現場で最も使うKYからAIに条文相談できないのは統合体験の穴。

## 7.2 アウトバウンド動線（チャットボット → 各機能）

- **全assistantメッセージの固定アクション行**（`chatbot-panel.tsx:1001`）: 事故レポート/戦略プラン/KY(`/ky?q=`)/化学物質RA(`/chemical-ra?name=`)/法令検索(`/laws?q=`)/サイネージ。**ただし`?q`/`?name`値が`answer.slice(0,40/80)`＝AI文の途中切れで品質低い**（doc05）
- **`suggestDigDeeperLinks`**（`chatbot-enrichment.ts:219`）: クエリ/条文駆動で事故DB・通達カテゴリ・業種レポート・法令検索へ（最大4、品質良い）
- **`suggestRelatedLaws`**: 関連法令→`/law-search?q=`

アウトバウンドは概ね機能しているが、**固定ピルのプレフィル品質**と**サイネージへの誘導が文脈なし（静的`/signage`）**が課題。

## 7.3 統一されたAI体験

- Gemini呼び出しは`gemini.ts`＋各機能で共通パターン。免責文も共通（`AI_LEGAL_DISCLAIMER`）。**一貫性は良好**

## 7.4 セッション継続

- localStorage（アクティブ＋保存セッション最大15）＋エクスポート/インポート＋base64共有
- **Supabase等のサーバ永続なし**＝クロスデバイス継続不可。共有はlossy（doc05）。認証必須化禁止のルール下では当面localStorageで妥当

## 7.5 改善（→doc13）

- **P1: KY・打合せ書・サイネージにチャットボット入口を追加**。特にKY→「この作業の法的根拠をAIに聞く」は現場価値大。`?q`プレフィルで作業内容を渡す
- **P1: アウトバウンド固定ピルのプレフィルをユーザ質問文/抽出エンティティに変更**（answer.slice廃止）
- **P2: チャットボット→サイネージに文脈（注意喚起文）を渡す**

## 7.6 結論

化学物質RAとの統合は模範的。一方、**KY/打合せ書/サイネージの片方向（チャット→各機能のみ、逆がない）**と**プレフィル品質**が統合体験の主なギャップ。双方向化は低リスクで現場価値が高くP1。
