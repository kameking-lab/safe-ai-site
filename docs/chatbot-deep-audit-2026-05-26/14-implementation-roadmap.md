# 14. 実装ロードマップ

## 14.1 段階

### Stage 1（本セッションPhase Bで着手）— 信頼性の土台
- **P0-1 化学物質管理者の条番号是正**（法令正確性、最優先）
- 検証: 新規テストで12条の5/12条の6到達固定＋既存866テスト維持

### Stage 2 — 90+到達コア（P1）
- P1-1 stream経路キャッシュ復活
- P1-2 停止＋再試行
- P1-3 同義語拡張（2024化学物質改正）
- P1-4 KY/打合せ書/サイネージ→チャットボット入口
- P1-5 プレフィル品質改善

### Stage 3 — 仕上げ（P2）
- P2-1〜P2-8（辞書単一ソース化・ばく露基準橋渡し・多段対話・監督署案内・レート制限・漢数字・UI整理・通達取り込み）

### Stage 4 — 将来（P3・要社長判断）
- 意味検索 / 多言語 / サーバ永続

## 14.2 各Stageの完了条件（共通）

1. 変更前後で`npm run build`成功
2. `npm run lint` 0 errors / `npx tsc --noEmit` 0 errors
3. `npm run test` 既存866件維持＋新規pass
4. branch→Draft PR→CI緑→Ready→squash merge→main pull
5. 本番デプロイ後に主要動線確認・他5機能の非破壊確認

## 14.3 リスクと停止条件

- **法令正確性**: 条文の追加/修正は必ずe-Gov・厚労省で個別検証。不確実は「未確認」明記し推測値を入れない
- **既存破壊**: PIN・fixture・他機能が参照する条番号変更は影響範囲を点検してから
- **コスト**: 新機能で月¥10,000超見込み・環境変数追加が必要なら即停止し社長判断
- **意味検索/多言語**: コスト・環境変数・基盤を伴うため本Phase非対象、社長判断

## 14.4 想定順序の根拠

P0（信頼性の致命傷）→ P1（90+到達に必須かつ低リスク）→ P2（仕上げ）→ P3（投資判断）。
「最後のメイン機能磨き込み」の達成は**P0完遂で信頼性の致命傷を除去し、P1で操作性・即応性・統合を底上げ**することで現実的に90+へ到達する。
