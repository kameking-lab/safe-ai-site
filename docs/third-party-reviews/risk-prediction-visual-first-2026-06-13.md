# リスク予測(/risk-prediction) ビジュアルファースト化 — 第三者レビュー（柱0バッチ2/9）

日付: 2026-06-13 / 担当: Opus 4.8 無人ループ / ブランチ: feat/risk-prediction-visual-first

## 背景（前任の起票）
柱0スイープ（noread-sweep-2026-06-11.md）で /risk-prediction は「判定機能なのに
結論カードが無い（実測 font 22px・多重h1(2)）」として起票されたバッチ2/9。
判定結果（安全スコア 0〜100 ＋ リスクレベル 高/中/低）が画面の主役になっていなかった。

## ペルソナ（無読）
「朝礼5分前の建設職長・47歳」— 段落は絶対に読まない。色とデカい数字だけ見る。
スマホで工種チップを1タップし、3秒で「今日はどれくらい危ないか／何をやるか」が
分からなければ閉じて紙のKYに戻る。

## 指摘（改善前）
1. 検索後の要約カード（TopSummaryCard）はあったが、(a) スコアが text-3xl（≒30px）で
   主役サイズに届かず、(b) 色が場当たり直書き（bg-rose-50 等）で JIS安全色トークンを
   経由しておらず、(c) 色帯（低→中→高の位置スケール）が無くスコアの大小が直感で読めない。
2. モバイル（390×844）では入力セクション（テキストエリア＋現場名＋チップ8＋例6）が縦に長く、
   結論カードが y=811（ほぼ画面外）に押し下げられ、無読では「結果が出たこと」自体に気づけない。
3. 多重h1(2): 画面の PageHeader h1 と、印刷用 PrintableReport の h1 が同一DOMに併存。
4. 「予測の仕組みについて」3項目の段落が常時展開で、判定とは無関係な説明が場所を占有。

## 改善（実装）
- 純関数モジュール `lib/risk/risk-prediction-visual.ts` を新設（テスト10件）。
  - `computeRiskPredictionConclusion(score)` = 安全スコアとリスクレベルから結論を組み立て。
  - リスクレベル→色トークン: 高=赤(danger) / 中=黄(warning) / 低=緑(safe)。
    solid チップは WCAG AA 準拠（amber/orange/emerald-500+白 の低コントラスト組合せを排除）。
  - `RISK_BAND_SEGMENTS` = 低0〜30 / 中30〜60 / 高60〜100（computeSafetyScore の
    しきい値 30/60 と一致・合計100%）。`riskScoreMarkerPercent` = スコアそのもの（0〜100%）
    にクランプ → デカ数字とマーカー位置が必ず一致。
- TopSummaryCard を再構築: 安全スコアを text-5xl（実測48px）のデカ数字に。リスクレベルの
  チップ＋色帯（3セグメント＋▼マーカー）＋トーン既定アイコン。「次にやること：◯◯」を
  カード内に明示。死亡・重傷件数は StatusBadge（danger）で。SafetyScorePanel のゲージ色も
  同じトークンに揃え、色の文法を一致。
- 結論カードを入力セクションの上（最上部）に移動し、検索完了時に conclusionRef へ
  smooth scroll（チップは画面中ほどにあるため）→ 判定後 y=16 で確実にファーストビュー。
- 多重h1是正: PrintableReport の h1 を非見出し要素(p)へ降格（正式書式の見た目=太字大は保持）。
- 「予測の仕組みについて」を CollapsibleDetail（初期折りたたみ）へ格納。本文は1文字も変えず、
  照合ロジック・データソース・精度の限界の3項目をそのまま詳細層へ（正確性は不可侵）。

## 無読テスト（再評価）
prod server（npm run start・390×844・serviceWorkers:block）で実機操作。
スクリプト: docs/third-party-reviews/scripts/risk-prediction-noread-2026-06-13.mjs（恒久保存）。

- ⑤ h1は画面に1個だけ — PASS（h1=1）
- ⑥ 「予測の仕組み」は初期折りたたみ — PASS（open=false）
- ① 判定後 結論カードがファーストビュー上部（y+100<844）— PASS（y=16）
- ② 安全スコアがデカ数字（≥40px）— PASS（48px "65/100"）
- ② スコアは0-100の数値 — PASS
- ③ リスクレベル「高リスク」がカード色(rose)と一致 — PASS
- ③ 低/中/高の色帯3セグメント＋▼マーカー — PASS（segs=3）
- ④ 「次にやること」が結論カード内に明示 — PASS

**8/8 PASS**。「高所」チップ1タップで 65/100 高リスク（赤）・▼が高ゾーン・
「次にやること：作業前に対策を徹底」が3秒で読める。

## 不可侵の確認
- 印刷帳票（PrintableReport）: 表題はp化したが見た目（太字大・区切り線）は不変。色帯・GHS等の
  装飾は print:hidden で印刷に出さない。判定ロジック（computeSafetyScore）は不変＝表示のみ追加。
- 法令・データの正確性: 説明文は消さず CollapsibleDetail へ格納のみ。数値・分類は不変。

## 申し送り
- 検索入力セクション内の h2「AIリスク予測」が PageHeader タイトルと重複（実害は小）。
  文字ダイエットの観点では将来まとめてよいが、本イテレーションは結論カード優先で見送り。
- リスクマトリクス/傾向分析タブは数値ダッシュボードとして既に機能。柱2（ダッシュボード強化）で
  デカ数字サマリー化の余地あり（別タスク）。
