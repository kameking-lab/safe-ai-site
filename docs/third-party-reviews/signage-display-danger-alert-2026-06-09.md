# 第三者レビュー: フルスクリーン無人キオスク(/signage/display)への高リスク警報 全画面赤アラート結線

実施日: 2026-06-09 / 対象: `/signage/display`(SignageMapClient) / 軸2(既存機能を第三者目線で使いやすく)

## 背景・タスク
BACKLOG「フルスクリーン無人キオスク /signage/display に高リスク警報の全画面赤アラート(自動発動)を展開＝
警報データ(byIso)形状に合わせて結線。無人運用こそ自動発動が要る。1画面フィット維持・useEffect自動発動を実機検証」。

先行PR#446 で `/signage`(ダッシュボード)には SignageDangerAlert を結線済み。本件はそれを
**無人で1日中流しっぱなしにするフルスクリーンキオスク** `/signage/display` に展開する。キオスクは
別データソース(`/api/signage/jma`)を使い、警報は `bundle.warnings.byIso`(都道府県→`{level, entries[]}`、
各 entry は `headline` と `warnings[]{code,status,level}`)という richer な形状で持つ。ダッシュボードの
`selectedWarnings`(`{code,status}[]`)とは形が違うため、そのまま渡せない。

## ペルソナ
**中村さん(52歳・建設現場の安全衛生責任者)**
- 現場事務所にタブレットを据え置き、朝ONにして1日中つけっぱなし。**普段は地図だけ見たいのでサイドパネルは閉じる**。
- 最も怖いのは大雨・暴風警報を作業員が見落とすこと。掲示物より「画面が勝手に知らせてくれる」ことを求める。
- ITは苦手で、毎回手操作が要るなら使わない。**設定が黙って無効化されるのが一番許せない**(安全機能のはずが無音で死ぬ)。
- 一方で、注意報レベルでいちいち全画面警報が出ると「またか」となり、本当の警報を信じなくなる(オオカミ少年)。

## 指摘(改善前に想定した致命/懸念)
Playwright(Chromium, 1920×1080)でキオスクを実操作し、モックで警報レベルの大雨・暴風警報を注入して検証。

1. **【致命】サイドパネルを閉じると安全機能が無音で死ぬ恐れ**: キオスクの危険アラート部品を素直に
   サイドパネル内に置くと、中村さんの常用フロー(パネルを閉じて地図だけ表示)で部品ごとアンマウントされ、
   自動発動の useEffect が止まる。「監視中のはずが、パネルを閉じた瞬間から警報が一切出ない」=
   BACKLOG が最も警戒する「安全機能の無音化」に直結。
2. **【致命】注意報での誤発動**: SignageDangerAlert は headline / warnings[].status を「警報」等の
   キーワードで照合して発動する。byIso には注意報(advisory)も混在するため、素のまま全件渡すと
   「大雨注意報」の "大雨" に反応して全画面警報が誤発動 → オオカミ少年化。
3. **【懸念】全画面オーバーレイが地図UIに負ける**: キオスクは Leaflet のズームコントロール(z≈1000)・
   サイドパネル(z-[450])を持つ。赤オーバーレイが地図コントロールの背面に回ると「全画面で覆って警告」に
   ならず、ズーム+/−ボタンが赤画面を突き破って見える。
4. **【懸念】1画面フィット崩壊**: バーを通常フローに足すと地図の縦が削れ、無人サイネージの1画面フィットが崩れる。
5. **【懸念】再読込で自動発動がOFFに戻る**: 無人運用では夜間の再起動・タブ復帰で再読込が起きる。設定が
   揮発すると、誰も気づかないまま監視が切れる。

## 改善(実装)
- **常時マウント**: `<SignageDangerAlert>` をサイドパネル内ではなく**ルート直下に常時マウント**
  (`signage-map-client.tsx` のルート `<div>` 直下、`absolute left-14 top-2`)。パネル開閉に一切依存せず
  自動発動の useEffect が走り続ける。→ 指摘1を解消。
- **高リスクのみ抽出(純関数)**: `web/src/lib/signage/danger-alert-source.ts` に `deriveDangerAlertInput(byIso)`
  を新設(テスト7件)。**warning(警報)・special(特別警報)レベルだけ**を `{code,status}` に整形し、
  注意報(advisory)は除外。headline は最も深刻なレベルの発表見出しを1件採用。同一 `code|status` は重複排除。
  → 指摘2を解消(注意報のみでは自動発動しないことを実機確認)。
- **z-index是正**: バー/オーバーレイのラッパーを `z-[1100]`(Leaflet コントロール z≈1000・パネル z-[450] より前面)に。
  transform を使わないラッパーにすることで、内部の `fixed inset-0` オーバーレイがビューポート全面を覆う。
  → 指摘3を解消(ズーム+/−が赤画面に覆われる)。
- **1画面フィット**: バーは `absolute`(left-14 でズームコントロールを回避)。通常フローを占有せず、
  オーバーレイは `fixed`。地図の縦は不変。→ 指摘4を解消。
- **設定の永続**: SignageDangerAlert 既存の `localStorage("signage-danger-autospeak")` をそのまま活用。
  → 指摘5は元々解消済みで、キオスクでも実機で再読込後の維持を確認。

部品本体(SignageDangerAlert)のロジックは不変。データ/スキーマ/取得ロジック不変・新規依存0。
