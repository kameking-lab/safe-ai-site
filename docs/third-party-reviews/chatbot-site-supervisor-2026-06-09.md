# 法令チャットボット(/chatbot) 第三者レビュー — 「条文の根拠をすぐ知りたい現場監督」

実施日: 2026-06-09 / レビュー方式: 厳しいペルソナ生成 → なりきってPlaywright実操作(iPhone 13エミュレート, dev server localhost:3000) → 欠点列挙 → 改善 → 再評価。

## ペルソナ

**佐藤（48歳・建設現場の現場監督／職長歴20年）**
- スマホ（型落ちAndroid/iPhone）を屋外・片手で使う。電波もバッテリーも余裕がない。
- 「足場の手すりは何センチだ？」と聞かれたとき、**条文番号と根拠を3秒で出して、その場で職人に示したい**。
- 我慢の限界: 「①返答が遅い ②どの条文か分からない、ふわっとした一般論しか出ない ③スクロールが重い・カクつく」のどれかがあれば二度と開かない。ChatGPTでいいやとなる。
- 信頼の核: 「安衛則第◯条」という**条番号と原文への導線**。これが無いAIは現場では使えないと断じる。

## 実操作と指摘（忖度なし）

iPhone 13エミュレートで `/chatbot` を開き、「足場の手すり高さは？」を入力→送信して計測。

### 🔴 致命: 画面が常時カクつく（無限再レンダリング）
- **送信すらしていない初期ロードだけで、コンソールに `Maximum update depth exceeded`（setStateの無限ループ）が64回発生**。
- 原因: `ChatbotBody` の「チャットを訪問済みにする」副作用が `useEffect(..., [copilot])` と書かれていた。Copilotの状態更新は毎回 `updatedAt: Date.now()` を打つため Context値の同一性が毎レンダリング変わり、effect→state更新→effect…の無限ループに突入していた。
- 体感影響: レンダリングが回り続け、`localStorage` 書き込みも回り続ける。**型落ちスマホでは入力中のもたつき・発熱・電池消費**として効く。佐藤が最も嫌う「重い・カクつく」の直接原因。
- 同じ書き方が `CopilotIndustrySync`（事故レポート/計画ジェネレーターの業種引き継ぎ）にもあり、そちら経由でも同症状が出る状態だった。

### 🟡 入力欄がモバイルで初期ビューより下（textareaTop≈1628px / viewport 664px）
- ただし空状態には上部に例題ボタン（タップで即送信）があり、「すぐ質問」する動線自体は確保されている。今回はインパクト最大の致命を優先。

### 🟢 出典の見やすさ自体は良好（致命解消後に再確認）
- 回答直下に **「📚 法令データベースから検索」バッジ → 参照条文(≤2件は自動展開) → 「📎 出典（条文番号＋施行日＋発出機関）」(常時展開)** が並ぶ。
- 「足場の手すり高さは？」で安衛則の該当条文原文＋e-Gov原文リンクが提示され、佐藤の「条番号と原文導線」要求を満たしていた。

## 改善（このPRで実施）

無限再レンダリングを根治。`copilot` オブジェクト全体ではなく、**安定な `useCallback` コールバック（`recordVisit` 等）とプリミティブ値のみ**を effect 依存に渡すよう変更。

- `web/src/app/(main)/chatbot/ChatbotBody.tsx`
  - `const recordVisit = copilot?.recordVisit;` を取り出し `useEffect(() => recordVisit?.("chatbot"), [recordVisit])` に。`recordVisit` は `useCallback([])` で安定なので**マウント時1回だけ**実行。
- `web/src/components/copilot/CopilotIndustrySync.tsx`
  - 同様に `setIndustry / recordVisit / addConcerns` を取り出し依存に。配列propの `concerns` は毎回別参照になり得るため、制御文字(\u0001)区切りで安定した文字列キー `concernsKey` に畳んで依存に渡す。

## 再評価（改善後）

同スクリプトで再計測:
- **`Maximum update depth` エラー: 64回 → 0回**（初期ロード・放置時）。
- 送信→ユーザー吹き出し描画 52ms、本文描画開始 232ms、`/api/chatbot/stream` 200。回答・参照条文・出典カード・信頼度バッジすべて正常描画。
- 全ゲート: `tsc --noEmit`=0, `lint`=0 errors, `vitest`=1092 pass, `build`=成功。

佐藤の評価: 「カクつきが消えた。聞けばすぐ条番号と原文が出る。これなら職人の前で開ける」。
残課題（次イテレーション候補・本PR対象外）: モバイルで入力欄を初期ビューに近づける／回答後に出典セクションへ即ジャンプする小導線。
