# 第三者レビュー: KY用紙 参加者選択「2タップ完了」化（手袋の職長目線）

対象: `/ky/paper` の参加者（作業員）選択
日付: 2026-06-09
手法: Playwright 実機操作（iPhone 12・390×844）。作業員マスターに14名（自社5・協力会社1次4・協力会社2次5、うち常用6名）を投入し、毎朝の選択フローを実測。

## ペルソナ
**山口さん・54歳・建設現場の職長**。20年以上紙のKY用紙でやってきた。スマホは持っているが薄手の作業手袋をしたまま片手で操作する。班は毎朝ほぼ同じ顔ぶれ（自社3名＋協力会社の常連3名）。
- 我慢の限界: 「毎朝おなじ6人を1人ずつポチポチ選ばせるなら、紙に判子押すほうが速い」。手袋だと隣のボタンを誤タップする。
- 期待: 「いつもの班」をパッと一発で呼び出したい。今日は人が違う日だけ足し引きしたい。

## 指摘（Before・忖度なし）
1. **【致命】常用6名を選ぶのに6タップ必要**（実測）。作業員はフラットなチップの羅列で、班・常用といったまとまりが一切ない。毎朝6回タップは手袋では誤タップ多発＋単純に面倒。「いつもの班ボタン」が存在しない（一括/全員/常用/班 系ボタン= 0個を実測）。
2. **14名がベタ並びで自分の班員を探しづらい**。自社・協力会社1次・2次が混在し、協力会社の人を探すのに目が泳ぐ（常用先頭ソートはあるが所属の境目が見えない）。
3. **全部外す手段がない**。一度選びすぎても1人ずつ外すしかない。
4. 良かった点: チップは104×44pxで既にタップサイズは合格。資格Noの引き継ぎも機能していた。

→ ボトルネックは「タップサイズ」ではなく「毎朝の繰り返し選択そのもの」。

## 改善
純粋関数 `src/lib/ky/participant-select.ts` を新設（`addParticipants` / `clearParticipants` / `groupWorkersByAffiliation`・9テスト）し、参加者セクションへワンタップ呼び出しのツールバーを追加。
- **⭐ 常用N名をまとめて選ぶ** … `isRegular`（＝毎日来る）作業員を1タップで全員追加。毎朝の「いつもの班」がこれ1発（6タップ→1タップ）。
- **{所属}全員** … 協力会社など所属区分が2つ以上あるとき、班（自社/協力会社1次…）ごとに全員を1タップ追加。「今日は山田工業だけ」も一発。
- **クリア** … マスター由来の選択を一括解除（手入力した氏名は温存）。
- 個別チップは**所属ごとにグルーピング**し小見出しを付けて、自分の班員を見つけやすく。
- すべて既存の `toggleWorker` 作法（空き行再利用・氏名キー）を踏襲。データモデル/localStorageスキーマ変更なし・新規依存なし・印刷体裁不変。再押下しても重複追加されない（純粋関数側で既選択を除外）。

## 再レビュー（After・Playwright実機）
- ⭐常用ボタン（201×44px）を**1タップ → 参加者（6名）**（Before 6タップ→After 1タップを実測）。
- クリア1タップ → 参加者（0名）。
- 「〜全員」班ボタンは3つ（自社・協力会社1次・2次）。協力会社（1次）全員を1タップ → 参加者（4名）。
- 二重押下チェック: 常用を続けて2回押しても 6名→6名 で重複なし。
- 山口さん「朝はこの⭐を押すだけ。手袋でも一発だから、これなら紙より速い」＝採用ライン到達。
