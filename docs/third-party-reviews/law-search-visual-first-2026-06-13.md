# 第三者レビュー: 法令検索系の脱テキスト・ビジュアルファースト化（柱0バッチ4/9）

- 日付: 2026-06-13
- 対象: `/law-search`（法令全文検索）・`/circulars`（厚労省通達・告示・指針一覧）・`/law-hierarchy`（法令階層マップ）
- ブランチ: `feat/law-search-visual-first`
- 無読テストスクリプト: `docs/third-party-reviews/scripts/law-search-visual-first-2026-06-13.mjs`（恒久保存）

## ペルソナ
「条文・通達の所在を3秒で知りたい安全担当（45歳・スマホ）」。段落は読まない。色とデカい数字・アイコンだけ見て「いま何件ヒットしているか」「どこを押せばよいか」を判断する。分からなければ閉じて e-Gov を直接開く。

## 改修前の指摘（忖度なし）
1. **/circulars が異常に縦長**: 旧実装は絞り込み後の上位 **200件を一括描画**（約39,461px）。ファーストビューに「該当何件か」のデカい手掛かりがなく、スクロール負荷が大きい。柱C-6の指摘どおり。
2. **絵文字アイコンの粒度バラつき**: `/circulars` の種別チップと `/law-hierarchy` の通達/告示/指針カードが絵文字（📄🏛📘）。OS/フォント依存でレンダリングが不安定、サイズも小さく無読での識別性が低い。
3. **出典注記が常時露出**: `/law-search` の「出典の見分け方」amber凡例、`/law-hierarchy` の使い方段落・末尾の注意書きが常時展開。現場ペルソナには情報過多で、肝心の「いまの状態」が埋もれる。
4. **結論カード不在**: 3画面とも「いまの状態（該当件数・収録規模）」を示すデカ視覚要素がなかった。

## 改修内容
### /circulars
- **初期描画24件＋「さらに表示（残りN件）」で24件ずつ伸長**（`INITIAL_RENDER`/`PAGE_STEP`）。旧 `MAX_RENDER=200` の一括描画を廃止。絞り込み条件変更時は描画中に前回条件と比較して表示件数を初期値へ戻す（React 公式の「描画中の state 調整」パターン＝effect 内 setState を避け `react-hooks/set-state-in-effect` 違反も回避）。
- 種別チップの絵文字 → **lucide アイコン**（すべて=ListFilter／通達=FileText／告示=Landmark／指針=BookOpen）＋タップ対象44px化。
- 最上部に**結論カード**新設＝該当件数を48pxデカ数字（info青／0件は warning黄「該当なし」＋緩和案内）。
- 判例セクションの説明段落を `CollapsibleDetail` へ格納（内容不変）。

### /law-search
- 最上部に**結論カード**（ヒット件数デカ数字・0件は warning「該当なし」）。検索語に応じてリアルタイム更新。
- 「出典の見分け方」amber凡例（現行版/施行当時版の区別）を `CollapsibleDetail` へ格納。**法令情報の正確性は不可侵のため内容は1文字も変えず、初期折りたたみにしただけ**。

### /law-hierarchy
- 通達/告示/指針カードの絵文字 → **lucide アイコン**（FileText/Landmark/BookOpen）。
- 最上部に**結論カード**＝収録法令数をデカ数字（info青）＋関連通達等の総件数を補足。
- 長文の使い方段落と末尾の出典注記（e-Gov・厚労省一次資料での確認喚起）を `CollapsibleDetail` へ格納（内容不変）。

### 不可侵の維持
- 法令・通達の本文／件数／出典は一切改変せず、露出のみ詳細層へ移動。
- 多重 h1 は3画面とも元から無し（無読テストで h1=1 を確認）。

## 無読テスト結果（再評価）
prod server（`npm run start`）・390×844・`serviceWorkers:"block"` で実機操作。**16/16 PASS**。
- /circulars: h1=1／結論カード y=600（ファーストビュー内）・デカ数字48px／種別チップに svg アイコン／初期描画24件／「さらに表示」押下で 24→48件／判例説明 折りたたみ。
- /law-search: h1=1／結論カード デカ数字48px／出典凡例 折りたたみ。
- /law-hierarchy: h1=1／結論カード デカ数字48px／絵文字📄🏛📘 消滅（アイコン化）／出典注記・使い方段落 折りたたみ。

ペルソナ再評価: 「該当件数がデカく出るので絞り込みの効きが一目で分かる。通達一覧がもう延々スクロールしない。これなら紙より速い」。

## 申し送り
- /circulars の結論カードが y=600（LawHubNav＋ヘッダーの下）。さらに上げるならヘッダーの収録件数行を結論カードへ一本化する余地あり（今回は CircularsHeader を別コンポーネントのまま温存）。
- /law-search の法令名チップ（24法令）は固有アイコンを割り当てると恣意的になるため絵文字化・アイコン化は見送り（チップは色とラベルで足りる）。
- ゲート: tsc0 / lint errors0（既存warning45） / vitest 1609 pass / build 成功。
