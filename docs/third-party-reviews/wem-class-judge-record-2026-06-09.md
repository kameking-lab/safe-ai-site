# 第三者レビュー: 管理区分 判定ツール（作業環境測定）— 評価記録の出力導線

- 対象: `/work-environment-measurement/management-class-judge`（`WorkEnvManagementClassJudge`）
- 日付: 2026-06-09
- 方式: 厳しい第三者ペルソナになりきり Playwright で実操作 → 欠点列挙 → 改善 → 再評価

## ペルソナ

**大野さん（48歳・化学系製造業の衛生管理者／第一種衛生管理者）**
- 有機溶剤（塗装ブース）・特定化学物質を扱う工場で、年2回の作業環境測定を外部の作業環境測定機関に委託。
- 測定機関から「A測定・B測定の値と管理濃度」が記載された報告書を受け取り、**自分で管理区分を確認し、評価結果を社内記録として残し、第3管理区分なら作業者へ周知する**のが仕事。
- 我慢の限界:「結果を画面で見られても、それを**記録に残せない**なら使う意味がない。作業環境測定の評価は3年（特別管理物質は30年）保存義務があるし、第3区分は掲示・周知が要る。電卓でしかないなら、結局Excelに転記するだけ。」

## 指摘（改善前・Playwright実機 1280px / 390px）

1. **【致命】判定結果に保存・印刷・記録化の導線が一切ない。** 結果カード内のボタン数=0（実測）。第1〜第3管理区分と改善措置はきれいに出るが、**スクリーンショットか手入力で転記するしかない**。作業環境測定の評価結果は安衛則に基づく記録保存義務（特別管理物質は30年）・第3区分は労働者周知義務があるのに、ツールはそれを残す手段を提供していない＝「電卓」止まりで実務の記録工程に乗らない。
2. **記録に必要な識別情報（単位作業場所名・対象物質名・測定実施日）を入力する場所がない。** 仮に印刷できても「どの作業場の・何の物質を・いつ測ったか」が無ければ記録として成立しない。
3. 判定の根拠（入力した管理濃度・A測定値・GSD・B測定値）が結果に残らないため、後から「なぜこの区分になったか」を記録から追えない。

（良かった点: 第1〜第3の色分け・A/B内訳バッジ・区分別の義務的改善措置・出典明記・横スクロール0px は良好。判定エンジン自体は upper confidence limit まで実装済みで精度面の不満なし。）

## 改善（実装）

死蔵していた「記録化」をゼロから足すのではなく、**結果を作業環境測定の評価記録様式（A4）として印刷／PDF保存できるようにした**（既存の化学物質RA `ChemicalRaReportHeader` と同じ印刷専用パーツのイディオムを踏襲）。

- **記録用の任意入力**（`<details>` で折りたたみ・判定フローは不変）: 単位作業場所の名称／測定対象物質名／測定実施年月日。空欄でも判定・印刷可（印刷時は手書き欄として出力）。
- **「🖨 評価記録を印刷 / PDF保存」ボタン**を結果カードに追加（`window.print()`）。
- **印刷専用パーツ**（`hidden print:block`・画面は不変）:
  - `ClassJudgeRecordHeader` = 評価記録タイトル＋評価実施日＋事業場名/所属/評価実施者氏名（手書き欄）＋単位作業場所/対象物質/測定実施年月日（入力値を反映）。
  - `ClassJudgeInputTable` = 管理濃度・A測定値・GSD・B測定値の内訳（判定の根拠を記録に残す）。
  - `ClassJudgeSignoff` = 確認欄（評価実施者／衛生管理者／統括安全衛生責任者）＋保存・周知の注記（特別管理物質30年保存／第3区分は労働者周知）。
- 印刷時は入力フォームと「判定基準」解説パネルを `print:hidden` で隠し、結果カードのみを白背景の記録様式として出力。
- 純関数 `formatMeasuredOn`（"YYYY-MM-DD"→「YYYY年M月D日」・不正値は空）を切り出し +4テスト。
- 判定エンジン・データ・スキーマは不変。新規依存0。管理濃度等の告示数値は引き続きユーザー入力（捏造なし）。
