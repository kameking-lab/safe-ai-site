# 第三者レビュー: 新規入場者 受入教育 記録（/site-records/induction）

対象: `/site-records/induction`（受入教育 実施記録ツール）
日付: 2026-06-09
方式: 厳しい第三者ペルソナ → なりきり Playwright 実操作（スマホ 390×844）→ 摩擦の洗い出し → 改善 → 再レビュー

## ペルソナ
**田村さん（34歳・中小ゼネコン現場監督・先月から安全兼務・実務経験ゼロ）**
- 前サイクル(/site-records 初見導線)で生成したペルソナの続き。記録キットHubの「はじめての方へ」スタートガイドの **ステップ2「新規入場者 受入教育 記録」** をタップして到着。
- 今日、同じ現場（田村ビル新築工事）に **3人の新規入場者** が入る。1回の受入教育（オリエンテーション）でまとめて教えた後、1人ずつ記録を残す必要がある。
- 我慢の限界: 「同じことを何度も入力させられたら紙の名簿に戻る」。

## 実操作で見つかった摩擦（忖度なし）
Playwright で記録ゼロ状態から実操作し、1人目を保存→「新規」で2人目に進む動線を再現。

1. **【致命・再入力地獄】1人保存して「新規」を押すと、現場名・教育実施者・実施日が全部消える。**
   実測: `新規` 押下後 → `現場名=""` `教育実施者=""`（実施日のみ今日に再設定）。
   - 同じ現場・同じ教育実施者・同じ教育内容なのに、入場者ごとに「現場名」「教育実施者（自分の名前）」を毎回打ち直し。3人なら3回、10人なら10回。
   - これは KY用紙・安全工程打合せ書で既に解消した「再入力埋没」と同型の欠点。受入教育は **1セッション=複数名** が現場の標準フローなので、影響はむしろ大きい。
2. **【大・再チェック地獄】教育項目13個のチェックも「新規」で全部クリアされる。**
   - 1回の受入教育で全員に同じ13項目を教えているのに、入場者ごとに「全て実施」を押し直し（または13個を再確認）。教えた内容は同一なのに記録の手間だけ人数倍。
3. （軽微）「保存」と「新規」しかなく、「この人は終わり、同じ現場で次の人」という最頻の操作に対応する1タップが無い。

→ 1人あたり最低でも 現場名+実施者+13チェック を再現する必要があり、人数分これを繰り返す。田村さん「これなら紙の名簿の方が速い」。

## 改善（インパクト最大の1点に集中）
最頻フロー「1回の受入教育で入場した複数名を記録」を1タップで回せるようにする。

- 操作行に **「👤 保存して同じ現場で次の人へ」** ボタンを新設（`handleSaveAndNextWorker`）。
  - 現在の入場者を保存 → 新しいIDを発番 → **現場名・教育実施者・実施日・教育項目（チェック状態）を引き継ぎ**、氏名・所属・職種・備考・本人/実施者確認だけクリア。
  - 氏名未入力なら保存せず警告（既存「保存」と同じガード）。
  - 文言「保存しました。同じ現場・実施者で次の入場者を記録できます。」で次の操作を明示。
- 既存の「保存」「新規（全項目クリア）」「印刷」「名簿CSV」はそのまま温存＝現場を変える時・1人だけの時の動線は不変。新規部品は作らず既存 store の `saveInduction` を再利用（作り直し0）。

## 再レビュー（Playwright 実機・スマホ）
1人目（入場一郎／一郎建設）を入力→「全て実施」→**「保存して同じ現場で次の人へ」**を押下:
- 実測: `現場名="田村ビル新築工事"` `教育実施者="職長 田村"` `教育項目=13/13 実施` が **引き継がれ**、`氏名=""` `所属=""` がクリア。
- 続けて2人目（入場二郎）を氏名・所属だけ入力して「保存」→ 保存一覧が **2件** に。
- 田村さん「現場名と自分の名前、チェックがそのまま乗る。あとは名前だけ。これなら全員ぶん記録できる」=採用ライン到達。

入力量: 1人あたり「現場名+実施者+13チェック+氏名+所属+職種」→「氏名+所属（+職種）」へ。人数が増えるほど効果が線形に効く。
