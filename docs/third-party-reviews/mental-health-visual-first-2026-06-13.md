# 第三者レビュー：メンタル・両立系のビジュアルファースト化（柱0バッチ6/9）

- 日付: 2026-06-13
- 対象: `/mental-health`・`/mental-health-management`・`/treatment-work-balance`（ハブ3枚）と `/treatment-work-balance/plan-builder`（プラン作成ツール）
- 社長指示「脱テキスト・ビジュアルファースト」（2026-06-10）に基づく文字ダイエット＋結論ファースト

## ペルソナ（標準ペルソナ群より）
「段落を絶対に読まず、色とデカい要素しか見ない現場/担当」。スマホ390×844で開き、3秒見て
「いまの状態」と「次にやること」が言えなければ不合格。

## 指摘（改善前）
1. **ハブ3枚**: 冒頭の「位置付け」ボックスが2〜3文の法令説明の長文段落。現場/担当は読まずに飛ばす。
   末尾の免責も同様に長文段落で、画面を占有するだけで「次にやること」を示さない。
2. **plan-builder**: ファーストビューに状態を示すデカい視覚要素がなく、いきなり選択フォーム。
   「これは何のツールで、いま自分はどの段階か」が3秒で分からない。生成後もプランが画面下に
   出るだけで「完了した」というデカい合図がない。生成ボタンが `px-4 py-2`（高さ約36px）で
   タップ対象44px未満、かつ他ボタンと同格で主役感がない。

## 改善
1. **ハブ3枚**: 位置付けの長文を「短い結論1行（色付きボックス＝現場が色で読む）＋詳細は
   `CollapsibleDetail` 折りたたみ」に分離。免責も折りたたみへ格納。**法令・免責の文言は1字も
   削らず詳細層へ移動**（正確性は不可侵＝隠すのは可・消すのは不可）。
2. **plan-builder**:
   - 最上部に `ConclusionCard`（共通視覚言語）を常設。未生成＝**青（info＝指示・案内の色。
     責めない）**「プラン未作成」＋「入力する」アクション。生成後＝**緑（safe）**「プラン作成完了」
     ＋病態名を反映した1行＋「プランを見る」アクション。判定は純関数 `planBuilderConclusion()`
     に一本化（テスト3件）。
   - 生成ボタンを `min-h-[48px]`・`text-base font-bold`・アイコン付き・モバイルで主役幅へ拡大
     （デカボタン）。
   - ページ冒頭の「位置付け」長文・末尾の出典注記も折りたたみへ（結論カードをファースト
     ビュー上部へ引き上げる効果＝card の y 位置 621→502px）。

## 無読テスト（再評価）
スクリプト: `docs/third-party-reviews/scripts/mental-health-visual-first-2026-06-13.mjs`
（本番ビルド `npm run build` → `npm run start` の localhost:3000 に対し実行。
serviceWorkers:"block"・viewport 390×844）

結果: **16/16 PASS**

  生成ボタン高さ48px / 生成後=緑(emerald)・「作成完了」表示・`#plan-output` 出現

## 不可侵ルールの遵守
- A4印刷帳票・サイネージは非対象（変更なし）。
- 法令・免責の正確な文言は削除・曖昧化せず、`CollapsibleDetail` 詳細層へ全文格納。
- 既存の医学的免責（plan-builder 出力上部の amber 警告・印刷フッター）は不変。

## 申し送り（残）
- ハブの InfoCard（ストレスチェック/4つのケア/カスハラ/VDT・病類別など）の本文も長文だが、
  カードはタイトル＋アイコンで一覧スキャンが効くため今回は対象外。必要なら次巡回で
  カード単位の折りたたみを検討。
