# 第三者レビュー: iOS「ホーム画面に追加」案内（柱1是正・2026-06-10）

## 背景（前任の判断 → Fableの発見）

- 前任(Opus 4.8)の採用レビューは「毎朝の習慣化は営業領域＝コードで打てる手はもう無い」と結論。
- Fable再検証（docs/fable-reexamination-2026-06-10/adoption-structural-limits-2026-06-10.md）で、
  `install-pwa-prompt.tsx` が `beforeinstallprompt` 専依存であることを発見。
  このイベントは iOS Safari では**永久に発火しない**ため、一人親方の主端末である iPhone には
  インストール促しが一度も表示されていなかった。習慣化の中間レバーが1本、丸ごと死んでいた。

## ペルソナ

- **佐藤健一（52）**: 鳶の一人親方。iPhone 14 / Safari。ITは苦手で、アプリのインストールは娘に頼む。
- 前提: KY用紙を2回使った常連（利用スコア10）。サイトには毎回 Google 検索で到達している。
- 我慢の限界: 説明が4行を超えたら読まない。専門用語（PWA・インストール等）が出たら閉じる。
  同じ案内が毎回出たらサイトごと使わなくなる。

## 改善前の指摘（忖度なし）

1. **iPhoneでは何度使っても案内ゼロ**: Android では利用スコア9以上で「追加する」バナーが出るのに、
   iPhone では存在自体が無い。「ホーム画面に追加」という機能を佐藤さんは知らないので、
   自力で発見する見込みもゼロ。紙のメモ帳がブックマーク代わり。
2. 仮に案内を出すとしても、iOS は `prompt()` が使えないため「追加する」ボタンは置けない。
   手順説明が長文になれば佐藤さんは読まない（4行の壁）。
3. 毎回出たら確実に嫌われる。既存のクールダウン（14日）と利用スコア条件を必ず引き継ぐこと。

## 改善内容

- `web/src/lib/pwa-install.ts` 新設（純関数・テスト14件）:
  - `isIosDevice`: UA判定＋iPadOS 13+ デスクトップ表示（MacIntel＋タッチあり）対応。
  - `detectInstallGuideKind`: iOS Safari → 3ステップ直案内 / iOS の Chrome・LINE 等 → Safari誘導込み案内 / それ以外 → 従来どおり。
  - `isStandaloneDisplay`: 追加済み（standalone起動）なら出さない。
- `install-pwa-prompt.tsx`: iOS 判定時は「①画面下の共有ボタン(↑) ②ホーム画面に追加 ③追加」の
  3ステップ番号リストを表示。「追加する」ボタンは出さず「あとで」のみ。
  利用スコア閾値(9)・14日クールダウン・1分ごと再判定は既存ロジックをそのまま共有。
- env追加なし・既存のAndroid/デスクトップ動線は不変。

## ペルソナ実機検証（Playwright・iPhone 14 エミュレーション・dev server）

| 確認項目 | 結果 |
|---|---|
| 常連(スコア10)のトップ訪問でバナー表示・3ステップが4行以内 | ✅ 表示。手順は番号付き3行 |
| 「あとで」で閉じる→リロードで再表示しない(14日クールダウン) | ✅ |
| 新規訪問者(スコア0)には出ない | ✅ |
| /ky でも表示され、底部ナビ・KY保存ツールバーを塞がない | ✅ 既存Androidバナーと同位置 |
| 「追加する」ボタン(iOSでは機能しない)が出ない | ✅ |

スクリーンショット確認: バナーは底部ナビの上に収まり、絵文字＋太字見出し＋3ステップで
「PWA」「インストール」という語を使っていない。佐藤さん評: **「これなら娘に聞かずにできる。使う」**

## 残課題

- LINEアプリ内ブラウザ等での「Safariで開く」誘導文は表示確認のみ（実機LINEでの検証は不可）。
- Web Push（習慣化のもう1本のレバー）は env 追加が必要なためオーナー判断待ち（BACKLOG記載済み）。
