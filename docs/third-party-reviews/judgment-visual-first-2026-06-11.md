## 改修前の指摘（なりきり操作で全部列挙）

1. **石綿チェッカー（A）**: 判定結果が常時 amber と rose の2箱 — 義務が「あってもなくても」黄・赤に塗られていて、色が何も語らない（chemical-RA「常時amber」と同型の色文法違反）。モバイルでは6入力の下に結果があり、結論が画面外。条文リストが常時全開で壁文字。
2. **管理区分判定（A）**: 判定後の結果カードに「第3管理区分」と小さめの見出しが出るだけで、3区分のどこにいるか・止まるレベルかが3秒で分からない。判定ボタンを押しても結果が画面外（フォームが長い）。第3の「改善完了まで保護具着用」が本文に埋没。
3. **ストレスチェック（B）**: 判定ボックスがページ最下部（実施手順11項目＋ベースライン11項目の下）— 無読ペルソナは絶対に到達しない。未回答でも「準備が必要」赤系が出る偽警報。early が一律 rose で、努力義務（50人未満）の事業場まで赤で責められる。
4. **健診スケジューラ結果（B）**: 期限超過の有無を知るには第5章まで約5画面スクロール。台帳のサマリーバッジは正しいが最上部に結論がない。

## 改修内容

- **共通**: 判定→1メッセージの集約を全て純関数に分離しテストで色文法を固定（class-visual.ts / asbestos-conclusion.ts / readiness-visual.ts / health-checkup-conclusion.ts、計23テスト）。solid チップは WCAG AA 固定（amber400+黒系文字・rose800白・emerald700白 = ra-visual と同じ組）。
- **石綿**: 最上部に「やること N件」結論カード（義務=指示の**青**・対象外=緑。赤黄は使わない — 義務は期限超過ではない）。義務4種（事前調査・調査者資格・労基署報告・自治体報告）をチップで列挙。詳細2箱は状態連動（義務あり=青/対象外=無彩）に是正、条文は件数付き折りたたみへ（内容不変）。
- **管理区分**: デカ表示「第N管理区分」＋状態チップ（良好/要改善/直ちに改善）＋3区分色帯▼マーカー＋次にやること。第3は深紅バナー「直ちに改善 — 改善完了まで有効な呼吸用保護具の着用が必要」を最前面。判定時に結論へ自動スクロール。**A4評価記録は不変**（結論ブロックは print:hidden、従来見出し/記録ヘッダ/内訳/確認欄の print 表示を機械検証済み）。
- **ストレスチェック**: 自己評価セクションをページ先頭へ（診断ファースト）。回答が揃うまでは青「回答のこりN問」（未回答を赤黄で責めない）。全問回答後に整備率%をデカ数字で ready=緑/partial=黄/early=**義務事業場のみ赤**（実施義務未達=停止級）・努力義務は黄。案内文は readinessGuidance をそのまま使用（言い換えゼロ）。
- **健診結果**: 最上部に結論カード = 期限超過(赤)→期限間近(黄)→記録のこり(青=未記録を緑にしない偽安心防止)→期限内(緑)。台帳と結論カードを共有ストア(useSyncExternalStore)で接続し、**台帳に日付を入れた瞬間に最上部の結論が変わる**（保存形式・キーは従来と同一=既存入力は引き継がれる）。
