# 第三者レビュー: 新着情報ハブ（/whats-new）— 毎朝チェックするコンサル目線

- 日付: 2026-06-09 JST
- 対象: 新着情報ハブ `/whats-new`（法改正・事故速報・通達・報道・重大災害事例の一元表示）
- 方式: ペルソナなりきり＋Playwright実機操作（iPhone相当 390×844 / 本番ビルド `npm start`）

## ペルソナ

**村上さん（47）／独立系の労働安全コンサルタント。**
顧問先は建設7社・製造2社が中心。毎朝コーヒー片手にスマホで「昨日からの動き」を3分で把握し、
関係する顧問先にだけLINEで一報を入れるのが日課。我慢の限界＝
「①前回見た後の"新しい分"だけ見たいのに全部出てくる」「②建設の客に関係ない医療・小売の法改正まで読まされる」。
どちらかでも起きると『結局e-Govと厚労省を自分で見る方が速い』と離脱する。

## 指摘（忖度なし）

1. **【致命・鮮度】「新着のみ」に絞れない。** ハブは「新着 N件（前回ご覧になった以降）」と件数は出すのに、
   一覧は全 **81件** を時系列でベタ表示。新着バッジは付くが、毎朝スクロールして新着バッジを"探す"作業になる。
   前回が5日前の常連でも、新着3件を見るために81件を上から目で追う必要がある。
2. **【致命・業種別】業種で絞れない。** 隣の `/laws`(法改正一覧)は業種マルチセレクト＋属性＋規模＋施行状況と
   フィルタが充実しているのに、毎朝の入口であるこのハブには**カテゴリ絞り込みしかなく、業種フィルタが無い**。
   建設専門の村上さんも、医療・小売・林業向けの法改正を全部読まされる。
   データ上は法改正項目に業種タグ（`industries`＝メール配信のセグメントに使用済）が既に付いているのに、UIで使われていなかった。
3. （軽微）「最終確認は公式で」等の注意書きは良いが、絞り込み行が無いぶん画面の主役が"説明文"になっていた。

## 改善（最小・非破壊・既存データ再利用）

`web/src/app/(main)/whats-new/whats-new-client.tsx` に2つの1タップ絞り込みを追加：

- **🆕 新着のみ トグル（鮮度）**: 前回閲覧日（localStorage）以降の項目だけに絞る。件数表示と同一基準。
  常連（前回=5日前）で **81件 → 3件**。毎朝の"探す"が消え、開いて1タップで差分だけ見える。
- **業種で絞る チップ（業種別）**: 法改正の業種タグで絞る。事故速報・通達など業種非依存の項目と
  「全業種向け法改正」は常に残す＝建設の客に関係ない他業種専用の改正だけが消える。
  実測 **法改正40件 → 建設26件 / 医療福祉16件**。業種タグを持つ項目が無い業種チップは自動で非表示。
- 絞り込み中は「絞り込み解除」を表示。業種選択時は「『建設』に関係する法改正＋業種を問わない速報・通達を表示中」と明示。

判定ロジックは純粋関数 `filterNewsHubItems` / `isNewSince` / `newsItemMatchesIndustry` として
`news-hub-types.ts` に切り出し、`news-hub-filter.test.ts`（8ケース）で
「件数表示と新着フィルタが同一基準で一致」「他業種専用だけ除外」等を固定検証。

## 再レビュー（本番ビルドで実操作）

- 前回=2026-06-04 の常連を localStorage で再現 → 「新着のみ」1タップで一覧が **3件**（2027-04-01・2027-01-01・2026-06-05、
  すべて前回以降）に。表示件数「新着 3件」と完全一致。
- 「法改正」＋「建設」で **26件**、「医療福祉」で **16件** と業種で実際に減ることを確認。
- 村上さん評価:「開いて新着のみ→建設、で関係する分だけ。これなら毎朝ここで完結する。e-Govを開き直さない」→ 採用ライン到達。
- 注記: 初期マウントの localStorage 読み取りは既存の `queueMicrotask` 遅延のままにした（同期読みは
  サーバ(null)とのハイドレーション不整合になるため）。表示件数と新着フィルタは同一の settled な `lastVisit` を読むので
  確定後は常に一致する（純関数テストで担保）。
