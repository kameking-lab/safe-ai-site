# サイネージ「現場の安全状態」パネル — 第三者レビュー（2026-06-10）

## 対象
/signage ダッシュボードへの /site-records 記録キット連携（柱2: ダッシュボード強化）。
これまでサイネージは気象・事故ニュース・法改正など**外部情報のみ**で、自現場の記録
（未是正指摘・要対策ヒヤリ・使用不可機械・委員会未開催）が一切出ていなかった断絶を解消する。

## 実装の要点
- 新規: `web/src/lib/signage/site-safety.ts`（純関数 `selectSignageSiteSafety` ＋テスト6件）、
  `web/src/components/signage/signage-site-safety.tsx`（パネル本体）。
- 集計は #462 の `buildDailyActions`（/site-records「今日やること」と同一ロジック）を再利用し、
  **判定の二重実装ゼロ**。サイネージ向けの選別だけを新設:
  - 現場レベルのソース（パトロール・ヒヤリ・点検・委員会）のみ掲示。
  - **健診は除外**（個人の健康情報を全員が見る大画面に掲示しない）。
  - **カレンダー（info）は除外**（要対応でない参考情報で画面を消費しない）。
- 配置: 右カラムの「本日のリスク予測」直下。`shrink-0` ＋ `xl:max-h-[30vh]` で
  下のトレンド・法改正（flex-1）が自動的に譲り、**1画面フィットを壊さない**。
- 記録がひとつもない端末（サイネージ専用機など）では**パネル自体を出さない**。
- 枠色で状態が遠目に分かる: 期限超過あり=赤 / 要対応のみ=橙 / 全対応済み=緑＋安心表示。

## localStorage 共有可否の実機検証（タスクの検討事項）
- 同一端末・同一ブラウザなら `/site-records`（入力タブ）と `/signage`（掲示タブ）は
  **同一オリジンの localStorage を共有**することを Playwright 実機で確認。
- 別タブで点検記録（使用不可機械）を保存 → `storage` イベントで**サイネージ側が再読込なしで
  即時更新**（要対応 4件→5件）されることを確認。リロード不要。
- 加えて5分間隔の再集計（同一タブ内での日付跨ぎ＝期日超過への昇格を拾う保険）と
  60分の既存ページ全体更新が重なる。**別端末とは共有されない**（既知の制約・
  Supabase同期はPath Aのため見送り）。パネル下部に「この端末に保存された記録から自動集計」と明記。

## 1回目レビューの指摘（忖度なし）
1. **「担当: 山田 / 重大期日 2026-06-05」と誤読する**: detail末尾の重大度表記と期日が密着し
   「重大期日」という存在しない語に見える。→ 期日の前に「／」区切りを挿入。
2. **使用不可機械が「ほか1件」に隠れて種別すら分からない**: 表示上限4件の運用で、
   非対話の常時掲示では隠れた項目は存在しないのと同じ。朝礼で言い忘れれば事故に直結。
   → 「ほか n 件（点検）」のように**隠れた項目の種別を列挙**し、口頭補足できるようにした。
3. **タイトル文字がトレンド見出しより小さくTVで読みにくい** → `lg:text-sm` に引き上げ。
