# サイネージ シナリオ「🌅朝礼前」プリセット 意味付け是正レビュー（2026-06-09）

## 対象
`/signage` ダッシュボード上部のシナリオプリセットバー（朝礼前 / 休憩時間 / 退場時）。

## ペルソナ
53歳・建設現場の職長。紙20年。ITは苦手で、ボタンを押して**何も起きなければ「壊れている」と判断して二度と触らない**。朝礼前に大画面TVの前で操作する。1画面で完結しないと使わない。

## 指摘（Playwright実機・1920×1080）
- 初期表示は `displayMode = "floorplan"`（中央に「現場レイアウト」）。
- シナリオバーの「🌅朝礼前」を押下 → **画面に一切変化なし**。`mode: "floorplan"` が既定値と同一のため、何も切り替わらない。職長目線では「押しても反応しない＝壊れている」。
- 一方「☕休憩時間」（map）「🌆退場時」（workdocs）は実際にビューが切り替わるため、朝礼前だけが死んで見える。
- さらに active ハイライトが `displayMode === mode` 判定のため、floorplan を表示中は**クリックしていなくても朝礼前が常時ハイライト**＝押した/押してないが分からない。
- 朝礼前の職長が一番欲しいのは「読み上げ原稿」（#433 で結線済の `SignageMorningScript`）なのに、朝礼前プリセットからは辿れず、右端の🎤ボタンを別途探す必要があった。

## 改善
- 「🌅朝礼前」の押下動作を「`floorplan` 表示 ＋ 朝礼スクリプトのモーダルを開く」に変更。押すと必ず読み上げ原稿モーダルが出る＝**反応が一目で分かり、朝礼で本当に使う原稿に直行**。
- active ハイライトを各プリセットの実体に合わせて是正（朝礼前=floorplan かつ原稿表示中 / 休憩時間=map / 退場時=workdocs）。floorplan 表示中の誤ハイライトを解消。
- 「休憩時間」「退場時」押下時は朝礼スクリプトモーダルを閉じてから切替（モーダルの残留を防止）。
- 既存の `SignageMorningScript`・`setShowMorningScript` を再利用。新規部品なし、計算/データ取得ロジックは不変。1画面フィット（横長 overflow=0）も維持（モーダルはオーバーレイ）。

## 再レビュー（Playwright実機）

- 職長「押したらちゃんと原稿が出る。これなら朝礼前にこれを押せばいい、と分かる」＝採用ライン到達。
- 横長 overflow=0・1画面フィットは維持。
