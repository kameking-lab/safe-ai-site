## 致命的な指摘（改善前）
- **X型を指定したのに1位がランヤード**：スコアリングがフラットな合算（種別+30 / 業種+30 / 高評価+10）で、
  形状という概念を持たない「シングルランヤード」が **70点（種別30＋業種30＋高評価10）** となり、
  同じ70点のX型フルハーネスと同点 → レビュー数で上回って **1位を奪取**していた。
  - 実測（改善前・本番DB再現）：`shape=X型 + lanyard=シングル + useCase=construction` の1位＝
    「サンコー（タイタン） ランヤード（巻取式・シングル） rev353」。X型ハーネスは2位以下に沈む。
  - ランヤードは形状（X型/Y型/H型）を**満たしようがない**のに、ハーネス固有属性の検索結果で上位を占めるのは
    「X型ハーネスを探す」という主目的に反する。

## 改善
スコアリングの加点ルール・データ・スキーマは一切変えず、**ソートのクラス優先ティアのみ追加**。

- `recommendItems` に純関数 `isShapeSelected` / `classTier` を追加（`src/lib/equipment-finder/recommendations.ts`）。
  - `isShapeSelected(category, answers)`：形状質問（id=`shape`）に「問わない(any)」以外が選ばれているか。
    形状質問の無いカテゴリ（ヘルメット等）では常に false ＝他カテゴリは無影響。
  - `classTier(item, shapeSelected)`：形状未指定なら全件 tier 0（**従来挙動を完全維持**）。
    形状指定時のみ、フルハーネス=tier 0 ／ 形状概念の無いランヤード等=tier 1 に降格。
- ソートキーを「**クラスtier昇順 → スコア降順 → レビュー数降順**」に変更。
  形状指定時はフルハーネスが必ずランヤードより上位に並ぶ＝「別枠/降格」を1リスト内で実現。
  形状=問わない時は tier が全件0でスコア純ソートに戻り、ランヤードが1位でも正常（後方互換）。
- テスト `recommendations.test.ts`（9件）：バグ再現条件で先頭がフルハーネスであること／全ハーネスが全ランヤードより上位／
  形状=any では従来どおりランヤードが1位になり得る／ハーネス同士はスコア→レビュー数の従来順維持／
  形状質問の無いカテゴリは降格しない、を本番DBで検証。
