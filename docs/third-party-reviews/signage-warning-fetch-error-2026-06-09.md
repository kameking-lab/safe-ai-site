# サイネージ ダッシュボード 気象警報パネル — 取得失敗を「警報なし」と誤認する問題のレビュー

対象: `/signage`（無人サイネージ ダッシュボード、`web/src/app/signage/page.tsx`）
日付: 2026-06-09 / 軸2（既存機能を第三者目線で安全に）

## ペルソナ

**佐々木さん（44歳・総合建設業の元請安全担当）**
- 現場事務所にタブレットを常設し `/signage` を1日中流しっぱなしにしている。
- 台風・大雨シーズンは**朝イチでサイネージの「本日の気象警報」だけを見て**、協力会社への一斉メール（作業中止/高所作業見合わせ）を出すか判断する。気象庁サイトをわざわざ開かない前提で運用している。
- 我慢の限界:「サイネージが『警報なし』と言ったから安心して作業させたら、実は通信が落ちてただけで本当は大雨警報が出ていた」——これが一度でも起きたら二度と信用しない。

## 指摘（実機・Playwrightで再現）

無人現場で最も起きる障害＝**ネットワーク断/サーバ502** を Playwright の `route.abort()` で再現し、図面モードの「本日の気象警報」パネルを観察。

- **致命1: 取得失敗が「警報なし」と区別できない。** 取得失敗時は `setBundle(null)` となり、パネルは `bundle?.jmaHeadline` の有無だけで分岐するため、ヘッドラインが無い＝「現在、選択地点に発表中の警報はありません。」と表示。**本当に警報が無いのか、取得に失敗して確認できないのかが画面上まったく同じ**。無人運用で誤った「全員OK」を生む。
  - 実測（before）: 警報パネル文言 = `本日の気象警報 現在、選択地点に発表中の警報はありません。`（API abort 状態）。
- **致命2: そもそも error 状態に到達しない経路があった。** `refreshAll()` は `await Promise.all([fetch(...), ...])` を try/catch せず実行していたため、`fetch` が **reject（ネットワーク断）** すると関数全体が throw し、`setBundleStatus("error")` が**一度も呼ばれず** `bundleStatus` が `loading` のまま固着。error の文言を足しても、最頻の障害（通信断）では表示されない。`!res.ok`（HTTP 5xx）か JSON parse 失敗のときだけ error になっていた。
- 隣の「地点リスク（日次）」パネルは取得失敗時に「取得に失敗しました」と明示していたのに、より重要な**気象警報パネルだけが沈黙**していた＝一貫性の欠如。
- 地図モードでは取得失敗時に**何も出ない**（ヘッドラインがある時しか描画しない）＝こちらも無言で失敗を隠す。

## 改善

1. **純関数 `resolveWeatherWarningPanelState(status, jmaHeadline)` を新設**（`web/src/lib/signage/weather-warning-panel-state.ts` ＋テスト5件）。`error / loading / headline / none` を明確に判定。**error は警報の有無に関わらず最優先**（取得失敗時に古いヘッドラインが残っていても error を出し、誤った安心を防ぐ）。
2. **図面モードの警報パネルを4状態に分岐**: 取得失敗＝赤枠（rose）で「⚠ 気象データの取得に失敗しました。警報の有無を確認できません。」＋「気象庁で確認 →」リンク／取得中＝「気象データを取得中…」／警報あり＝ヘッドライン／本当に警報なし＝従来文言。
3. **地図モードにも取得失敗の赤文言＋気象庁リンクを追加**（従来は無言）。
4. **データ層を是正**: `signage-data` の fetch を「絶対に reject しない」ガード（`.then(...).catch(() => ({ ok: false }))`）に変更し、**ネットワーク断でも確実に `bundleStatus="error"` に落ちる**ように。risk/revision サービスとの並列取得（Promise.all）は維持。

判定ロジック・気象/地震データ・スキーマ・1画面フィットは不変。新規依存0。気象庁URLは既存の `jmaLink` を再利用（捏造なし）。

## 再評価（Playwright実機）

- 取得失敗（API abort）→ `本日の気象警報 ⚠ 気象データの取得に失敗しました。警報の有無を確認できません。気象庁で確認 →`（赤枠）。**「警報なし」と明確に別物**。overflowX=0px。
- 正常取得 → 実データのヘッドライン（伊豆諸島の強風・高波等）を従来どおり表示、console error 0、overflowX=0px。**誤検知（正常時に失敗表示）なし**。
- 佐々木さん「これなら『警報なし』と『取れてない』が一目で違う。赤が出たら自分で気象庁を見る。サイネージの一言で安全判断を間違える事故が防げる」＝採用ライン到達。
