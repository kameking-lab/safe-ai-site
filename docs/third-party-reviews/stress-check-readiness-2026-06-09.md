# 第三者レビュー: ストレスチェック実施準備（/mental-health-management/stress-check）

実施日: 2026-06-09 / 軸2（既存機能を第三者目線で使いやすく）
対象タスク: 「50人になったばかりの事業場の衛生担当」ペルソナでレビュー→実施手順の分かりやすさ・抜け漏れ防止

## ペルソナ

**佐々木さん（46歳・製造業・専任ではない総務兼務の衛生担当）**
- 会社が今年ちょうど常時50人を超え、ストレスチェックが「努力義務 → 義務」に変わった。**今年が初回**。
- やったことが無いので「何を・どの順番で・いつまでに」が一番知りたい。法令の条文羅列より段取り表が欲しい。
- 我慢の限界:「要件は分かったが結局どこから手を付ければいいの？」で止まると、結局コンサルに丸投げするか後回しにする。
- スマホ（現場巡回の合間）でも開く。横スクロールやガタつきは一発で見限る。

## 指摘（Playwright実機・忖度なし）

PC(1280) / スマホ(390) で実操作。常時労働者数を 52 人（＝義務事業場）にして自己評価を「全て未整備」で診断。

1. **【致命】「実施手順（段取り）」が存在しない。** ページは①ベースライン要件11項目（＝“何が満たされているべきか”の一覧）②自己評価フォーム の2本立てだが、**“何を・どの順番で・いつまでに”という時系列の手順が一切無い**（`hasJisshiTejun: false` を実測）。初回の事業場が一番欲しい「年間の進め方」が欠落＝タスクの主眼「実施手順の分かりやすさ・抜け漏れ防止」が未達。要件一覧は概念順で、準備→実施→事後→報告の段取りには読み替えが要る。
2. **【バグ】自己評価の判定文が義務バッジと矛盾。** 52人入力で上部に「**義務（50人以上）**」バッジが出るのに、判定本文は「実施前にベースライン要件の整備が必要です。**50人未満事業場はさんぽセンターの活用を検討してください。**」と表示（実測）。義務事業場の担当者に「50人未満は…」と案内する＝自分向けの情報か判断できず信頼を損なう。原因は `early` 判定の本文が tier 非依存のハードコードで 50人未満向け文言を固定していたため。
3. **（中）自己評価から次の行動への導線が無い。** gap 一覧は質問文（「〜していますか？」）を再掲するだけで、「で、次に何をするのか」へ繋がらない＝抜け漏れ防止の最後の一押しが不足。

## 改善実装

- **実施手順（年間の流れ）セクションを新設**（`#procedure`、使い方の直後・要件一覧の前に配置）。準備期→実施期→事後対応期→報告・保存期 の **10ステップを時系列で**提示。各ステップにフェーズ色チップ・🕒目安時期・「義務事業場のみ」バッジ（労基署報告）・操作手順の説明文を付与。データは `STRESS_CHECK_PROCEDURE`（`src/data/mental-health-rules/stress-check.ts`）に型付きで定義し、各ステップは対応するベースライン要件IDを保持（相互参照・テストで網羅性を担保）。
- **判定文を tier 連動の純関数 `readinessGuidance(verdict, tier)` に切り出し（`mental-health-flow.ts`）**。義務（50人以上）には「50人未満」「さんぽセンター」を**一切出さない**／努力義務（50人未満）にはさんぽセンター無料支援を案内、という出し分けで矛盾を解消。`VERDICT_META` から `body` を削除し本文を動的算出。
- **判定ボックスに「次にやること：実施手順で進め方を確認する →」CTA**（`#procedure` アンカー）を gap がある時に追加＝自己評価→手順への一気通貫。

### テスト（+9）
