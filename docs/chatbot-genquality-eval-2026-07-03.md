# チャットボット生成品質evalの確立と本番ベースライン（2026-07-03・F3）

作成: Claude (Fable 5) / 根拠診断: `docs/fable-diagnosis-2026-07-02/04-chatbot-accuracy.md`（T7）

## 1. 何を作ったか

診断04の核心「検索evalは98〜100%なのに実機正答60%。生成品質を測るevalが存在しない」に対し、
「質問→最終回答」の品質を機械採点する測定系を新設した。

- **ゴールデンセット** `web/src/lib/chatbot-genquality.fixture.ts`
  診断04の実機23問を固定化。各問に (a) 結論キーフレーズ（AND of OR）、(b) 誤結論パターン
  （正規表現）、(c) gold条文、(d) コーパス突合エビデンス＋**e-Gov正本アンカー**（法令番号・条・
  正本上の事実）を持つ。goldの出所は正本であってコーパスではない（F2と同じ反循環設計）。
- **機械採点器** `web/src/lib/chatbot-genquality-scorer.ts`
  診断04の○/△/×判定を5チェックで機械化: 結論キーフレーズ／誤結論パターン／根拠条文の引用
  （構造化出典→sources→本文の3段照合・枝番誤マッチ防止）／範囲外対応（範囲外質問の誠実な
  no-hit・範囲内正答への偽の範囲外警告）／プレースホルダ漏出。失敗は診断04 §3 の分類
  （conclusion-missing / conclusion-wrong / citation-missing / false-scope-warning /
  out-of-scope-mishandled / placeholder-leak）にマップされる。
- **CIゲート（常設）** `web/src/lib/chatbot-genquality.test.ts`＋`chatbot-genquality-scorer.test.ts`
  APIキー不要層のみを `npm run test` で常時回帰（下記 §2）。
- **実機測定ハーネス** `npm run eval:chatbot-gen`（`web/scripts/chatbot-eval-phase2.ts` →
  `chatbot-genquality-live.test.ts`）。`chatbot-phase2-metrics.test.ts:192` で「次タスク」の
  まま未実装だった chatbot-eval-phase2 の実装体。環境変数ゲートでCIでは全skip。
  `CHATBOT_EVAL_BASE_URL` で対象環境、`CHATBOT_GENQUALITY_INPUT` で過去レポートの再採点、
  `CHATBOT_GENQUALITY_TARGET` で正答率下限のexit code判定。

## 2. CIで何が落ちるようになったか

1. **コーパス正本突合**: 結論の根拠事実（例: 派遣法45条の「派遣元」、安衛則44条の「1年以内
   ごとに1回」、安衛則52条の21の「50人」）がコーパス条文から消える編集で落ちる。エラーには
   e-Gov正本アンカーが表示される。
2. **RAG到達性**: 23問（既知ギャップ除く22問）でgold条文がLayer1ホワイトリスト（top10）から
   外れる・スコアがno-hit閾値(0.5)を割ると落ちる（診断04 Q6「頻度」型の実質未回答の再発防止）。
   既知ギャップ（GQ23）はratchetテストで台帳管理＝解消したらfixture更新を強制。
3. **テンプレ層回帰**: 範囲外質問（天気・投資信託）の範囲内化／PIN確定トピックのadjacent
   誤ヘッダ復活（T8）／gold法令の正式名・略称引用への偽の範囲外警告（T1・長音分断含む）／
   プレースホルダsanitizer劣化（T3）で落ちる。
4. **誤答検出の常設実証**: 診断04の実誤答4型（Q20派遣先誤断定・Q12実質未回答・Q8偽警告・
   Q10プレースホルダ漏出）を再構成した応答が×/△判定されることをテストで固定。採点器自身の
   劣化でも落ちる。

## 3. 本番ベースライン実測（2026-07-03）

対象: `https://www.anzen-ai-portal.jp/api/chatbot`（23問・直列4秒間隔・全問HTTP 200）。
生ログ＝一次記録: `docs/chatbot-genquality-baseline-2026-07-03.json`（回答全文・出典・警告込み）。

- **完全正答（○）: 19/21 = 90.5%**
- **概ね有用（○＋△）: 21/21 = 100%**
- **範囲外対応: 2/2 適切**（条文引用なし・低確信・公式誘導。港湾労働法ノイズも再発なし）
- 明確な誤答（×）: 0件／偽の範囲外警告（gold法令への誤警告）: 0件／プレースホルダ漏出: 0件

### 診断04（2026-07-02実機60%）との整合

診断04の60%は**レーン是正前**の値。その後 T1〜T3（#583/#588）・T5系コーパス補強
（#601/#605/#615/#639）・T6/T8/T9（#632）が本番に入っており、当時の×3件（Q6健診頻度・
Q12ストレスチェック・Q20派遣）と△の大半が本測定では○に転じた。60%→90.5%の差分は
これらのPRの効果と整合する（機械採点の甘さではない。採点器が診断04の誤答4型を×/△判定
することは§2-4のテストで確認済み）。診断04と同じ判定が残った箇所も一致する（下記）。

### 残存する△ 2件（失敗分類: いずれも conclusion-missing）

- **GQ02（酸欠の資格）**: 特別教育（酸欠則12条）のみで作業主任者技能講習（酸欠則11条）が
  欠落＝診断04の△と同一。真因を特定済み: 「酸欠作業に必要な資格」のRAG top10に**11条が
  入らず**（12条・2条・6条・9条…のみ）、ホワイトリスト制約下のGeminiは11条を引用できない。
  → 是正はRAG層（synonyms/PIN: 酸欠×資格→11条）。診断04 T4の完了条件
  「『酸欠 資格』で酸欠則11条がtop5」は未達のまま。
- **GQ23（解雇予告）**: 労基法20条は#615で収録済だが「解雇予告のルール」のRAGスコアが
  0.12でno-hit経路に落ち、relatedに労基法20条を示すのみで「30日」に答えない。
  → 是正はRAG層（口語→条文の同義語）。fixture上は既知ギャップとしてratchet管理中。

## 4. 検出実証（F2「誤区分1件でCI落ち」と同型）

1. **コーパス改変→CIゲート検出**: `haken-anzen-eisei.ts` 45条要約から元/先の区別を消す編集
   （診断04 Q20の真因＝#588是正前の状態を疑似再現）を1件仕込む→ `chatbot-genquality.test.ts`
   が「GQ20: 派遣法第45条 から根拠事実が欠落 [派遣元]（正本: 労働者派遣法（昭和60年法律第88号）
   第45条…）」で失敗。復元後は12/12全緑（実施ログ: 2026-07-03）。
2. **応答改変→採点器検出**: ベースラインJSONのGQ20回答の「派遣元の事業主の義務」を
   「派遣先の事業主の義務」に1箇所改ざん→再採点モードで verdict が correct→**incorrect**
   （conclusion-wrong）に転じ、完全正答率 90.5%→85.7%、`CHATBOT_GENQUALITY_TARGET=0.9`
   指定時に exit code 2。なお誤結論regexは「派遣先ではなく派遣元が…」型の正答複文を
   誤検知しないことを本番実回答で検証済み。

## 5. 既知の限界（正直な注記）

- 結論キーフレーズ判定は表記ゆれの近似であり、「条文は正しいが結論が読み取りにくい」
  （診断04 Q14の逐語引用型）は機械では△にできないことがある。読みやすさの評価は対象外。
- 23問は診断04由来の固定セット。fixture過学習の検知（rag-100q-freshに相当する別セット）は
  問数を増やす段階の課題。
- 実機測定はGeminiの非決定性を含む（同一質問でも○↔△が揺れうる）。ゲートはCIの決定的層に
  限定し、実機値は傾向監視として扱う。
