# 用紙ファースト型UI 設計 (KY / 安全衛生日誌)

最終更新: 2026-05-23
担当: claude opus 4.7
対象ブランチ: feat/ky-diary-paper-first-redesign
関連ペルソナテスト: docs/persona-test-2026-05-23/report.md (建設業職長A、◯率37.5%)

---

## 1. 設計の狙い

現状: フォーム入力 → 送信 → 結果表示の3段階。最終アウトプット (印刷物・PDF) の形が入力中に見えず、職長が「これでよかったか」を確認できない。

目標: ページを開いた瞬間に「完成形のA4 1枚」が画面に表示され、用紙の各欄に直接書き込む体験。フォーム要素 (input/textarea) は用紙レイアウトの上に直接埋め込み、編集UIと完成形が一致する。

ペルソナA改善見込み: 朝礼3分の◯率を 37.5% → 60% に。要因は「最終アウトプットの理解しやすさ」「片手操作で書ける」「印刷プレビューが瞬時に分かる」。

---

## 2. 共通レイアウト原則

- 画面全体: A4縦相当 (アスペクト比 約 1:1.414)。`max-w-[820px]` で中央寄せ
- PC: 用紙全体を一画面に。周辺ツールバーは右側に縦並びで固定
- スマホ: 用紙幅を画面幅にフィット (1カラム)、縦スクロール
- サイネージ: 用紙ファースト適用外 (既存ルートを維持)
- @media print: ツールバー・ナビ・余白を非表示にし、用紙のみA4印刷

### スタイル指針
- 紙の色: `bg-white`
- 罫線: `border-slate-400` (印刷時にも視認可能な濃さ)
- 余白: `p-6 sm:p-8 print:p-4` (印刷時はA4余白を最大化)
- ヘッダ帯: `bg-slate-100 border-slate-400` (印刷でも分かる薄グレー)
- フォーム要素: `border-b` のみで紙の罫線と同化。`focus:bg-amber-50` で編集中欄を視覚化

## 3. KY用紙 (危険予知活動記録表)

オリジナル意匠で、構造のみ建設業界の慣行 (4ラウンド法) を参考。

```
┌────────────────────────────────────────────────────────────┐
│ ◆KY活動記録表 (危険予知活動)                  No.____      │
├──────────┬────────┬──────────┬───────────┬─────────────┤
│ 日付     │ 天候   │ 現場名   │ 作業時間  │ 作成者      │
│ YYYY/M/D │  ☀☁☂  │ ___      │ __:__〜__:│  ___       │
├──────────┴────────┴──────────┴───────────┴─────────────┤
│ ◆作業内容                                                  │
│ ________________________________________________________ │
├────────────────────────────────────────────────────────────┤
│ ◆どんな危険が潜んでいるか? (1R 現状の把握)                 │
│ ┌──────────────────────────────────────────────────────┐ │
│ │ 作業シーン (テキスト or イラスト枠 - 任意)            │ │
│ └──────────────────────────────────────────────────────┘ │
│ ① ___________________________ (危険要因)                  │
│ ② ___________________________                              │
│ ③ ___________________________                              │
│ ④ ___________________________                              │
├────────────────────────────────────────────────────────────┤
│ ◆危険のポイント (2R 本質追究) [◎をつける]                 │
│ ◎ 最重要要因: ___________________________                  │
├────────────────────────────────────────────────────────────┤
│ ◆私達はこうする (3R 対策の樹立)                            │
│ ① ___________________________                              │
│ ② ___________________________                              │
│ ③ ___________________________                              │
├──────────────────────┬─────────────────────────────────┤
│ ◆今日の重点目標      │ ◆指差呼称                       │
│ (4R 目標設定)        │                                  │
│ _________________   │ 「____ ヨシ!」                  │
├──────────────────────┴─────────────────────────────────┤
│ ◆参加者 (確認印 / サイン)                                  │
│ □ □ □ □ □ □ □ □  (氏名と印 / 8人欄)                    │
└────────────────────────────────────────────────────────────┘
```

### 編集UI
- 各 `___` 欄は `<input>` / `<textarea>` の border-bottom only
- 危険要因 ①〜④ は4行のテキスト入力 (行追加可能)
- 対策 ①〜③ は同様
- 「実例から引用」ボタンで KY_EXAMPLES (150件) から industry/workType フィルタで候補表示 → 1クリックで①〜③に流し込み
- 「AIたたき台」ボタンで /api/ky-assist 呼び出し (既存)

### データマッピング (既存 KyInstructionRecordState を流用)
| 用紙欄 | KyInstructionRecordState フィールド |
|---|---|
| 日付 | `workDateYear` + `workDateMonth` + `workDateDay` |
| 天候 | `weather` |
| 現場名 | `reportStamps[0]` (協力会社名と兼用) |
| 作業内容 | `workRows[0].workDetail` |
| 危険要因 ①〜④ | `riskRows[1..4].hazard` |
| 危険のポイント (重点◎) | `riskRows[1..4]` のうち `targetLabel === "◎"` |
| 対策 ①〜③ | `riskRows[1..3].reduction` |
| 重点目標 | `closingNote` |
| 指差呼称 | `riskRows[0].reduction` (自由記述欄に格納) |
| 参加者氏名 | `participants[i].name` |
| 参加者印 | `participants[i].preWork` |

→ 既存の localStorage キー `ky-record` をそのまま使用。データ破壊ゼロ。

├──────────┴────────┴───────────┴──────────────┴──────────┤
│ ◆作業内容                                                  │
│ ________________________________________________________ │
│ ________________________________________________________ │
├────────────────────────────────────────────────────────────┤
│ ◆KY結果 (朝礼で出した危険・対策)                           │
│ ________________________________________________________ │
│ ________________________________________________________ │
│  [/ky から転記] [AIたたき台]                              │
├────────────────────────────────────────────────────────────┤
│ ◆ヒヤリハット / 異常 (有/無)                               │
│ ⃝ 有  ⃝ 無                                              │

├────────────────────────────────────────┬────────────────┤
│ ◆安全活動・パトロール所見             │ ◆翌日の予定    │
│ ___________________________________   │ _____________  │
│ ___________________________________   │ _____________  │
├────────────────────────────────────────┴────────────────┤
│ ◆記入者 ____________   ◆現場代理人 ___________  印     │
└────────────────────────────────────────────────────────────┘
```

### 編集UI
- 必須5項目 (日付・天候・現場名・作業内容・KY結果・ヒヤリハット) は赤い * マーク
- 任意4項目 (パトロール所見・翌日予定など) は灰色 任意ラベル
- 「昨日の日誌から流用」ボタン (`?fromYesterday=1` 既存挙動継承)
- 「/kyから転記」リンク (既存)
- 「保存」ボタン → 既存 store.ts の `addEntry` を使用 → localStorage key `safety-diary-v3`

### データマッピング
| 用紙欄 | RequiredFields/OptionalFields |
|---|---|
| 日付 | `required.date` |
| 天候 | `required.weather` |
| 現場名 | `required.siteName` |
| 業種 | `industry` (エントリ root) |
| 作業内容 | `required.workContent` |
| KY結果 | `required.kyResult` |
| ヒヤリハット | `required.nearMissOccurred` + `nearMissDetail` |
| パトロール所見 | `optional.patrolRecord` |
| 安全指示 | `optional.safetyInstructions` |
| 翌日予定 | `optional.nextDayPlan` |
| 記入者 | (新規 metadata - 既存 schema にない場合は注記欄に保存) |

→ 既存 schema 完全互換。F-010 B縮小実装と並存。

```

## 6. 印刷 / PDF 出力

- ブラウザ印刷ベース (plan-generator と同パターン)
- @media print:
  - `.print:hidden` でツールバー・ナビ・実例パネルを非表示
  - 用紙のみA4 1枚に収まるよう余白 `print:p-4`、フォントサイズ縮小 `print:text-[10px]`
  - 編集欄の border-bottom は印刷時 `border-black` に
- 「PDF出力」ボタン → `window.print()` 呼び出し → ユーザーが PDF保存

## 7. 既存機能との並存

| 機能 | 用紙ファースト後の扱い |
|---|---|
| /ky の simple/detail/custom モード | `?mode=legacy` で従来UI維持。デフォルトは paper |
| /safety-diary 一覧 | `/safety-diary?view=list` で表示。デフォルトは paper エントリ |
| /safety-diary/new (必須5項目) | `/safety-diary?view=list` から「新規」リンクで遷移可能 (既存) |
| /safety-diary/new/detail (任意8項目) | リンク維持 |
| KY実例DB 150件 | 用紙ファースト UI 内の「実例から引用」ボタンで活用 |
| F-010 B縮小 (詳細/月次/印刷アーカイブ) | 影響なし。redirect 維持 |
