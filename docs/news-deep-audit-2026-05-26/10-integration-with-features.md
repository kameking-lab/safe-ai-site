# 10. 既存6機能との統合体験

対象: KY / 打合せ書(safety-diary) / サイネージ / 化学物質RA / 事故DB / チャットボット

## 10.1 現状の連携

- 事故速報→事故DB詳細分析: `/accidents`内で完結（良好）
- 法改正→チャットボット: チャットボットの横断CTAに`/laws?q=`あり（前監査でプレフィル改善済）。逆（法改正→`/chatbot?q=`）は弱い
- 法改正→化学物質RA: 化学物質規制改正と化学物質RAの直接リンクは弱い
- 法改正→KY/打合せ書: 改正→該当作業の注意喚起導線なし

## 10.2 改修（→doc13）

- **【P1】法改正カード→チャットボット**: 各改正に「この改正をAIに質問」`/chatbot?q=<法令名+改正名>`（前セッションのcontext prefill流用）
- **【P1】法改正→化学物質RA**: 化学物質関連改正（category=化学物質）に`/chemical-ra`誘導
- **【P2】法改正→事故DB**: 改正の背景となった災害種別へ`/accidents?q=`
- **【P2】新着ハブ→各機能**: `/whats-new`の各項目から該当詳細機能へ送客（ダイジェスト→詳細）
- **【P3】法改正→KY/打合せ書**: 施行間近の改正を該当作業のKY注意喚起に（「現場に何かさせる」UIは禁止のため、情報提示に留める）

## 10.3 統一AI体験

- 法改正AI要約・事故トレンド要約・チャットボットは同じGemini呼出＋免責＋公式誘導パターン（doc07）で統一。一貫性を担保

## 10.4 結論

事故速報の機能内連携は良好。法改正は「→チャットボット」「→化学物質RA」の双方向化がP1。新着ハブが各機能へのダイジェスト入口となり統合体験を底上げする。いずれも情報提示の動線追加で低リスク。
