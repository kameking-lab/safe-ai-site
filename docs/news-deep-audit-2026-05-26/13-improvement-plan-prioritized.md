# 13. 改修案（優先度付き P0/P1/P2/P3）

各案: 対象 / 現状問題 / 改修内容 / 競合での位置づけ / 工数 / 流用パターン / 誤読リスク。

## P0（必須・最優先）

### P0-1: 法改正「施行前／施行済」区別【見せ方の核】
- 対象: `components/law-revision-list.tsx`、`lib/types/domain.ts`（LawRevision）
- 現状問題: 施行日はテキスト表示のみ。施行前/済の区別・「今対応が要る改正」フィルタが無い
- 改修内容: (1)ステータス判定（e-Gov`current_revision_status`を一次、無い手書きは`enforcement_date`vs今日）(2)バッジ「施行済/施行前(あとN日)/施行日未定」(3)「施行前のみ/施行済のみ」フィルタ(URL同期)(4)施行前カウントダウン
- 競合: 中小の「気づかず違反」防止に直結。汎用ニュースに無い実務価値
- 工数: 中（半日）/ 流用: 既存フィルタUI・URL同期パターン / 誤読リスク: 低（日付事実の表示）

## P1（90+到達に必須）

### P1-1: e-Gov法令API v2 自動取込ETL【自動取得の核・社長要件#2】
- 対象: 新規`scripts/etl/egov-revisions-fetch.ts`＋`.github/workflows/etl-egov-revisions.yml`＋`revisions-ingest`にe-Gov v2マッパー
- 現状問題: 法改正の実データ自動取込が空（手書き100件）
- 改修内容: 対象安衛法令のlaw_idを順次e-Gov API取得→構造データ抽出（公布日/施行日/施行状況/番号/category/URL）→形式検証＋未確認明記→JSONL diff-onlyコミット→既存pipelineで表示
- 競合: 「主要安衛法令の改正・施行状況を自動追従」は他に無い
- 工数: 中〜大 / 流用: 月次速報ETL（形式検証・未確認明記・推測禁止・diff-only）＋`officialDbRevisionImportMapper` / 誤読リスク: 低（事実データのみ・出典明示）/ **新規env不要・GitHub Actions月次で無料枠内**

### P1-2: 新着情報ハブ `/whats-new`＋ホーム訴求
- 対象: 新規`/whats-new`、ホーム
- 現状問題: 最新が複数ページに分散、一望できない
- 改修内容: 法改正(施行前含む)＋月次速報＋報道feed＋通達/解説新着を1画面集約。新着バッジ(localStorage前回閲覧比較)。ホームに新着カード
- 工数: 中 / 流用: 既存データの再構成のみ・新規ETL不要 / 誤読リスク: 低（事実集約・公式リンク必須）

### P1-3: RSS/Atom配信 `/feed.xml`
- 対象: 新規Route Handler
- 現状問題: 認証不要の通知手段が無い
- 改修内容: `/whats-new`集約ロジックを共有しRSS出力。自サイト生成サマリ＋公式誘導（厚労省RSS本文は転載しない）
- 工数: 小〜中 / 流用: 集約ロジック共有 / 誤読リスク: 低 / コンプラ: doc11遵守

### P1-4: 事故速報→トレンドAI連動
- 対象: `/api/accidents/trend-summary`、`lib/accidents/trend.ts`
- 現状問題: トレンド要約がcuratedサンプル依存で「直近1か月0件」になりがち
- 改修内容: 月次速報を要約入力に併用。既存ハルシネーションガード維持
- 工数: 小〜中 / 誤読リスク: 低（ガード既存）

### P1-5: 法改正→チャットボット/化学物質RA 双方向動線
- 対象: `law-revision-list.tsx`
- 改修内容: 各改正に「この改正をAIに質問」`/chatbot?q=`、化学物質改正に`/chemical-ra`誘導（前セッションのprefill流用）
- 工数: 小 / 誤読リスク: 無
