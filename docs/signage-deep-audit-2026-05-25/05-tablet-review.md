# 05 タブレット実機検証（768px相当）（軸5）

検証手段: Playwright で本番 /ky/morning を 768×1024 実レンダリング。スクショ: `screenshots/tablet-768.png`。

## 実測値（系統1, 768px）
- 横スクロール: **なし**（docWidth=768）✓
- コンテンツ幅: 768px（フル）
- 「本日の主な作業」見出し: 48px（`sm:text-5xl` 発火）— 共用タブレット設置に十分
- リスクTop3: `sm:text-4xl`（36px相当）で見やすい

## 想定ユース: 現場事務所/詰所に共用タブレット常設
- 朝礼: タブレットを立てて全員で唱和 → 系統1で実用的。
- 6桁コード入力で当日のKYを呼び出す運用が刺さる（職長スマホで作成→共用タブレットに6桁入力）。
  - ただし**本番では6桁共有が permission denied で不通**（Phase A）。インフラ修正が前提。

## 課題（タブレット）
- T-1: タブレット横置き(1024×768)時、系統1は max-w-6xl 内に収まり余白は少ないが、フォント上限が48pxで据え置き。離れて見る詰所掲示にはもう一段大きくしたい。
- T-2: 系統2 `/signage` はタブレットだと xl(1280px)未満のためダッシュボードのグリッドが単列化し、本来の2カラムサイネージにならない。タブレット横向きでも「TVレイアウト」を選べるトグルが無い。
- T-3: タッチ操作のシナリオプリセット（系統2）は min-h-[36-44px] 確保済みで良好。系統1のカウントダウン開始ボタンも大きい。
- T-4: キオスク運用（タブレットを朝礼端末固定）時のオートロック/スリープ抑止が無い（Wake Lock 未使用）。

## タブレット総評
系統1はタブレット朝礼に好適。6桁コード運用の体験が中核だが**本番不通**が最大の阻害。Wake Lock とフォント上限引き上げが効く（Phase C候補）。
