# 06 大画面実機検証（50-80インチTV / 1920px〜4K相当）（軸6）

検証手段: Playwright で本番 /ky/morning を 1920×1080 実レンダリング。スクショ: `screenshots/largescreen-1920.png`。

## 実測値（系統1, 1920px）— 重大課題
- 横スクロール: なし
- **コンテンツ幅: 1280px のみ**（`max-w-6xl`）→ 1920px画面で**左右に計640px（各320px）の未使用黒帯**。
- 「本日の主な作業」見出し: **48px のまま**（`sm:` 以上のブレークポイント指定が無く、lg/xl/2xl で拡大しない）。
  - 50〜80インチTVを朝礼で数m離れて見る場合、48pxは小さい。視認距離の目安では本文を画面高の3〜5%以上にしたい。

→ つまり系統1の朝礼サイネージは「大画面に映すと中央に小さく表示され、周囲が真っ黒」になる。
CLAUDE.md のレイアウト方針「サイネージはフルスクリーン前提」と実装が乖離している（朝礼サイネージは (main) レイアウト内の通常ページとして max-w-6xl 制約を受けている）。

## 系統2 `/signage` の大画面
- `xl:grid-cols-12` の2カラムダッシュボードが 1280px以上で発火 → 1920pxでは機能する。
- ただしこちらも全幅を使うが要素が小さめ（情報密度優先）。4K(3840px)での文字スケールは未最適化。

## 4K（3840px）想定
- 系統1: 1280px中央表示のまま → 画面の1/3しか使わず実用外。
- 系統2: グリッドは広がるが rem 固定フォントで相対的に小さくなる。CSS の clamp()/vw 単位や container query 由来の自動スケールが無い。

## 課題まとめ
- L-1（P0級）: 系統1 /ky/morning の `max-w-6xl` 撤廃＋大画面でのフォント自動拡大（lg/xl/2xl もしくは clamp/vw）。事務所・休憩室常設TVの主役に。
- L-2: フルスクリーンAPI導線（リモコン操作前提のTVでもブラウザのフルスクリーンが望ましい）。
- L-3: 4K向けに clamp() ベースの流体タイポgrafィ。視認距離からの逆算で本文最小サイズを規定。
- L-4: 画面焼き付き対策（常設TVは同じ画面を何時間も表示）— 微小な要素移動や定期リフレッシュ。

## 大画面総評
**最大のギャップは系統1の大画面非対応**（中央1280px固定・フォント据え置き）。ここは「賛否なし・手戻りゼロ」で改善でき、効果が大きい（Phase C P0候補 L-1）。
