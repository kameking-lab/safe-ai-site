# 12 「他SaaSを使う理由」全件抽出（軸9/12）

現場が当社でなく他SaaS（Buildee/ANDPAD/SPIDERPLUS/らくらく現場/現場ポケット/汎用サイネージ）を選ぶ理由を全件洗い出し。

| # | 他SaaSに行く理由 | 当社方針との関係 | 対応 |
|---|---|---|---|
| O1 | 朝礼画面に当日工程表/図面を出したい | 系統2にworkdocs/図面あり、KY朝礼との統合は弱い | P2 統合 |
| O2 | CCUS連動の入退場・出面管理 | 当社方針「現場に何かさせない/認証なし」と相反 | 対象外（方針堅持） |
| O3 | 安全書類(グリーンファイル)の作成・期限管理 | 別機能領域。サイネージのスコープ外 | 対象外 |
| O4 | 電子小黒板付き写真・改ざん検知 | 施工管理領域。スコープ外 | 対象外 |
| O5 | 本部から複数現場を集中管理・一斉配信 | 6桁/URL共有で部分対応、集中管理は未実装 | P3 |
| O6 | 既に全社でANDPAD等を導入済み | 併用前提。無料・登録不要で“朝礼だけ当社”が成立 | 差別化で対応 |
| O7 | 多言語の安全掲示 | 当社で多言語トグル実装すれば優位に立てる | **P1/P2** |
| O8 | 休憩室での常設情報表示（ニュース/天気） | 系統2が既に対応 ✓ | 維持・統合 |
| O9 | きれいな見た目・ブランド統一 | デザイン磨き込みで対応可 | P2 |

## 結論
当社が**戦うべきでない領域**: O2(入退場/CCUS)・O3(安全書類)・O4(電子黒板) — これらは施工管理スイートの土俵で、CLAUDE.md「大型施工管理スイート化禁止」「現場に何かさせるUI追加禁止」に明確に抵触。深追いしない。

当社が**勝てる領域**: 「無料・登録不要・手持ち端末・6桁共有・日本語×多言語の朝礼/安全掲示サイネージ」。
他SaaSは高機能ゆえ重く有料。当社は“朝礼掲示だけを最速・無料・登録不要で”という一点突破。
O7(多言語)・O1(KYと工程/打合せの統合)・O8(常設情報)を磨けば、併用ニーズに食い込める。
