# 11 「専用サイネージ機に戻る理由」全件抽出（軸9/11）

現場が当社Webサイネージを捨て専用機（クェスタ/シャープ/パナソニック等）を選ぶ理由を全件洗い出し、対応可否を判定。

| # | 専用機に行く理由 | 当社で解消可能か | 対応 |
|---|---|---|---|
| R1 | 通電だけで自動再生してほしい（無人運用） | △ 近似可（フルスクリーン＋Wake Lock＋URL固定） | Phase C P1 |
| R2 | ネット断でも表示が止まらない | △ 部分可（Service Worker/PWAキャッシュ） | P2 |
| R3 | 朝礼中に画面が消える/スリープする | ○ Wake Lock API | **P0/P1** |
| R4 | 大画面TVに大きく映したい（離れて視認） | ○ max-w撤廃＋流体フォント | **P0** |
| R5 | フルスクリーンで余計なUIを消したい | ○ Fullscreen API | **P1** |
| R6 | 複数現場を本部から一斉更新したい | △ 6桁/URL共有で部分対応、集中管理は未対応 | P3 |
| R7 | 複数画面のフレーム同期 | ✕ ブラウザでは困難 | 対象外 |
| R8 | 輝度自動調整・焼き付き対策 | △ ダーク基調＋定期リフレッシュ＋微小モーション | P2 |
| R9 | 緊急/防災の一斉割込 | ○ 緊急表示モード内製 | **P1** |
| R10 | 多言語で外国人に伝えたい | ○ 多言語トグル＋AI翻訳 | **P1/P2** |
| R11 | 24時間連続稼働の堅牢ハード | ✕ ハード領域（汎用TV/タブレット運用の限界） | 対象外 |
| R12 | コンテンツ配信スケジュール管理(CMS) | △ 時刻自動モード切替で部分対応 | P1/P2 |

## 結論
専用機固有の優位は **R7（物理同期）/R11（堅牢ハード）** に集約され、これは原理的に追えない。
逆に R3/R4/R5/R9/R10 という「現場が実際に困っている点」は**全て安価なWeb標準/既存資産で解消可能**。
ここを潰せば、中小現場が専用機（初期数十万＋月額）に戻る合理的理由はほぼ消える。
