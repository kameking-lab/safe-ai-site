# 14 KY全面再設計から得た教訓のサイネージ適用（軸9）

KY/打合せ書の全面再設計（完成度88〜）で確立したパターンを、サイネージ改修でも踏襲する。

## 適用する教訓
1. **ローカルファースト＋背景同期**（storage-adapter.ts）
   - サイネージも localStorage / 静的データを真実とし、クラウド(6桁共有)は保険。
   - クラウド障害（現在の本番Supabase不通）でも表示は止めない設計を徹底。
2. **クラウド未設定でも壊れない**（503→localStorage）
   - 新モード（緊急/休憩/終業）は**クラウド非依存**で作る＝手戻りゼロ・本番障害の影響を受けない。
3. **AI提案の免責・出典明示**（KY_SUGGEST_DISCLAIMER/grounded/basis）
   - 多言語翻訳・AI生成は「参考」明記を継承。
4. **純粋関数＋注入可能トランスポート**（normalizeKyInstructionRecord / KyCloudTransport）
   - ロジックを純粋関数化しユニットテストを厚く。新モードのデータ整形も同様に。
5. **用紙ファースト/一本化**（/ky, /pdf を /ky/paper へ集約）
   - サイネージも「2系統並存」を解消し、入口を1つに集約する方向で整理（段階的に）。
6. **賛否なし・手戻りゼロを優先**
   - レイアウト/フォント/スリープ抑止/フルスクリーン等、議論不要で効果が出る改善から着手。

## サイネージ特有の追加原則
- 「現場に何かさせない」: 閲覧専用。入力・ログインを現場作業員に求めない。
- 「フルスクリーン前提」(CLAUDE.md): 系統1の max-w 制約を外し、サイネージらしい全画面に。
- テスト維持: 既存テスト（約772件想定）を壊さず、新規は純粋関数＋コンポーネントに対して追加。
