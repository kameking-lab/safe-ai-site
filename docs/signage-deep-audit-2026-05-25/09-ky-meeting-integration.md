# 09 KY・打合せ書との統合体験（1日フロー）（軸9）

## 想定する1日フロー
1. 前日昼: 打合せ書（`/safety-diary`）で翌日の作業・各社調整を作成。
2. 当日朝: 職長がKY用紙（`/ky/paper`）を作成 → 「サイネージへ」で `/ky/morning` 全画面 → 全員で唱和。
3. 別端末共有: 6桁コードを共用タブレット/詰所TVに入力して同じKYを表示。
4. 作業中: 詰所TVで `/signage`（気象警報・労災ニュース・法改正）を常設。
5. 終業: 本日ゼロ災害の確認・翌日の打合せへ。

## 現状の接続状況
- KY用紙 → 朝礼サイネージ: `ky-paper-view` に「サイネージへ」「別端末で共有」あり ✓
- 打合せ書 → サイネージ: **直接導線が弱い**。打合せ書を全画面共有する朝礼前モードが無い（系統2のworkdocsは図面/書類アップロードで、打合せ書連携ではない）。
- 朝礼サイネージ(系統1) ↔ 現場ダッシュボード(系統2): **相互リンクがほぼ無い**。朝礼後に「現場ダッシュボードへ」遷移する導線が無い。
- 6桁コード共有が**本番不通**（Phase A）で、ステップ3が現状成立しない。

## ギャップ（統合体験）
- I-1: 打合せ書サイネージモードが無い（前日昼の全社共有・朝礼前の打合せ確認に使える）。
- I-2: 系統1↔系統2の往来導線が無く、ユーザーは2つの別物を行き来できない。
- I-3: 「時刻でモード自動切替（朝=朝礼/昼=打合せ/夕=終業）」が無い。常設TVの理想は無人で1日を回ること。
- I-4: 終業時モード（本日ゼロ災害達成表示）が無い。

## あるべき統合
- 朝礼サイネージ・打合せ書サイネージ・現場ダッシュボードを「1つのサイネージ・ハブ」配下のモードとして束ね、相互遷移と時刻自動切替を持たせる。
- 6桁コードを KY だけでなく打合せ書にも拡張（同一の signage_sessions 基盤を流用可能）。

## 効果
1日を通して詰所TV/共用タブレットが無人で「朝礼→作業中情報→終業」を回せると、専用サイネージ機・他SaaSを使う理由が大きく減る（軸11/12）。
