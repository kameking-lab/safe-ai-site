# 13 改修案の優先順位付け（軸10）

各案: 対象/問題/改修内容/工数/流用パターン/「賛否なし手戻りゼロ」判定。
工数目安: S=半日, M=1日, L=2日+。

## P0（即実装・効果大・賛否なし・手戻りゼロ・クラウド非依存）

### P0-1 大画面フルスクリーン化（max-w撤廃＋流体フォント）
- 対象: 系統1 `/ky/morning`（ky-morning-signage.tsx）
- 問題: 1920pxでコンテンツ1280px固定・左右黒帯640px、見出し48px据え置き（実測・doc06）。
- 改修: サイネージ用ラッパで max-w を撤廃しフル幅化。見出し/リスクを `clamp()`/vw で大画面ほど拡大。
- 工数: M / 流用: 既存Tailwind＋clamp / **賛否なし手戻りゼロ: YES**

### P0-2 画面スリープ抑止（Wake Lock API）
- 対象: 系統1（および系統2）
- 問題: 朝礼/常設中に画面が暗転（doc04 S-2, doc11 R3）。専用機への明確な弱点。
- 改修: `navigator.wakeLock.request("screen")` をサイネージ表示中に取得、可視性復帰で再取得。非対応ブラウザは無害フォールバック。
- 工数: S / 流用: signage-today-documents の既存利用例 / **賛否なし手戻りゼロ: YES**

## P1（高優先・賛否なし・主にクラウド非依存）

### P1-1 フルスクリーンAPI導線
- 対象: 系統1/2。問題: ブラウザUIが残る（doc04 S-1, R5）。
- 改修: 「全画面表示」ボタン → `requestFullscreen()`。退出も対応。
- 工数: S / **手戻りゼロ: YES**

### P1-2 QRコード併設（6桁コード共有の体験改善）
- 対象: ky-paper-view（共有）/ ky-morning（入力案内）。
- 問題: 6桁は通知テキストのみ（doc01）。別端末入力が手間。
- 改修: 6桁コードと並べてQR（`/ky/morning?code=XXXXXX` へのQR）を表示。QRは軽量ライブラリ or 自前SVG生成。
- 工数: M / 注: クラウド(6桁)復旧前でもUI実装・テスト可。 / **手戻りゼロ: YES**

### P1-3 表示モード追加（休憩啓発/終業ゼロ災害/緊急/イベント告知）
- 対象: 系統1にモードハブを追加（doc08）。
- 問題: 朝礼のみ。1日を通した常設運用ができない。
- 改修: モード切替UI＋各モード（全てクラウド非依存・静的/localStorage設定）。緊急モードは連絡先/避難経路を現場編集可。
- 工数: L / 流用: 系統2のシナリオプリセットUI / **手戻りゼロ: YES（既存朝礼は既定モードのまま）**

### P1-4 多言語トグル（固定ラベル辞書・API不要）
- 対象: 系統1表示ラベル。問題: 全日本語（doc07）。
- 改修: 「本日の主な作業/リスク/対策/唱和」等の固定ラベルを5〜6言語辞書化＋言語トグル。KY本文は日本語のまま（翻訳併記はP2）。
- 工数: M / **手戻りゼロ: YES**

### P1-5 唱和音声・サウンド強化
- 対象: 系統1カウントダウン。問題: 880Hzビープ1回のみ（doc04 S-3）。
- 改修: 開始/カウント/唱和の段階的サウンド、任意で SpeechSynthesis による行動目標の読み上げ。音量・ON/OFF設定。
- 工数: M / **手戻りゼロ: YES（既定OFF or 控えめ）**

### P1-6 時刻による自動モード切替
- 対象: モードハブ。改修: 朝=朝礼/日中=休憩啓発/夕=終業を時刻で自動選択（設定で無効化可）。
- 工数: S / **手戻りゼロ: YES（既定OFF推奨）**

### P1-7 印刷/PDFモード
- 対象: 系統1。改修: サイネージ内容をA4印刷用CSSで出力（掲示板貼り出し用）。KY用紙印刷の `@media print` 流用。
- 工数: S / **手戻りゼロ: YES**
