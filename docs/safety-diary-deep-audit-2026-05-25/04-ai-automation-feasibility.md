# 04 AI自動化の技術的実現可能性

> 既存KYの実装（`/api/ky/suggest` ＝ Gemini 2.5 Flash＋150件RAG、`evalScore()`、`estimateQualifications()`、チャットボット安衛法RAG）を**どこまで流用できるか**をコード根拠で判定。コスト試算は概算（**要確認**＝実トークンは作業内容長で変動）。

調査時刻: 2026-05-25

---

## 自動化6項目 × 流用可否

| 項目 | 既存資産 | 流用度 | 追加開発 |
|---|---|---|---|
| **① 必要資格 推論** | `presets.estimateQualifications()`（正規表現）＋業種プリセット | 高 | 業者行ごとに適用＋Gemini補完（曖昧作業）。低コスト |
| **② 予想災害 提案** | `/api/ky/suggest`（Gemini RAG・危険箇所を返す） | **最高** | ほぼそのまま。出力を打合せ書の"予想災害"列へ |
| **③ リスク評価（重大性×可能性×優先度）** | KYの `evalScore()`＋等級判定（Phase5） | 高 | **優先度（I〜IV区分）の算出ルールを追加**（評価値→区分マップ） |
| **④ 安全衛生指示事項 生成** | 無（新規） | 中 | **新規プロンプト**。作業内容＋予想災害→指示文を生成 |
| **⑤ 点検項目 自動チェック** | 無（新規） | 中 | 作業内容→該当カテゴリ/項目を推論（ルール＋Gemini） |
| **⑥ 法令・指針 引用** | チャットボット安衛法RAG（`/api/chatbot` Phase1-4）＋通達URL添付 | 高 | 指示事項に根拠条文を1-2件添付（既存検索を内部呼び） |

→ **②③⑥は既存をほぼ流用、①は既存強化、④⑤のみ新規プロンプト**。技術的実現性は **高い**。新規はGeminiプロンプト2本（指示文・点検推論）に集約できる。

---

## 新規プロンプト設計案

### ④ 安全衛生指示事項生成（Gemini 2.5 Flash）
入力: 作業内容（業者別配列）／予想災害／リスク評価（重大性・可能性）／天候・気温
出力JSON:
```json
{ "instructions": [
  { "contractor": "とび工", "instruction": "足場最上層は親綱2丁掛け徹底。強風時(10m/s超)は組立中止。", "basis": "安衛則518条" }
] }
```
- 制約: 1業者あたり1-2文・現場語・命令形。`basis` は⑥で安衛法RAGに内部問い合わせて付与（無ければ空）。
- フォールバック: KYと同様、API失敗時は定型文（業種×災害テンプレ）を返す。**捏造条文は出さない**（basisは検索ヒット時のみ）。

### ⑤ 点検項目 自動チェック（Gemini or ルール優先）
入力: 作業内容文字列／使用機械リスト
出力JSON:
```json
{ "checked": { "掘削": ["土止め支保工","酸欠測定"], "クレーン":["定格荷重表示","アウトリガー"], "足場":["手すり","幅木"] },
  "reason": { "土止め支保工": "「掘削」を検出" } }
```
- **まずキーワード→カテゴリのルール表で確定**（掘削/クレーン/足場/電気/危険物…）、曖昧分のみGemini。8カテゴリ・各項目に ○（該当・実施）/該当無 を初期セット、×は人が当日入力。
- これは「現場に入力させない（教訓3）」の核心＝**元請が前日にAIチェック済み状態を確認するだけ**。

## コスト試算（概算・要確認）

前提: Gemini 2.5 Flash、1打合せ書作成で AI呼出 = ②災害＋④指示＋⑤点検＋⑥法令 ≒ **4コール**（①資格はルールで無料）。1コール平均 入力1.5k＋出力0.7kトークンと仮定。

- 1打合せ書 ≒ 4コール ≒ 入出力合計 ~9kトークン
- 1現場が毎営業日作成 = 月20枚 → 80コール/月/現場 ≒ 0.18Mトークン/月/現場
- Flash料金（要確認・概算 $0.15/1M入力・$0.60/1M出力級）→ **月数十円/現場オーダー**。サーキットブレーカ＋IP制限（KY既存 `/api/ky/suggest` の10回/分）を流用すれば暴走なし。

→ **コストは無視できる水準**。KYで導入済みのレート制限・フォールバック・grounded判定をそのまま使えば安全。

---

## 強い判定
- **②③⑥は即流用、④⑤の新規プロンプト2本だけ作ればAI自動化は完成**。技術リスク低。
- **点検項目⑤こそ最大の差別化**（Buildeeも紙も「人が30項目チェック」を強いる。本サイトはAIが前日に下チェック）。
- **捏造防止**: 法令basisは検索ヒット時のみ付与、未ヒットは空欄。リスク区分は数式（決め打ちしない）。KYの「定型フォールバック・grounded表示」を踏襲。
