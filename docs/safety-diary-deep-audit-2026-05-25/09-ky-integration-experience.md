# 09 KY機能との統合体験（1日フロー）

調査時刻: 2026-05-25 / コード根拠: `from-ky.ts`, `deep-link-prefill.ts`, `today-safety-dashboard`

---

## 重要な気づき：打合せ書とKYの「向き」は逆

社長要件の整理：
- **打合せ書（元請・前日作成）** → 翌朝、各社職長がそれを見て **KY（下請・朝作成）** → 当日作業 → 当日 **実績/実施状況を打合せ書に追記**。
- 現状の `from-ky.ts` は **KY→安全衛生日誌**（職長が自分のKYを当日ログに流す）という**別フロー**。打合せ書フローの「打合せ書→各社KYの配布起点」「当日結果→打合せ書の実施状況」は**未実装**。

## あるべき1日フロー（打合せ書中心）

| 時間 | 主体 | 書類 | 連携 |
|---|---|---|---|
| 前日夕方 | 元請 現場代理人 | **打合せ書を作成**（AIで災害/資格/点検/指示/法令） | 前回打合せ書を複製→日付変更 |
| 前日〜当朝 | 元請→各社 | 打合せ書を **6桁コード/リンクで配布** | KYパターンH流用 |
| 当朝 | 各社 職長 | **KY作成**（打合せ書の自社行＝作業内容・予想災害が初期値） | 打合せ書→KY転記（**新規・要実装**） |
| 当日 | 各社 | 作業・ヒヤリ | |
| 当日夕 | 各社→元請 | **実績人員・実施状況・サインを返す** | KYの承認/提出パターン(P1-B)流用 |
| 翌日 | 元請 | 次の打合せ書（実績を踏まえ） | |

## 現状の統合度評価：45 / 100
- ◎ 双方向リンクの土台はある（`?fromKy` / `?fromDiary`）
- △ 転記が**文字列の流し込み**（構造化されていない。打合せ書の各社行→KYの作業行、の構造マッピングが無い）
- ✕ **共通マスター未活用**（KYは `worker_master`／業者名を持つが日誌・打合せ書は使っていない＝二重入力）
- ✕ **クラウド分断**（KYはSupabase対応、日誌はlocalStorageのみ＝同じ現場のデータが別世界）
- ✕ 当日実績の打合せ書フィードバック動線なし

## 強い判定
1. **共通マスター（業者・作業員・現場）をKYと打合せ書で完全共有**。二重入力をゼロに（教訓3）。
2. **打合せ書→KY配布**を新設（打合せ書の自社行がKYの初期値に）。KYパターンH（6桁共有）＋C（自動取得）流用。
3. **当日実績フィードバック**はKYのP1-B承認/提出ロックを流用（協力会社が実績を返すと打合せ書がロック→元請確認）。
4. **打合せ書もSupabase保管**（KYと同じ `device_id` モデル＋localStorageフォールバック）。これ無しに「朝礼→現場→事務所」の1日は回らない。
