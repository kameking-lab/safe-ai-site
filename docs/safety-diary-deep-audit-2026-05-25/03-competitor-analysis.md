> **方針（教訓2遵守）**: 実在確認できたサービスのみ記載。各社に **出典URL＋調査時刻** を付す。スペック/料金は公式ページ要旨ベースで、数値は **要確認**（変動するため）。創作・架空サービスは載せない。

## 最重要結論

**「安全工程打合せ書／作業間連絡調整（調整会議）」の直接競合は Buildee（リバスタ）一強。しかも当該機能は無料で、700社超のゼネコンが利用中。** グリーンサイト/Greenfile.workは"提出用のグリーンファイル"中心で日次の打合せ書とは土俵が違う。ANDPADは高額な統合スイート。現場ポケットは日報中心。
→ 本サイトが狙うべきは **「Buildeeのエコシステム（元請会員登録＋協力会社巻き込み）が重すぎる中小ゼネコン」**。登録不要・無料・AIで前日サッと1枚・北海道労働局様式特化・自社書式にカスタム可、で差別化する。

## 各社の所見（打合せ書視点）

### Buildee 調整会議（最警戒）
- 「当日作業状況報告→現場巡視の指摘・是正→翌日作業調整（作業予定・車輛搬出入・揚重機・機材・現場配置図）」をリアルタイムに回す＝**まさに打合せ書/調整会議のデジタル版**。安全衛生日誌・現地KY記録表・作業間連絡調整書を含む30超帳票が**無料**。
- 弱点（本サイトの勝ち筋）: **元請が会員登録し協力会社も巻き込むエコシステム型**で導入が重い。中小・スポット現場・「とりあえず1枚作りたい」には過剰。AIによる自動起票（災害/資格/点検）は前面に出ていない（要確認）。
- 出典: [機能](https://service.buildee.jp/spec/meeting/) [料金](https://service.buildee.jp/price/meeting/) [導入事例](https://service.buildee.jp/customer/case/meeting/)（2026-05-25確認）

### グリーンサイト / Greenfile.work
- **提出用グリーンファイル（再下請通知書・作業員名簿・持込機械等）が主戦場**で、日々の打合せ書とはレイヤーが違う。両者を混同しないこと。中小は「グリーンサイト有料が嫌でExcel」という層が分厚い（→本サイトの紙ユーザー獲得余地）。
- 出典: [グリーンサイト](https://www.gs.kensetsu-site.com/) [Greenfile.work料金](https://greenfile.work/prices)（2026-05-25確認）

### ANDPAD / 現場ポケット / KANNA
- ANDPAD=見積〜引渡し一気通貫の基幹スイート。打合せ書単体目的にはオーバースペック＆高額。現場ポケット=日報・写真のシンプル管理（打合せ書様式は持たない）。KANNA=無料施工管理だが安全帳票特化ではない。
- 出典: [ANDPAD](https://andpad.jp/) [現場ポケット料金](https://gempo.info/1144) [KANNA](https://lp.kanna4u.com/)（2026-05-25確認）

## 「他サービスを使う理由」になっている差（要改修）
1. **クラウド保管・複数端末**: Buildee/ANDPADは当然クラウド。本サイトはlocalStorageのみ＝**致命的に負けている**。
2. **協力会社との共有・実績フィードバック**: Buildeeは協力会社が当日実績を返す。本サイトは6桁共有(KY)はあるが日誌側未対応。
3. **帳票の網羅**: Buildeeは30超。本サイトは日誌1種＋打合せ書ゼロ。
4. **元請700社の安心感（実績）**: 個人サイトの信用差。→ 法令引用・北海道労働局様式準拠・コンサル監修で補う。

## 本サイトだけが取れる差（伸ばすべき）
1. **登録不要・無料・即印刷**（Buildeeの会員登録・協力会社巻き込みが要らない）。
2. **AIで前日5分起票**（災害・資格・点検項目・指示文・法令引用を自動）。Buildeeに無い切り口（要確認）。
3. **自社書式へのカスタマイズ**（行列追加＝05番docs）。
4. **KYとの一気通貫**（朝KY→当日作業→夕方実績→翌日打合せ書）。
