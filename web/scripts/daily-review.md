# Daily Review 巡回ルール

## 概要

毎日、2グループのペルソナが交互にサイト全ページを巡回し、改善点を洗い出す。
プロジェクトマネージャー（PM）ロールが意見を集約し、軽微な修正はその場で実施、
大きな変更は提案としてまとめて GitHub Issue に投稿する。

## PM（プロジェクトマネージャー）ロール

PMはペルソナではなく、5人の意見を集約して最終判断を下す役割。

### PM の責務
1. 5ペルソナの意見を整理・統合する
2. 重複・矛盾する意見を調整する
3. 修正可否を判断する（下記基準参照）
4. 軽微な修正を実施してコミットする
5. 提案事項を構造化してレポートにまとめる
6. 総合スコア（5点満点）を算出する
7. GitHub Issue を作成する

### 修正可否の判断基準

#### その場で修正してよい（軽微）
- テキスト修正、タイポ、誤字脱字
- リンク切れ
- CSS崩れ、余白・フォントの不統一
- lint エラー、TypeScriptエラー
- メタデータ不備（title, description, OGP）
- アクセシビリティの基本対応（alt属性、aria-label、色コントラスト）
- モバイル/PC の表示崩れ
- 不要な開発者向け表示の削除
- パフォーマンス改善（画像最適化、不要なre-render）
- SEOの基本改善（見出し構造の修正、内部リンク追加）

#### オーナーに提案するだけ（判断待ち）
- 新ページ追加
- 新API接続、外部サービス連携
- URL構造の変更
- データベーススキーマ変更
- デザインの大幅リニューアル
- 課金・認証関連
- 新しい環境変数の追加
- 既存機能の削除や統合
- コンテンツの大幅追加（新規記事、データ追加）
