# BACKLOG — 無人自走ループのタスクキュー（Fable 5・2026-06-10〜）

このファイルは loop-runner.ps1 が各イテレーションで参照するタスクキュー。
各起動で **未着手(`[ ]`)の最上位タスクを1つ**取り出して実行し、完了したら `[x]` に更新する。
**未着手が3件未満になったら、実行前に柱1〜3から新タスクを自分で起こして補充**（空にして止めない）。
ミッション全文・契約・絶対ルールは loop-prompt.txt を参照。

## 運用ルール（毎イテレーション厳守）
- まず CI緑の自分の未マージPRをsquashマージ→`git checkout main && git pull --ff-only`→clean確認（前イテレーションのPR回収）。衝突時は origin/main をブランチへ通常マージ。
- 次に未着手タスクを1つ実行。1ブランチ→ゲート(tsc0/lint errors0/test全pass/build成功)→push→PR作成。
- 完了タスクを `[x]` にし、cycle-log(docs/court-cases-db-2026-06-06/cycle-log.md)に記録。
- main常時デプロイ可能・各始終clean・安全タグ fable5-loop-start-2026-06-10（既存タグも）維持。捏造0・水増し0・既存破壊0。env/DB独断変更禁止(Path A)。main直接コミット禁止。CI緑のみマージ。

## 第三者レビュー方式（柱3および機能改善タスクの標準手順）
1. 対象機能に対し、厳しい第三者ペルソナを具体的に生成（性格・前提・我慢の限界）。
2. そのペルソナになりきり Playwright で実操作（dev server か本番URL）。screenshotで確認。
3. 「面倒/分かりにくい/紙の方が速い」点を忖度なく全部書き出す。
4. インパクトの大きい欠点を改善実装。
5. 再度ペルソナ目線で操作し「これなら使う」と言えるまで磨く。
6. docs/third-party-reviews/ に（ペルソナ・指摘・改善・再評価）を記録。
