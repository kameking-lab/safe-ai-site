# BACKLOG — 無人自走ループのタスクキュー

このファイルは loop-runner.ps1 が各イテレーションで参照するタスクキュー。
各起動で **未着手(`[ ]`)の最上位タスクを1つ**取り出して実行し、完了したら `[x]` に更新する。
**キューの未着手が3件未満になったら、実行前に自分で新しい改善タスクを補充してから続ける**（空にして止めない）。

## 運用ルール（毎イテレーション厳守）
- まず CI緑の自分の未マージPRをsquashマージ→`git checkout main && git pull`→clean確認（前イテレーションのPRを回収）。
- 次に未着手タスクを1つ実行。1ブランチ→ゲート(lint0/tsc0/test全pass/build成功)→push→PR作成。
- 完了タスクを `[x]` にし、cycle-log(docs/court-cases-db-2026-06-06/cycle-log.md)に記録。
- main常時デプロイ可能・各始終clean・安全タグ all-features-loop-start-2026-06-05 維持。架空0・水増し0・既存破壊0。env/DB独断変更禁止(Path A)。
- main直接コミット禁止（必ずブランチ）。テスト落ちはマージしない。

## 第三者レビュー方式（軸2の各タスクで標準）
1. 対象機能に対し、厳しい第三者ペルソナを具体的に生成（性格・前提・我慢の限界）。
2. そのペルソナになりきり Playwright で実操作（dev server か本番URL）。screenshotで確認。
3. 「面倒/分かりにくい/紙の方が速い」点を忖度なく全部書き出す。
4. インパクトの大きい欠点を改善実装。
5. 再度ペルソナ目線で操作し「これなら使う」と言えるまで磨く。
6. docs/third-party-reviews/ に（ペルソナ・指摘・改善・再評価）を記録。
