# AGENTS.md

## このプロジェクトのゴール
安全AIサイトを「現場運用ポータル」として継続拡張し、朝礼・常時表示・個人確認の3用途で使える状態にする。
現フェーズの主対象は以下。
- 法改正一覧
- AI要約表示
- 質問チャットUI
- 今日の現場リスク（地域選択・作業種別）
- 事故DB
- /signage（サイネージ/朝礼モード）
- 第2弾たたき台（通知、メール配信、Eラーニング、KY用紙、PDF、マイク入力、事故DB拡張、ホーム再構成）

## 最重要ルール
- 必ず PRD.md と TASKS.md を読んでから作業開始する
- TASKS.md の順番を守る
- 1タスクずつ進める
- タスク完了ごとに progress.md を更新する
- エラーが出たら自分で修正して続行する
- ユーザー確認なしで進める
- 迷ったら現フェーズの目的に対して小さく実装する
- 既存導線を壊す大改造は避ける
- 動作確認を優先する
- UIはスマホ優先で作る
- 実装後は変更ファイルを簡潔にまとめる

## 技術ルール
- Next.js App Router を使う
- TypeScript を使う
- Tailwind CSS を使う
- コンポーネントを分ける
- ハードコードしすぎず、将来API差し替えしやすくする
- ダミーデータでよい
- APIキーや秘密情報は絶対に埋め込まない

## 作業ルール
- 作業前に短い実行計画を書く
- 作業後に何をやったかを progress.md に記録する
- テストや起動確認をできる範囲で行う
- 失敗した試行は簡潔に記録する
- 詰まったら次善策で前進する
- 不必要なライブラリは追加しない
- 未完成機能でも、レビュー可能な導線・文言・状態表示まで作る
- mock/live の境界は service と types に寄せ、ページ直書きを増やしすぎない

## 禁止事項
- 認証の追加
- 課金機能の追加
- DB本接続の追加
- 外部APIの本接続
- APIキーや秘密情報の埋め込み
- 破壊的な既存仕様変更（既存トップ・既存導線の無断削除）

## 現フェーズで許可する範囲
- 必要なページ追加（例: Eラーニング、KY用紙）
- 音声入力のクライアント側たたき台（ブラウザAPI + 手入力フォールバック）
- PDF出力の軽量たたき台（プレビュー、印刷、将来差し替え可能な構造）
