【唯一のゴール】安全関連ユーザーに日本一使いやすいサイト。あなたの担当は「社長がプロンプトを投げなくても診断→起票→実装→出荷が回り続ける運用機構」＝点火(launcher/スケジューラ)・補給(planner)・点検(critic)・報告(loop-status)の4系統。設計正本は docs/fable-diagnosis-2026-07-02/08-autonomous-operations.md。

- リポジトリ直下: loop-runner.ps1 / loop-launcher.ps1 / loop-config.json / loop-prompt*.txt / BACKLOG*.md の運用機構部分（各レーンのタスク中身は当該レーンの所有＝needed時は冒頭注入のみ・既存行の書換え禁止）。

- 掃除系タスクに限り web/src を触ってよいが、他レーン稼働中はスキップして次のタスクへ。

【タスク源】BACKLOG-ops.md の未着手[ ]最上位を1つ。3件未満なら運用4系統の欠陥から補充（機能改善は積まない）。

【各イテレーションの契約】
1) 自分のCI緑の未マージPRを squashマージ → `git checkout main && git pull --ff-only` → clean確認。
2) BACKLOG-ops.md の最上位タスクを1つ、ブランチ接頭辞 `ops/` で着手。
3) ゲート: PowerShellスクリプトは -WhatIf/ドライラン相当の自己検証を付ける。web/src を触った場合のみ `cd web` で tsc/lint/vitest/build の全ゲート。スケジューラ変更は登録内容を `Get-ScheduledTask | Format-List` で実測記録。
4) commit(末尾に Co-Authored-By 行・モデル名はセッションの実モデル)→push→`gh pr create`。CI待ちは次イテレーションで回収。
5) BACKLOG-ops.md を[x]更新、docs/court-cases-db-2026-06-06/cycle-log-ops.md に1段落記録。working tree clean で終える。
5.5) 【報告一元化】イテレーション末尾に `powershell -NoProfile -ExecutionPolicy Bypass -File ./loop-report-status.ps1 -Lane ops -Note "<直近の判断1行>"` を実行し docs/loop-status.md の自レーン行（最終稼働・直近マージPR・残open・判断1行）を更新（gitignore対象・環境変数 SAFE_AI_LOOP_STATUS で集約先へ自動ルーティング・失敗は非致命なので気にせず次へ）。
5.6) 【死んだレーンの復活=常駐watchdog】報告の直後に `powershell -NoProfile -ExecutionPolicy Bypass -File ./loop-launcher.ps1 -HealOnly` を実行。ops は常時生存する唯一のレーンなので、個別に死んだ（runnerがwedge/クラッシュした）他レーンをここで即復活させる＝次のログオン/07:00 launcher pass（最悪~22h）を待たない。-HealOnly は死んだレーンのrunner再起動のみ（deadline/planner/critic/status/git は一切触らない・生きているレーンは runner 自身の単一インスタンスガードで自動 no-op・processスキャンが0件なら二重起動防止で何もしない）＝安全。失敗は非致命。
5.7) 【生成品質evalの夜間自動測定=O18】healの直後に `powershell -NoProfile -ExecutionPolicy Bypass -File ./loop-eval-nightly.ps1` を実行。ops は常時生存する唯一のレーンなので、社長のプロンプト無しで本番23問の生成品質evalを1日1回だけ自動測定する（点火＝この班の担当）。スクリプトは自己冪等＝当日分の記録(.loop-eval/genquality/<date>.json)が既にあれば即no-opなので、~195s間隔で毎回叩いても実測は1日1回に収束＝本番Geminiは23問/晩に限定。evalは本番URLへのHTTP 23問なのでローカルCPUを食わず他レーンのbuildと競合しない（perf実測の静穏窓制約は非該当）。結果は機械可読で .loop-eval/genquality/ 配下(gitignore)へ日次追記。閾値未達のdocs/loop-status.md警告と/about併載はO18(b)。失敗は非致命。
